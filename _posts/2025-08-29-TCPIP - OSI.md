# 개방형 시스템 상호 연결 (OSI) 참조 모델 통합 가이드

## OSI 참조 모델의 역사 및 개요

개방형 시스템 상호 연결(OSI) 참조 모델은 1970년대 말부터 1980년대 초에 걸쳐 국제표준화기구(ISO)에 의해 개발된 개념적 프레임워크입니다. 초기 컴퓨터 네트워킹 환경의 "벤더 종속성" 문제를 해결하고 공통의 표준을 제공하기 위해 만들어졌습니다. 1984년에 발표된 이 모델은 실제 프로토콜 세트라기보다 네트워크 기능을 이해하고 설계하기 위한 **참조 모델**로 사용됩니다.

## 참조 모델의 중요성과 필요성

* **복잡성 관리**: 거대한 네트워크 통신 작업을 관리 가능한 7개의 계층으로 분해합니다.
* **상호 운용성**: 서로 다른 벤더의 제품이 공통 구조를 기준으로 개발되도록 유도하여 호환성을 높입니다.
* **문제 해결**: 장애 발생 시 특정 계층으로 문제를 격리하여 "세션 연결 끊김"이나 "전송 타임아웃" 등 정확한 진단이 가능하게 합니다.
* **학습 및 설계**: 새로운 기술이 어느 계층에서 작동하는지 파악하기 쉽게 하며, 시스템 설계의 틀을 제공합니다.

## OSI 참조 모델의 비유: 국제 우편 시스템

복잡한 통신 과정을 우편 시스템에 비유하면 다음과 같습니다:

1. **응용 (7L)**: 편지를 쓰는 사람.
2. **표현 (6L)**: 편지 번역 및 암호화.
3. **세션 (5L)**: 대화의 시작과 끝 관리.
4. **전송 (4L)**: 등기/일반 우편 결정 및 봉투 분할/재조립.
5. **네트워크 (3L)**: 주소 분석 및 최적 경로(우체국 경유지) 결정.
6. **데이터 링크 (2L)**: 인접 우체국 간 운송 수단 결정 및 오류 검출.
7. **물리 (1L)**: 실제 편지가 실리는 트럭, 철로, 비행기 등 물리적 매체.

## 각 계층별 상세 설명

### 1. 물리 계층 (Physical Layer)

* **핵심 기능**: 비트 스트림의 물리적 전송.
* **설명**: 데이터를 전기, 광, 무선 신호로 변환하여 전송 매체를 통해 실제로 보내는 기초 계층입니다. 데이터의 의미는 관여하지 않고 0과 1의 신호 규정을 다룹니다.
* **주요 요소**: 케이블 종류, 커넥터 모양, 핀 배열, 전압 레벨, NIC, 허브(Hub).

### 2. 데이터 링크 계층 (Data Link Layer)

* **핵심 기능**: 동일 네트워크 내의 신뢰성 있는 프레임 전달.
* **설명**: 비트들을 **프레임(Frame)** 단위로 구성하며, **MAC 주소**를 사용하여 로컬 네트워크 내 목적지에 전달합니다. 오류 탐지와 매체 접근 제어(MAC)를 수행합니다.
* **주요 요소**: 흐름 제어, 스위치(Switch), 브리지.

### 3. 네트워크 계층 (Network Layer)

* **핵심 기능**: 서로 다른 네트워크 간의 경로 설정(Routing).
* **설명**: **IP 주소**를 사용하여 출발지부터 목적지까지 최적의 경로를 결정하고 데이터를 **패킷(Packet)** 단위로 캡슐화합니다.
* **주요 요소**: 라우팅 테이블, 조각화(Fragmentation), 라우터(Router).

### 4. 전송 계층 (Transport Layer)

* **핵심 기능**: 종단 간(End-to-End) 신뢰성 있는 데이터 전송.
* **설명**: 특정 애플리케이션 프로세스까지 데이터를 정확히 전달하며, **포트 번호**를 사용합니다. 흐름 제어와 오류 제어를 수행합니다.
* **주요 프로토콜**: TCP(연결 지향, 신뢰성), UDP(비연결, 속도 중심).

### 5. 세션 계층 (Session Layer)

* **핵심 기능**: 통신 세션의 설정, 유지, 종료 관리.
* **설명**: 두 장치 간의 연결을 하나의 **대화(Session)**로 관리하며, 동기화 점(체크포인트)을 설정하여 통신 흐름을 조절합니다.
* **주요 요소**: RPC, NetBIOS.

### 6. 표현 계층 (Presentation Layer)

* **핵심 기능**: 데이터 형식 변환, 암호화, 압축.
* **설명**: 서로 다른 시스템이 이해할 수 있도록 공통 데이터 형식으로 변환(예: ASCII 변환)하거나 데이터의 암호화 및 압축을 담당합니다.
* **주요 요소**: JPEG, MPEG, SSL/TLS(초기 단계).

### 7. 응용 계층 (Application Layer)

* **핵심 기능**: 사용자 네트워크 서비스 인터페이스.
* **설명**: 사용자가 직접 상호작용하는 계층으로, 이메일, 웹 브라우징 등 실제 네트워크 서비스를 제공합니다.
* **주요 프로토콜**: HTTP, SMTP, FTP, SSH.

## 주요 통신 개념 및 구조

* **수직 통신(인터페이스)**: 동일 시스템 내 인접 계층 간의 통신이며 서비스 액세스 점(SAP)을 통해 이루어집니다.
* **수평 통신(프로토콜)**: 서로 다른 시스템의 동일 계층 간에 약속된 규약으로 대화하는 방식입니다.
* **데이터 캡슐화**: 상위 계층 데이터(SDU)에 헤더를 붙여 하위 계층의 PDU(세그먼트 → 패킷 → 프레임)를 만드는 과정입니다.
* **장비별 처리 계층**:
* 허브: 1계층만 처리.
* 스위치: 2계층까지 처리 (MAC 주소 기반).
* 라우터: 3계층까지 처리 (IP 주소 기반).



## 결론

OSI 7계층 모델은 실제 프로토콜인 TCP/IP 모델보다 먼저 구상된 이론적 틀이지만, 네트워크 통신의 전체 흐름을 이해하고 문제를 분석하는 데 있어 가장 명확한 지도를 제공합니다. 각 계층의 독립성과 책임을 이해하는 것은 효율적인 네트워크 설계와 문제 해결의 필수 기초입니다.
