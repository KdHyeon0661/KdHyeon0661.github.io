# 네트워킹의 모든 것: 기본 원리부터 OSI 모델까지

## 네트워킹 소개

### 네트워킹이란 무엇인가?

네트워킹은 컴퓨팅 장치들을 서로 연결하여 통신과 자원 공유를 가능하게 하는 실천적 기술입니다. 이 상호 연결된 시스템은 컴퓨터들이 데이터를 교환하고, 작업에 협력하며, 공유 하드웨어 및 소프트웨어 자원에 접근할 수 있도록 하여 현대 디지털 통신의 중추를 형성합니다. 간단히 말해, 네트워크는 개별적인 시스템들이 하나의 통합된 협력체로 작동할 수 있게 하는 디지털 고속도로 시스템입니다.

### 네트워킹의 장점

네트워크를 구축하는 주요 이점은 효율성과 협력의 향상에 있습니다. 가장 중요한 장점은 **자원 공유**로, 프린터나 스캐너 같은 고가의 하드웨어 장치를 여러 사용자가 공유할 수 있어 비용을 절감하고 공간을 효율적으로 활용할 수 있습니다. 또한 소프트웨어 애플리케이션과 데이터 파일을 중앙에서 관리 및 배포할 수 있어 일관성을 유지하고 관리 부담을 줄입니다.

**향상된 통신**은 또 다른 핵심 장점입니다. 네트워크는 이메일, 인스턴트 메시징, 화상 회의, 공동 작업 플랫폼을 통해 정보의 신속한 교환을 가능하게 하여 지리적 거리에 구애받지 않는 팀워크와 의사 결정을 촉진합니다.

**중앙 집중식 데이터 관리**를 통해 데이터를 한곳에 저장하고 정기적으로 백업할 수 있어 데이터 무결성, 보안, 접근성이 향상됩니다. 마지막으로, 네트워크는 추가 장치와 사용자를 수용할 수 있도록 확장이 가능하여 조직의 성장에 유연하게 대응할 수 있는 **확장성**과 중복 하드웨어 및 소프트웨어 라이선스의 필요성을 줄여 **비용 효율성**을 제공합니다.

### 네트워킹의 단점

네트워크의 이점은 명확하지만, 구현과 유지 관리에는 여러 가지 어려움이 따릅니다. 가장 큰 장애물은 **비용**입니다. 라우터, 스위치, 케이블, 서버 등 초기 하드웨어 구매 비용과 네트워크 운영 체제 및 관리 도구와 같은 소프트웨어 비용이 발생합니다. 또한, 유지 관리, 업그레이드, 네트워크 관리자나 보안 전문가와 같은 숙련된 인력에 대한 지속적인 비용도 고려해야 합니다.

**보안 위험**은 가장 심각한 단점 하나입니다. 네트워크에 연결된다는 것은 무단 접근, 데이터 도난, 맬웨어 감염, 서비스 거부 공격 등에 취약해질 수 있음을 의미합니다. 이는 방화벽, 암호화, 침입 탐지 시스템, 지속적인 모니터링 등 강력한 보안 조치에 대한 투자를 필요로 합니다.

**복잡성과 의존성**도 문제가 됩니다. 네트워크 시스템은 설계, 구성, 문제 해결이 복잡할 수 있으며, 네트워크에 한 부분이라도 장애가 발생하면 연결된 많은 사용자와 프로세스의 작업이 중단될 수 있습니다. 또한, 성능이 대역폭 제한, 정체, 지연 시간 등의 영향을 받을 수 있으며, 네트워크가 커질수록 성능 관리와 최적화가 더욱 어려워집니다.

## 네트워크의 기본 특성

효율적인 네트워크는 몇 가지 기본 특성을 바탕으로 구축됩니다. **신뢰성**은 네트워크가 지속적으로 작동하고 오류에 견고해야 한다는 것을 의미하며, **확장성**은 현재와 미래의 수요를 수용할 수 있도록 쉽게 확장될 수 있어야 함을 나타냅니다. **성능**은 처리량(데이터 전송 속도), 지연 시간(데이터 전송 지연), 그리고 지터(jitter, 지연 시간의 변동) 같은 요소들로 측정됩니다. 마지막으로, **보안**은 무단 접근과 오용으로부터 데이터와 리소스를 보호하는 지속적인 고려 사항입니다.

## 네트워킹 계층, 모델 및 아키텍처

복잡한 네트워크 통신을 관리하기 위해 네트워크 기능은 계층으로 나누어집니다. 각 계층은 특정한 책임을 가지며, 바로 위와 아래 계층과만 상호작용합니다. 이러한 모듈화된 접근 방식은 디자인을 단순화하고, 개발을 촉진하며, 서로 다른 벤더의 제품이 함께 작동할 수 있도록 합니다.

가장 유명한 참조 모델은 **OSI(개방형 시스템 상호 연결) 7계층 모델**입니다.

```
응용 계층 (7) - 사용자 인터페이스 (예: HTTP, FTP)
표현 계층 (6) - 데이터 번역, 암호화
세션 계층 (5) - 대화 관리
전송 계층 (4) - 종단 간 연결, 신뢰성 (예: TCP, UDP)
네트워크 계층 (3) - 논리적 주소 지정, 경로 지정 (예: IP)
데이터 링크 계층 (2) - 물리적 주소 지정, 오류 감지 (예: Ethernet)
물리 계층 (1) - 전기/광 신호, 케이블, 커넥터

```

실제 인터넷의 기반이 되는 것은 더 실용적인 **TCP/IP 4계층 모델**입니다.

```
응용 계층 - 사용자 애플리케이션 서비스 (HTTP, SMTP, DNS 등 포함)
전송 계층 - 호스트 간 통신 (TCP, UDP)
인터넷 계층 - 패킷의 경로 지정 및 전달 (IP, ICMP)
네트워크 인터페이스 계층 - 물리적 네트워크와의 통신

```

## 프로토콜: 그들은 정확히 무엇인가?

프로토콜은 네트워크 장치들이 통신하기 위해 따라야 하는 공식적인 규칙과 규약의 집합입니다. 이는 데이터의 형식이 어떻게 되어야 하는지, 오류를 어떻게 처리할 것인지, 메시지를 어떻게 확인할 것인지 등을 정의합니다. 서로 다른 제조사나 소프트웨어를 사용하는 장치들도 공통의 프로토콜을 사용하면 원활하게 상호작용할 수 있습니다. TCP, IP, HTTP, Ethernet 등이 모두 프로토콜의 예입니다.

## 회선 교환과 패킷 교환 네트워크

데이터가 네트워크를 통해 전송되는 방식에는 두 가지 근본적인 패러다임이 있습니다.

**회선 교환 네트워크**에서는 통신 세션을 시작하기 전에 발신지와 목적지 사이에 전용 통신 경로(회선)가 설정됩니다. 이 경로는 세션이 끝날 때까지 해당 연결만을 위해 독점적으로 사용됩니다. 전화 통신망(PSTN)이 고전적인 예입니다. 이 방식은 안정된 대역폭과 예측 가능한 성능을 보장하지만, 전용 경로가 유휴 상태일 때도 리소스가 낭비될 수 있어 비효율적입니다.

**패킷 교환 네트워크**에서는 전송할 메시지가 여러 개의 작은 조각인 패킷으로 분할됩니다. 각 패킷은 독립적으로 네트워크를 통해 전송되며, 목적지에 도착하면 원래의 메시지로 재조립됩니다. 패킷은 가용성에 따라 서로 다른 경로를 통해 이동할 수 있습니다. 인터넷이 이 방식의 대표적인 예입니다. 이는 네트워크 대역폭을 훨씬 더 효율적으로 활용하고 견고하지만, 패킷이 서로 다른 경로를 통해 이동할 경우 지연 시간이나 패킷 순서 불일치가 발생할 수 있습니다.

## 연결 지향형과 비연결형 프로토콜

전송 계층 및 기타 계층에서 프로토콜은 신뢰성에 대한 접근 방식에 따라 분류될 수 있습니다.

**연결 지향형 프로토콜**(예: TCP - Transmission Control Protocol)은 데이터를 교환하기 전에 발신지와 수신지 사이에 공식적인 연결을 설정합니다. 이 연결을 통해 데이터 전달의 신뢰성을 보장하며, 패킷 손실 시 재전송을 수행하고, 패킷이 올바른 순서로 도착하는지를 관리합니다. 이는 신뢰성이 중요하지만 약간의 오버헤드가 발생하는 파일 전송이나 웹 페이지 로딩에 적합합니다.

**비연결형 프로토콜**(예: UDP - User Datagram Protocol)은 사전 연결 설정 없이 데이터를 전송합니다. 각 데이터그램은 독립적으로 전송되며, 전달이나 순서를 보장하지 않습니다. 이는 속도와 효율성이 신뢰성보다 중요한 실시간 스트리밍, VoIP, 온라인 게임 같은 애플리케이션에 적합합니다.

## 메시지: 패킷, 프레임, 데이터그램 및 셀

네트워크를 통해 전송되는 데이터의 단위는 사용되는 프로토콜 계층과 기술에 따라 다양한 이름으로 불립니다.

* **패킷**: 네트워크 계층(IP)에서 사용되는 일반적인 용어입니다. 패킷은 논리적 주소(IP 주소)를 포함한 헤더와 실제 데이터인 페이로드로 구성됩니다.
* **프레임**: 데이터 링크 계층(예: Ethernet)에서 사용되는 데이터 단위입니다. 프레임에는 물리적 주소(MAC 주소)와 오류 검출을 위한 순환 중복 검사(CRC)와 같은 정보가 포함된 헤더와 트레일러가 있습니다.
```
[ Ethernet Frame ]
| 프리앰블 | 목적지 MAC | 출발지 MAC | 타입 | 데이터 (패킷) | CRC |

```


* **데이터그램**: 종종 비연결형 서비스(예: UDP)와 연관되어 사용되며, 전송 계층의 독립적인 데이터 단위를 지칭하는 데 사용됩니다.
* **셀**: 고정 길이의 작은 데이터 단위입니다. 비동기 전송 방식(ATM) 네트워크에서 사용되며, 빠르고 효율적인 스위칭을 가능하게 합니다.

## 메시지 형식: 헤더, 페이로드, 트레일러

네트워크 메시지는 일반적으로 다음과 같은 구조로 구성됩니다.

* **헤더**: 메시지의 시작 부분에 위치하며, 라우팅, 순서 지정, 오류 검출, 프로토콜 식별 등 메시지를 전달하고 처리하는 데 필요한 모든 제어 정보를 포함합니다. 이는 편지봉투의 주소와 우표와 같습니다.
* **페이로드**: 전송되는 실제 데이터나 메시지 내용입니다. 상위 계층에서 받은 데이터(예: 이메일의 일부 또는 웹페이지 조각)를 포함합니다.
* **트레일러(또는 푸터)**: 메시지의 끝 부분에 위치하며, 주로 순환 중복 검사(CRC)와 같은 오류 검출 코드를 포함하여 전송 중 데이터가 손상되지 않았는지 확인하는 데 사용됩니다. 모든 프로토콜이 트레일러를 사용하는 것은 아닙니다.

## 메시지 주소 지정 및 전송 방법: 유니캐스트, 브로드캐스트, 멀티캐스트

네트워크에서 패킷이 목적지로 전송되는 방식에는 세 가지 주요 유형이 있습니다.

* **유니캐스트**: 하나의 발신지가 네트워크 상의 **하나의 특정 목적지**에게 메시지를 전송하는 가장 일반적인 통신 형태입니다. 예: 웹 브라우저가 웹 서버에 페이지를 요청할 때.
* **브로드캐스트**: 발신지가 **동일한 네트워크 세그먼트 내의 모든 장치**에게 메시지를 전송합니다. 주소 확인 프로토콜(ARP)이나 라우팅 프로토콜 공지와 같이 로컬 네트워크 관리에 자주 사용됩니다. 라우터는 일반적으로 브로드캐스트를 다른 네트워크로 전달하지 않아 트래픽을 제한합니다.
* **멀티캐스트**: 발신지가 **특정 그룹에 가입한 장치들(멀티캐스트 그룹)** 에게 메시지를 전송합니다. 이는 동일한 데이터(예: 실시간 비디오 스트림, 주식 시세)를 여러 수신자에게 효율적으로 배포해야 할 때 유니캐스트의 비효율성과 브로드캐스트의 과도한 트래픽 사이의 균형을 제공합니다.

## 네트워크 구조 모델: 클라이언트/서버 및 P2P 네트워킹

네트워크의 논리적 배치는 주로 두 가지 모델로 구분됩니다.

**클라이언트/서버 모델**에서는 중앙 집중식인 강력한 컴퓨터인 **서버**가 파일, 이메일, 웹 페이지, 데이터베이스와 같은 리소스나 서비스를 제공합니다. **클라이언트**(예: 데스크톱, 노트북, 스마트폰)는 이러한 서버에 접속하여 리소스를 요청하고 사용합니다. 이 모델은 중앙 집중식 관리, 보안, 백업에 유리합니다. 대부분의 기업 네트워크와 웹이 이 모델을 기반으로 합니다.

**P2P(Peer-to-Peer) 모델**에서는 네트워크에 참여하는 각 장치(동등한 계층, '피어')가 클라이언트와 서버의 역할을 모두 수행할 수 있습니다. 피어들은 서버 없이 서로 직접 리소스(파일, 처리 능력, 대역폭)를 공유합니다. 이는 분산적이고 탄력적이지만, 보안, 관리, 검색 기능이 더 복잡할 수 있습니다. 파일 공유 네트워크(예: 초기 BitTorrent)와 일부분의 블록체인 네트워크가 이 모델을 사용합니다. 현대 시스템들은 경우에 따라 하이브리드 접근 방식을 사용하기도 합니다.

---

## 네트워크 분류: LAN, WLAN, WAN 및 그 변형

근거리 통신망(LAN)은 건물이나 캠퍼스와 같은 제한된 지리적 영역 내에서 장치를 연결합니다. 유선 이더넷이 일반적이지만, 무선 LAN(WLAN)은 Wi-Fi를 통해 유연성을 제공합니다. 광역 통신망(WAN)은 도시, 국가 또는 대륙 간과 같은 넓은 지역을 연결합니다. 인터넷이 가장 큰 WAN의 예입니다.

이 외에도 여러 특화된 형태가 있습니다:

* **CAN(캠퍼스 영역 네트워크)**: 대학 캠퍼스나 대규모 산업 단지와 같이 여러 건물을 아우르는 네트워크입니다.
* **MAN(메트로폴리탄 영역 네트워크)**: 하나의 도시나 대도시 지역을 연결하는 네트워크입니다.
* **PAN(개인 영역 네트워크)**: 개인 작업 공간 주변의 매우 짧은 범위(예: 블루투스로 연결된 장치)를 위한 네트워크입니다.

---

## 네트워크 구조 개념: 세그먼트, 네트워크, 서브넷 및 인터네트워크

* **세그먼트(Segment)**: 공유 미디어(예: 동일한 케이블 또는 허브)를 공유하는 장치 그룹으로, 충돌 도메인을 구성합니다.
* **네트워크(Network)**: 공통 주소 체계를 공유하고 라우터 없이 서로 직접 통신할 수 있는 장치들의 집합입니다.
* **서브넷(Subnetwork)**: 더 큰 네트워크의 논리적 부서로, 효율적인 관리를 위해 IP 주소 범위를 나눈 것입니다.
* **인터네트워크(Internetwork)**: 라우터와 같은 네트워킹 장비로 연결된 두 개 이상의 개별 네트워크를 말합니다. 인터넷이 궁극적인 인터네트워크입니다.

---

## 인터넷, 인트라넷 및 엑스트라넷

* **인터넷(Internet)**: 전 세계적으로 공개적으로 접근 가능한, 상호 연결된 컴퓨터 네트워크의 거대한 시스템입니다.
* **인트라넷(Intranet)**: 인터넷 기술(예: TCP/IP, 웹 브라우저)을 사용하지만 조직 내부에서만 비공개로 운영되는 네트워크입니다. 내부 정보 공유 및 협업에 사용됩니다.
* **엑스트라넷(Extranet)**: 인트라넷의 확장으로, 신뢰할 수 있는 외부 파트너(예: 공급업체, 주요 고객)에게 선택적으로 접근을 허용합니다. 보안 게이트웨이와 인증을 통해 제어됩니다.

## 네트워크 성능 이해

### 관점에 따른 네트워크 성능

네트워크 성능은 단순히 '빠름'을 넘어 다양한 요소로 평가됩니다. 사용자 경험, 애플리케이션 요구사항, 비용 효율성 등의 맥락에서 종합적으로 바라봐야 합니다. 예를 들어, 원격 회의 시스템은 낮은 지연 시간을, 대용량 파일 전송은 높은 처리량을 더 중요하게 요구합니다.

### 성능과 비성능 특성 간의 균형

네트워크 설계 및 운영에서는 순수한 성능 수치 외에 다음과 같은 특성과의 균형을 찾아야 합니다:

* **보안(Security)**: 암호화, 방화벽, 접근 제어는 성능에 일부 오버헤드를 초래할 수 있습니다.
* **신뢰성(Reliability) & 가용성(Availability)**: 중복 경로, 고품질 장비는 초기 비용을 증가시키지만 장기적 정상 운영 시간을 보장합니다.
* **확장성(Scalability)**: 미래 성장을 수용할 수 있는 능력은 초기 설계의 복잡성과 관련이 있습니다.
* **관리의 용이성(Manageability)**: 복잡한 최적화 설정보다는 단순하고 견고한 구성이 전체적인 운영 효율성을 높일 수 있습니다.
* **비용(Cost)**: 장비, 유지보수, 대역폭 비용은 항상 성능 목표와 타협점을 찾아야 합니다.

### 주요 성능 측정 지표

* **속도(Speed)**: 데이터가 네트워크를 통해 이동할 수 있는 이론적 최대 속도로, 일반적으로 비트/초(bps) 단위로 표시됩니다.
* **대역폭(Bandwidth)**: 특정 시간 동안 네트워크 연결을 통해 전달할 수 있는 최대 데이터 양의 이론적 용량입니다. 속도와 혼용되지만, '파이프의 너비'로 비유할 수 있습니다.
* **처리량(Throughput)**: 실제로 달성되는 데이터 전송률로, 대역폭보다 항상 낮습니다. 프로토콜 오버헤드, 네트워크 정체, 장비 성능 등에 영향을 받습니다.
* **지연 시간(Latency)**: 데이터 패킷이 출발지에서 목적지까지 이동하는 데 걸리는 시간(밀리초 단위). 왕복 지연 시간(RTT)이 중요 지표로 사용됩니다. 높은 지연은 실시간 응용 프로그램에 치명적입니다.

### 성능 측정 단위 이해

네트워크 속도는 일반적으로 **초당 비트 수(bps)** 로 측정됩니다(예: Kbps, Mbps, Gbps). 데이터 양은 **바이트(Byte)** 로 측정됩니다(예: KB, MB, GB). **1바이트 = 8비트**라는 점이 혼동의 요인이 될 수 있습니다. 따라서 100 Mbps(메가비트/초) 대역폭의 이론적 최대 파일 전송 속도는 약 12.5 MB/s(메가바이트/초)입니다.

---

## 이론적 처리량 vs 실제 처리량 및 성능 영향 요인

이론적 대역폭은 실제 처리량을 보장하지 않습니다. 실제 처리량을 저하시키는 주요 요인은 다음과 같습니다:

* **프로토콜 오버헤드**: TCP/IP 헤더, 에러 검출 등 관리용 데이터가 실제 데이터 외에 추가로 전송됩니다.
* **네트워크 정체(Congestion)**: 너무 많은 데이터가 동시에 전송되어 라우터 버퍼가 포화되고 패킷 손실 및 재전송이 발생합니다.
* **노이즈 및 간섭**: (특히 무선에서) 신호 품질을 저하시켜 재전송률을 높입니다.
* **장비 성능**: 라우터, 스위치, 네트워크 카드의 처리 능력 한계.
* **전파 지연(Propagation Delay)**: 신호가 물리적 매체(구리선, 광섬유)를 통해 이동하는 데 필수적으로 소요되는 시간.

---

## 통신 운영 방식: 단방향, 전이중, 반이중

* **단방향(Simplex)**: 한 방향으로만 통신이 가능한 방식(예: 전통적인 TV/라디오 방송).
* **반이중(Half-Duplex)**: 양방향 통신이 가능하지만, 동시에 진행될 수는 없는 방식. 한 번에 한 쪽만 전송할 수 있습니다(예: 무전기). 충돌(Collision)이 발생할 수 있습니다.
* **전이중(Full-Duplex)**: 동시에 양방향 통신이 가능한 방식(예: 현대적 전화 통화, 스위치에 연결된 이더넷). 별도의 송수신 경로가 필요하며 충돌이 발생하지 않습니다.

---

## 서비스 품질(QoS)

QoS는 제한된 네트워크 자원 하에서 다양한 애플리케이션, 사용자 또는 데이터 흐름에 우선순위를 부여하여 성능을 관리하는 메커니즘입니다. 중요한 트래픽(예: VoIP, 화상 회의)에 예측 가능한 대역폭, 지연, 지터(지연 변동), 패킷 손실을 보장하는 데 목적이 있습니다.

**주요 QoS 기술:**

* **트래픽 분류 및 표시**: 패킷의 중요도에 따라 태그를 부여합니다(예: DSCP, 802.1p).
* **대기열(Queuing)**: 우선순위가 높은 패킷을 먼저 처리하기 위한 다양한 스케줄링 알고리즘(예: 우선순위 큐, 가중 공정 큐).
* **트래픽 형성(Shaping) 및 제한(Policing)**: 트래픽 흐름의 속도를 제어하거나 제한합니다.
* **혼잡 회피**: 네트워크 정체가 발생하기 전에 사전에 제어합니다(예: RED - Random Early Detection).

## 네트워크 표준과 표준화 기구

네트워킹의 핵심은 서로 다른 시스템이 원활하게 통신할 수 있도록 하는 **표준**에 있습니다. 표준은 규칙과 지침의 집합으로, 장비와 소프트웨어가 상호 운용될 수 있는 공통의 기반을 제공합니다.

### 표준의 유형

* **독점 표준(Proprietary Standards)**: 특정 회사가 개발하고 통제하는 표준입니다. 초기에는 해당 회사의 제품 간 호환성을 제공하지만, 다른 벤더의 제품과는 호환이 되지 않을 수 있습니다. (예: 과거 Apple의 Lightning 커넥터)
* **개방형 표준(Open Standards)**: 공개적으로 사용 가능하며, 일반적으로 이해관계자들의 합의를 통해 비영리 표준화 기구에서 제정됩니다. 모든 벤더가 자유롭게 구현할 수 있어 혁신과 경쟁을 촉진합니다. (예: TCP/IP, HTTP, Ethernet)
* **사실상 표준(De Facto Standards)**: 공식 표준화 과정을 거치지 않았으나, 시장의 광범위한 채택으로 인해 사실상의 표준이 된 것입니다. (예: PC의 QWERTY 키보드 배열)

### 네트워킹 표준화 기구

표준은 국제적, 지역적, 산업별 다양한 기구들을 통해 제정됩니다.

#### 국제 표준화 기구

* **ISO (국제 표준화 기구)**: 가장 널리 알려진 국제 표준화 기구로, OSI 7계층 참조 모델이 대표적인 업적입니다.
* **ITU-T (국제 전기 통신 연합 전기 통신 표준화 부문)**: 전신, 전화, 데이터 통신 시스템을 위한 표준을 주로 담당합니다.
* **IEEE (국제 전기 전자 기술자 협회)**: 특히 **IEEE 802 위원회**가 LAN/MAN 표준(예: 이더넷(IEEE 802.3), 무선 LAN(IEEE 802.11))을 개발한 것으로 유명합니다.

#### 인터넷 표준화 기구

인터넷의 표준은 보다 개방적이고 협력적인 구조 하에 만들어집니다.

* **ISOC (인터넷 협회)**: 인터넷의 개방적 발전을 지원하는 최상위 조직입니다.
* **IAB (인터넷 아키텍처 위원회)**: 인터넷의 전반적인 아키텍처와 표준화 프로세스를 감독합니다.
* **IESG (인터넷 엔지니어링 스텝 그룹)**: IETF의 기술 활동 관리와 표준 승인을 담당합니다.
* **IETF (인터넷 엔지니어링 태스크 포스)**: **실질적인 인터넷 표준(RFC 문서)을 개발하는 핵심 조직**입니다. 실제 작업은 HTTP, 라우팅, 보안 등 주제별 WG(Working Group)에서 이루어집니다.
* **IRTF (인터넷 연구 태스크 포스)**: 장기적인 인터넷 관련 연구 주제를 다룹니다.

#### 인터넷 등록 기관

인터넷의 고유 자원(주소, 번호, 도메인명) 관리를 담당합니다.

* **IANA (인터넷 할당 번호 관리기관)**: IP 주소, 프로토콜 번호, 도메인 이름 시스템 루트 존 등의 최상위 관리자 역할을 합니다. 현재 ICANN 산하 기관으로 운영됩니다.
* **ICANN (국제 인터넷 도메인 이름 및 주소 관리 기관)**: IANA 기능을 총괄하며, 전 세계 도메인 이름과 IP 주소의 정책 조정을 담당합니다.
* **RIR (지역 인터넷 등록기관)**: ICANN/IANA로부터 할당받은 IP 주소를 각 지역의 ISP나 기관에 다시 할당하는 5개의 기구입니다.

### 인터넷 표준과 RFC 프로세스

인터넷 표준은 **RFC(Request For Comments)** 라는 일련의 문서로 공개됩니다. RFC는 단순한 표준뿐만 아니라 실험적 아이디어, 최선의 실행 방법(BCP), 정보 제공용 문서도 포함됩니다. 표준화 과정은 매우 엄격하여, 제안된 기술은 광범위한 검토와 실제 구현, 테스트를 거쳐야만 '인터넷 표준'으로 승격될 수 있습니다.

## 컴퓨팅의 데이터 표현과 수학적 기초

네트워크를 통해 전송되는 모든 정보는 컴퓨터가 이해할 수 있는 형태, 즉 **숫자**로 변환되어야 합니다.

### 정보의 기본 단위

* **비트(Bit)**: 정보의 최소 단위로, 0 또는 1의 값을 가집니다.
* **바이트(Byte)**: 일반적으로 **8비트**를 묶은 단위입니다. 문자 하나를 표현하는 기본 단위로 사용됩니다.
* **니블(Nibble)**: 4비트, 즉 16진수 한 자리에 해당합니다.
* **옥텟(Octet)**: 8비트를 명확히 지칭할 때 사용하는 용어입니다. 네트워킹 분야에서는 '바이트' 대신 '옥텟'을 주로 사용합니다.

### 숫자 체계

컴퓨터는 이진법(Binary)을 사용하지만, 인간이 이해하고 표현하기 쉽도록 다른 진법도 활용합니다.

1. **십진수(Decimal)**: 인간이 일상적으로 사용하는 0-9의 10개 기수 체계.
2. **이진수(Binary)**: 컴퓨터의 근본 언어. 0과 1만 사용하는 2개 기수 체계.
3. **팔진수(Octal)**: 과거 시스템에서 사용되던 8개 기수(0-7) 체계.
4. **십육진수(Hexadecimal)**: 2진수를 간결하게 표현하는 데 매우 유용합니다. 0-9와 A-F의 16개 기수로 표현되며, 2진수 4비트(1 니블)가 16진수 한 자리로 매핑됩니다.

### 부울 논리와 비트 마스킹

컴퓨터 내부의 논리 연산과 비트 단위 제어는 **부울 논리(Boolean Logic)** 에 기반합니다.

* **기본 논리 함수**: AND, OR, XOR(배타적 OR), NOT
* **비트 마스킹(Bit Masking)**: 이러한 논리 연산을 사용하여 데이터 내 특정 비트들을 조작하는 기술입니다. '마스크'라는 비트 패턴을 적용하여 원하는 비트만을 선택적으로 처리합니다.

## 개방형 시스템 상호 연결 (OSI) 참조 모델

### OSI 참조 모델의 역사

개방형 시스템 상호 연결(OSI) 참조 모델은 1970년대 말부터 1980년대 초에 걸쳐 국제표준화기구(ISO)에 의해 개발된 개념적 프레임워크입니다. 이 모델은 각기 다른 제조사들 간의 "벤더 종속성"을 해결하고 네트워크 시스템의 설계, 구현, 운영을 위한 공통의 표준을 제공하기 위해 만들어졌습니다. 1984년에 최종 발표된 OSI 모델은 실제 널리 사용되는 프로토콜 세트보다는 네트워크 기능을 이해하고 설계하기 위한 **참조 모델**로서 네트워킹 교육과 문제 해결의 핵심 도구로 남아 있습니다.

### 일반적인 참조 모델의 중요성

참조 모델은 복잡한 시스템을 이해하기 위한 지도와 같습니다.

* **복잡성 관리**: 네트워크 통신을 관리 가능한 계층으로 분해합니다.
* **상호 운용성**: 서로 다른 벤더 제품 간의 호환성을 촉진합니다.
* **표준화**: 개발과 발전을 위한 공통 언어를 제공합니다.
* **문제 해결**: 문제를 특정 계층으로 격리하여 효율적으로 진단합니다.

### OSI 참조 모델을 이해해야 하는 이유

네트워크 전문가가 이 모델을 이해하면 정확한 문제 진단, 효율적인 커뮤니케이션, 체계적인 학습, 그리고 견고한 설계 및 구현이 가능해집니다.

### OSI 참조 모델 활용 방법

1. **하향식 분석**: 응용 계층부터 시작하여 아래로 내려가며 확인합니다.
2. **상향식 분석**: 물리적 연결부터 시작하여 위로 올라가며 문제를 찾습니다.
3. **계층별 도구 사용**: 각 계층에 적합한 진단 도구(ping, traceroute 등)를 활용합니다.
4. **설계 템플릿**: 새로운 프로토콜 설계 시 구조적 틀을 제공합니다.

### OSI 참조 모델과 네트워크 아키텍처 비교

OSI 모델은 이론적 참조 모델인 반면, 실제 인터넷의 기반은 TCP/IP 모델입니다.

```
OSI 7계층 모델과 TCP/IP 4계층 모델 비교
+-------------------------------+-------------------+
| OSI 참조 모델 (이론적)       | TCP/IP 모델 (실제)|
+-------------------------------+-------------------+
| 7. 응용 계층 (Application)    |                   |
| 6. 표현 계층 (Presentation)   | 4. 응용 계층      |
| 5. 세션 계층 (Session)        |                   |
+-------------------------------+-------------------+
| 4. 전송 계층 (Transport)      | 3. 전송 계층      |
+-------------------------------+-------------------+
| 3. 네트워크 계층 (Network)    | 2. 인터넷 계층    |
+-------------------------------+-------------------+
| 2. 데이터 링크 계층 (Data Link)| 1. 네트워크 접근  |
| 1. 물리 계층 (Physical)       | 계층              |
+-------------------------------+-------------------+

```

### OSI 참조 모델의 주요 개념

#### OSI 계층 그룹화

* **상위 계층 (5-7)**: 응용 프로그램 관련 처리 (소프트웨어적 구현).
* **전송 계층 (4)**: 종단 간 연결 및 신뢰성 담당.
* **하위 계층 (1-3)**: 전송 및 경로 지정 (하드웨어적 구현).

#### 인터페이스와 프로토콜

* **인터페이스 (수직 통신)**: 동일 시스템 내 인접 계층 간 통신으로 서비스 액세스 점(SAP)을 통해 이루어집니다.
* **프로토콜 (수평 통신)**: 서로 다른 시스템의 동일 계층 간 논리적 통신입니다.

#### 데이터 캡슐화

데이터가 상위에서 하위로 내려갈 때 제어 정보가 추가되는 과정을 **캡슐화**라고 합니다. 상위에서 받은 **SDU(Service Data Unit)**에 헤더를 붙여 해당 계층의 **PDU(Protocol Data Unit)**를 생성합니다. (예: 4계층-세그먼트, 3계층-패킷, 2계층-프레임, 1계층-비트)

#### 간접 장치 연결 및 메시지 라우팅

중간 장치는 특정 계층까지 처리합니다:

* **허브**: 1계층
* **스위치**: 2계층
* **라우터**: 3계층

## 결론

네트워킹 기본 원리는 상호 연결된 디지털 세계의 토대를 형성합니다. 효율적인 네트워크를 구축하기 위해서는 표준과 데이터 표현의 본질을 이해하고, OSI 모델과 같은 체계적인 프레임워크를 통해 복잡한 통신 과정을 관리할 수 있어야 합니다. 이러한 기본 구성 요소들을 이해하는 것은 현대 컴퓨팅 환경을 탐색하고, 문제를 해결하며, 견고한 디지털 인프라를 구축하는 데 필수적인 첫걸음입니다.
