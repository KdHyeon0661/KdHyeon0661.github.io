# CSS 단위 정리 (px, em, rem, %, vw, vh 등)

## 0. 큰 그림 요약

- **절대 계열**: `px`(논리 픽셀), `Q`(쿼터-밀리미터, 드물게 사용)
- **상대 계열(폰트 기준)**: `em`(현재 요소), `rem`(루트), `ex`(x-height), `ch`(“0” 폭), `ic`(CJK 글자폭), `cap`(대문자 높이), `lh`/`rlh`(행간)
- **상대 계열(컨테이너/뷰포트/로직 기준)**: `%`(부모), `vw`/`vh`/`vmin`/`vmax`, `dvh`/`svh`/`lvh`, `vi`/`vb`(논리 뷰포트), `cqw`/`cqh`(컨테이너 쿼리)
- **수학 함수**: `calc()`/`min()`/`max()`/`clamp()`
- **권장 흐름**: 크기/여백은 **rem**을 기준으로 설계하고,
  **em**은 컴포넌트 내부 비율, **%**는 부모 비례 레이아웃, **vw/vh(및 dvh)** 는 히어로/풀스크린,
  **gap**과 함께 쓰면 간격 제어가 단순해집니다.

---

## 1. px — 논리 픽셀(absolute-ish)

```css
.box {
  width: 200px;
  font-size: 16px;
  border-width: 1px;
}
```

특징
- 예측 가능한 수치. 아이콘/테두리/1px 라인 등에 적합.
- **브라우저 확대(zoom)** 시 px도 스케일됨(“절대”가 아님).
- **DPR(Device Pixel Ratio)** 2x/3x 환경에서도 **CSS 픽셀**은 논리 단위.

사용 팁
- 텍스트 크기/여백은 **rem**으로, 픽셀 정밀성 필요한 곳만 `px`.

---

## 2. % — 부모 기준 비례(상대)

```css
.container { width: 100%; }
.child     { width: 50%; } /* 부모의 절반 */
```

특징
- `width`·`left` 등은 **컨테이너의 해당 축 크기** 기준.
- `height: %`는 부모 높이가 **명시**되지 않으면 기대와 다르게 동작.
- `padding: %`는 **컨테이너 폭** 기준(수직 패딩도 폭 기준).

---

## 3. em — 현재(또는 상속된) 폰트 크기 기반

```css
html { font-size: 16px; }
.card {
  font-size: 1.125rem; /* 18px */
  padding: 1em;        /* 18px (card의 font-size 기준) */
}
```

특징
- **누적**됨(중첩 시 커짐): 상위에서 `font-size`가 바뀌면 함께 변동.
- 컴포넌트 내부 **상대 비율** 표현에 좋음(버튼 padding 등).

주의
- 깊게 중첩되면 예상이 어려워질 수 있음 → 기본 스케일은 `rem` 권장.

---

## 4. rem — 루트(html) 기준, 예측 가능한 단위

```css
html { font-size: 16px; } /* 사용자 기본값을 존중하면 접근성에 유리 */
.title { font-size: 2rem; } /* 32px */
.wrapper { padding: 1rem; } /* 16px */
```

특징
- 어디서든 **html의 font-size** 기준 → **예측 가능/유지보수 우수**.
- 폰트/여백/레이아웃 전반에 **기본 단위**로 추천.

변환 공식
- $$ \text{px} = \text{rem} \times \text{rootFontSize(px)} $$
- 예: `1.25rem × 16 = 20px`

---

## 5. vw / vh — 뷰포트 기준

```css
.hero {
  width: 100vw;       /* 뷰포트 폭 100% */
  min-height: 100vh;  /* 뷰포트 높이 100% */
}
```

특징
- 창 크기에 **즉시 반응**.
- 모바일 주소창/툴바 변화로 **vh가 출렁**일 수 있음 → 아래 확장 단위 참고.

### 5.1 확장: dvh / svh / lvh
- `dvh`: **Dynamic** VH, 주소창 등 UI 변화를 **반영**(실측 높이)
- `svh`: **Small** VH, UI가 **최대**로 차지하는 상황의 **최소 높이**
- `lvh`: **Large** VH, UI가 **최소**일 때 **최대 높이**

```css
.hero { min-height: 100dvh; } /* 모바일에서 안정적 섹션 높이 */
```

### 5.2 논리 뷰포트: vi / vb
- `vi` = inline 축 뷰포트 %, `vb` = block 축 뷰포트 %
  (RTL/세로쓰기에서도 올바른 축 대응)

```css
.section { padding-inline: 5vi; padding-block: 5vb; }
```

---

## 6. vmin / vmax — 작은/큰 축 기준

```css
.square {
  width: 50vmin;  /* 더 짧은 축의 50% */
  height: 50vmin; /* 정사각형 만들기 */
}
```

- 가로/세로 비율이 바뀌는 환경에서 안정적인 비례 상자 작성에 유리.

---

## 7. 타이포·서식 단위 — ch / ex / ic / cap / lh / rlh

```css
.input {
  width: 30ch;  /* “0” 문자 30개 너비 기준 */
}
.kicker {
  font-size: 1rem;
  line-height: 1.5;      /* 단위 없는 line-height는 폰트 비례 */
  margin-block: 1lh;     /* 실제 계산된 line-height 기준 */
}
.cjk {
  width: 12ic;           /* CJK 전각 글자폭 기반 */
}
.title { margin-top: 1cap; } /* 대문자 높이 기준 */
```

- `ch`: 모노/프로포셔널 폰트에 따라 편차 있지만 **폼 필드/코드 폭**에 유용.
- `ex`: 소문자 x 높이(폰트 의존성 커서 실전 사용 적음).
- `ic`: CJK 글자폭 기준, **한글/일본어/중국어** 레이아웃에 실전적.
- `cap`: 대문자 cap height 기준, **타이포 리듬**에 도움.
- `lh`/`rlh`:
  - `lh`: **해당 요소의** 계산된 line-height.
  - `rlh`: **루트 요소의** line-height.

---

## 8. 컨테이너 쿼리 단위 — cqw / cqh / cqi / cqb / cqmin / cqmax

컨테이너(조상 요소)의 크기에 반응하는 **진짜 컴포넌트 단위**.

```css
.card {
  container-type: inline-size; /* 이름도 가능: container-name: card; */
}
.card__title {
  font-size: clamp(1rem, 2cqw, 1.75rem);
}
```

- `cqw`/`cqh`: 컨테이너의 width/height 1%.
- 컨테이너 기반 **모듈식 반응형** 구현에 최적(페이지 전체 뷰포트 대신 “카드 폭”에 반응).

---

## 9. 계산 함수 — calc / min / max / clamp

```css
.box { width: calc(100% - 2rem); }

.title {
  font-size: clamp(1rem, 2.5vw, 2rem); /* 최소–우선–최대 */
}

.sidebar {
  width: min(32rem, 80vw); /* 좁을 땐 vw, 넓으면 32rem로 고정 */
}
```

- **clamp**로 유동형 타이포/여백을 간단히 구현.
- **min/max**로 상한/하한 장치.

---

## 10. 접근성과 줌(Zoom) — 루트 글자 크기를 존중

- 사용자가 브라우저 설정으로 **기본 글자 크기(보통 16px)** 를 바꿀 수 있습니다.
- 텍스트/여백의 기반 단위를 **rem**으로 두면 줌/접근성에 강함.
- 터치 대상은 최소 **44px**(또는 `2.75rem` 가량) 히트영역 권장.

---

## 11. 모바일 뷰포트 주의 — 주소창/툴바, 키보드 열림

- 전통 `100vh`는 모바일에서 **주소창 접힘/펼침**에 따라 “튀는” 현상.
- `100dvh`로 개선하거나, 섹션 시작부에 **`min-height: 100dvh`** 사용.
- 키보드가 열리면 안전영역을 고려해 **스크롤 패딩**과 함께 조정:
```css
:root { scroll-padding-top: 72px; }
.section-title { scroll-margin-top: 72px; }
```

## 12. 실전 패턴 레시피

### 12.1 유동형 타이포그래피
```css
:root { font-size: 16px; } /* 사용자 기본값 존중 */
h1 { font-size: clamp(1.75rem, 4vw, 2.5rem); }
h2 { font-size: clamp(1.5rem, 3vw, 2rem); }
p  { font-size: 1rem; line-height: 1.7; }
```
- 좁은 화면에서 작아지고, 넓은 화면에서 커지되 **과대 확대 방지**.

### 12.2 카드 그리드(콘텐츠 폭 기반)
```css
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(18rem, 1fr));
  gap: 1rem;
  padding: 1rem;
}
.card { padding: 1rem; border: 1px solid #e5e7eb; border-radius: .75rem; }
```
- 열 수는 **그리드 자동**에 맡기고, 카드는 **rem 기반** 내부 간격.

### 12.3 히어로 섹션(모바일 안정 높이)
```css
.hero {
  min-height: 100dvh; /* 주소창 출렁 방지 */
  display: grid; place-items: center;
  padding-inline: clamp(1rem, 6vi, 3rem);
}
.hero__title { font-size: clamp(2rem, 6vw, 3rem); }
```

### 12.4 폼 필드 폭과 줄바꿈 제한
```css
.input { width: 28ch; }         /* 평균 28자 입력 폭 */
.code  { max-inline-size: 72ch; }/* 본문 줄길이 제어 */
```

### 12.5 컴포넌트 내부 비율(em) + 외부 스케일(rem)
```css
:root { --space: 1rem; }
.button {
  font-size: 1rem;
  padding: .625em 1em;     /* 내부 비율은 em */
  border-radius: .5rem;    /* 외곽 스케일은 rem */
  margin: var(--space);    /* 전역 토큰 */
}
```

### 12.6 컨테이너 반응형(카드 제목만 가변)
```css
.card { container-type: inline-size; }
.card__title { font-size: clamp(1rem, 5cqw, 1.5rem); }
```

## 13. 단위 선택 체크리스트

| 목적/상황 | 권장 단위 |
|---|---|
| 기본 텍스트/여백/레이아웃 | **rem** |
| 컴포넌트 내부 상대 비율(버튼 padding 등) | **em** |
| 부모 비례 레이아웃 | **%** |
| 풀 화면/히어로 | **dvh**(또는 vh+보정), **vw** |
| 화면 작은/큰 축 비례 도형 | **vmin/vmax** |
| CJK/코드 입력 폭 | **ic/ch** |
| 타이포 리듬 기반 간격 | **lh/rlh/cap** |
| 컨테이너 폭 반응 | **cqw/cqh** |
| 픽셀 정밀 요소(1px 라인, 아이콘) | **px**(선별 사용) |

---

## 14. 수학과 단위 — 공식 모음

### 14.1 px ↔ rem
$$
\text{px} = \text{rem} \times \text{rootFontSize(px)}, \quad
\text{rem} = \frac{\text{px}}{\text{rootFontSize(px)}}
$$

### 14.2 clamp로 유동값
$$
\text{value} = \min(\max(\text{preferred}, \text{min}), \text{max})
$$

예:
```css
padding-inline: clamp(1rem, 4vw, 3rem);
```

## 15. 흔한 함정과 해결책

| 문제 | 원인 | 해결 |
|---|---|---|
| `height: 100%` 안 먹음 | 부모 높이 미지정 | 상위에 명시 높이/`min-height` 지정 또는 레이아웃 재구성 |
| 모바일 `100vh` 출렁 | 브라우저 UI 변화 | `100dvh` 사용 또는 `min-height` + safe-area 보정 |
| em 폭주 | 상위 font-size 누적 | 외곽 레이아웃은 rem, 내부 비율만 em |
| 폼 폭 불안 | 글꼴 차이 | `ch/ic`로 자수 기반 폭 지정 |
| 지나치게 고정된 px | 접근성 저하 | 루트 글꼴을 존중하고 rem 중심으로 전환 |

## 16. 확장 예제 — 반응형 카드(개선 버전)

```css
:root {
  --space: 1rem;
  --radius: .75rem;
}

.page {
  width: min(92vw, 72rem);
  margin-inline: auto;
  padding: clamp(1rem, 4vw, 2rem);
}

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(18rem, 1fr));
  gap: clamp(.75rem, 2vw, 1.25rem);
}

.card {
  container-type: inline-size;
  padding: clamp(1rem, 2cqw, 1.5rem);
  border: 1px solid #e5e7eb;
  border-radius: var(--radius);
  background: #fff;
}

.card__title { font-size: clamp(1rem, 4cqw, 1.5rem); }
.card__meta  { font-size: .875rem; margin-block: .5lh; }
.card__body  { font-size: 1rem;   line-height: 1.7; }
```

해설
- 전체 폭은 `min(92vw, 72rem)`로 **상한/하한** 제어.
- 카드 간 `gap`은 `clamp`로 **유동 여백**.
- 카드 내부 패딩/제목 크기는 **컨테이너 단위(cqw)**로 **진짜 컴포넌트 반응형**.

---

## 17. 결론

- **rem 중심**으로 기본 스케일을 잡고, **em**으로 내부 비율을 다듬으세요.
- 뷰포트 기반 레이아웃은 **dvh/vh/vw + clamp**로 “튀지 않는” 유동형을 만드세요.
- 폼/코드 영역엔 **ch/ic**, 타이포 리듬에는 **lh/rlh/cap**, 컴포넌트 반응형엔 **cqw**가 강력합니다.
- 무엇보다 **접근성(루트 폰트)**을 존중하면, 반응성과 유지보수성이 함께 좋아집니다.
