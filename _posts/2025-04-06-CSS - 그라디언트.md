# CSS 그라디언트 사용법

## 빠른 개요

- **`linear-gradient()`**: 한 방향(각도/키워드)에 따라 **직선상** 색이 변함
- **`radial-gradient()`**: 지정한 중심에서 **원형/타원형**으로 색이 퍼짐
- **`repeating-…`**: 지정 구간을 **주기적으로 반복**하여 패턴 생성
- **레이어**: `background:`에 **콤마로 여러 겹**을 쌓아 복합 효과
- **투명/오버레이**: `rgba()`/`hsla()`/`transparent`로 이미지 상단 덮기
- **컬러스페이스**: 최신 스펙(CSS Color 4/5)의 `lab()`/`lch()`/`oklab()` 등으로 **자연스런 보간** 가능(지원 브라우저 한정)

---

## `linear-gradient()` — 선형 그라디언트 완전 정리

### 구문

```css
background: linear-gradient([<angle> | to <side-or-corner>], <color-stop> [, <color-stop>]*);
```
- `<angle>`: `0deg`는 **위→아래**, `90deg`는 **왼→오**.
  키워드 `to right|left|top|bottom|top right` 등도 가능.
- `<color-stop>`: `색상 [정지점%|px]` 형태. 중간 위치를 지정해 **전환 지점**을 제어.

### 방향(각도 vs 키워드)

```css
/* 키워드 */
background: linear-gradient(to right, red, blue);   /* 좌→우 */

/* 각도(시계방향) */
background: linear-gradient(45deg, orange, purple); /* 좌하→우상 */
```

> 좌표 직관 요약
> `0deg`: 위→아래, `90deg`: 좌→우, `180deg`: 아래→위, `270deg`: 우→좌.

### 색상 정지점(Stops)과 하드 에지

```css
/* 정확한 중간 전환 */
background: linear-gradient(to right, red 0%, yellow 50%, green 100%);

/* 하드 컷(날카로운 경계) — 같은 위치에 두 스톱 */
background: linear-gradient(90deg, #111 0 40%, #111 40%, #eee 40% 60%, #eee 60%, #111 60% 100%);
```
- 같은 위치에 두 스톱을 겹치면 **경계가 날카로워**(밴딩 없이) 패턴처럼 보임.

### — 보간 흐름 가이드

일부 엔진은 **힌트**(stop 사이의 보간 비율 암시)를 해석합니다.
현대 브라우저에선 **명시적 스톱 위치**로 충분하며, 힌트는 실무에서 드뭅니다.

### 퍼센트/픽셀 단위 혼용

```css
background: linear-gradient(90deg, red 0 120px, blue 120px 100%);
```
- 고정 픽셀 구간 + 퍼센트 구간을 혼용해 **정보 밀도**가 다른 영역을 만들 수 있음.

## `radial-gradient()` — 원형/타원 그라디언트 완전 정리

### 구문

```css
background: radial-gradient([<shape> || <size>] [at <position>], <color-stop> [, <color-stop>]*);
```
- `<shape>`: `circle`(원, 기본) | `ellipse`(타원)
- `<size>`: `closest-side | farthest-side | closest-corner | farthest-corner | <length>% …`
- `at <position>`: 중심점(기본 `center`). 예: `at 30% 70%`, `at top left`

### 모양·사이즈 키워드

```css
/* 원형, 컨테이너를 가장 멀리 감싸는 코너까지 */
background: radial-gradient(circle farthest-corner at center, #ff6a00, #ee0979);

/* 타원형, 가까운 면까지만 */
background: radial-gradient(ellipse closest-side at 30% 70%, #00c6ff, #0072ff);
```

> 사이즈 키워드 요약
> - **closest-side**: 중심에서 가까운 **한쪽 면**까지
> - **farthest-side**: 중심에서 먼 **한쪽 면**까지
> - **closest-corner**: 가장 가까운 **코너**까지
> - **farthest-corner**: 가장 먼 **코너**까지(기본적으로 꽉 채우는 느낌)

### 위치

```css
background: radial-gradient(circle at top left, red, blue);
background: radial-gradient(circle at 30% 70%, #f00, #00f);
```
- `at` 뒤에 키워드/퍼센트/픽셀로 중심 조정.

## 투명도/오버레이 — `rgba()`·`hsla()`·`transparent`

```css
/* 아래에서 위로 어둡게 덮는 오버레이 */
background: linear-gradient(to top, rgba(0,0,0,.6), rgba(0,0,0,0));
```
- 이미지/비디오 위 텍스트 가독성 확보에 유용.
- `transparent`는 **투명**이지만, 보간 과정에서 중간 색이 **검은 기운**을 띨 수 있음 →
  **같은 색의 투명 버전**으로 쓰면 더 자연스러움:
```css
/* 더 자연스러운 투명 그라디언트(파란색 → 투명 파란색) */
background: linear-gradient(180deg, rgba(59,130,246,1), rgba(59,130,246,0));
```

## 반복 그라디언트 — 줄무늬/격자/체커보드

### `repeating-linear-gradient`

```css
/* 45도 스트라이프(10px 검정 + 10px 흰) 반복 */
background: repeating-linear-gradient(45deg, #000 0 10px, #fff 10px 20px);
```

### `repeating-radial-gradient`

```css
/* 동심원 패턴 */
background: repeating-radial-gradient(circle, #111 0 6px, #eee 6px 12px);
```

### 격자/체커보드(레이어 2겹)

```css
background:
  repeating-linear-gradient(0deg,  #0000 0 19px, #0002 19px 20px),
  repeating-linear-gradient(90deg, #0000 0 19px, #0002 19px 20px);
```
- 첫 레이어: 가로 줄, 두 번째: 세로 줄 → **격자**
- 투명도를 낮춰 **은은한 질감** 생성

## 복수 그라디언트 레이어 — 이미지와 합성

```css
.hero {
  background:
    linear-gradient(to bottom, rgba(0,0,0,.6), rgba(0,0,0,.6)), /* 오버레이 */
    url("bg.jpg") center/cover no-repeat;                       /* 이미지 */
  color: #fff;
}
```
- `background`는 **앞에 쓴 것이 위**에 올라감.
- **여러 그라디언트만** 겹쳐도 깊이감 있는 배경 연출 가능.

## 컬러스페이스와 자연스런 보간(지원 브라우저 한정)

전통적으로 CSS는 **sRGB**에서 보간합니다. **밴딩**이나 **회색 기운**이 돌 수 있죠.
최신 스펙은 다음 함수로 더 자연스런 보간을 지원합니다(브라우저 지원 확인 필요).

```css
/* LCH 공간: 색상 고정하고 밝기/채도로 자연스러운 그라디언트 */
background: linear-gradient(90deg, lch(70% 50 30) 0%, lch(70% 50 330) 100%);

/* OKLab/OKLCH — 지각 균등 기반 */
background: linear-gradient(90deg, oklch(0.75 0.08 30) 0%, oklch(0.75 0.08 330) 100%);
```

> 지원이 애매하면 **보색·중간색을 수동 삽입**해 밴딩을 줄이거나,
> **디더링**(아주 약한 노이즈 텍스처)을 **상단 레이어**로 더해 시각적으로 부드럽게 만들 수 있습니다.

---

## 수학적 직관 — 색상 보간과 위치 매핑

그라디언트 선상의 **정규화 좌표** \(t \in [0,1]\)에 대해 두 색 \(C_0, C_1\)의 선형 보간은
$$
C(t) = (1-t)\,C_0 + t\,C_1
$$
이며, **픽셀 좌표 \(x\)**(선형 방향 길이 \(L\))와의 관계는
$$
t = \frac{x}{L}
$$
입니다.
**여러 스톱**이 있을 때는 해당 구간의 \(t\)를 **부분 구간**으로 다시 정규화하여 보간합니다.

## 실전 레시피 모음

### 카드/섹션 배경(그라디언트 페이드)

```css
.card {
  background:
    linear-gradient(180deg, rgba(0,0,0,.05), rgba(0,0,0,0) 40%),
    linear-gradient(0deg,  rgba(255,255,255,.6), rgba(255,255,255,0) 60%),
    #fff;
  border: 1px solid #e5e7eb; border-radius: .75rem;
}
```

### 텍스트 그라디언트

```html
<h1 class="gtext">Gradient Text</h1>
```
```css
.gtext{
  background: linear-gradient(90deg, #ff8a00, #e52e71);
  -webkit-background-clip: text;  /* Safari/구버전 대응 */
  background-clip: text;
  color: transparent;
}
```

### 버튼 호버 스와프

```css
.button {
  background: linear-gradient(90deg, #4facfe 0%, #00f2fe 100%);
  color: #fff; transition: background .3s ease;
}
.button:hover {
  background: linear-gradient(90deg, #43e97b 0%, #38f9d7 100%);
}
```

### 이미지 위 어두운 오버레이(가독성 향상)

```css
.hero {
  background:
    linear-gradient(to bottom, rgba(0,0,0,.6), rgba(0,0,0,.6)),
    url('bg.jpg') center/cover no-repeat;
  min-height: 100vh; color:#fff;
  display:flex; align-items:center; justify-content:center;
}
```

### 카드

```css
.glass {
  background:
    linear-gradient( to bottom, rgba(255,255,255,.6), rgba(255,255,255,.2) ),
    radial-gradient( circle at 20% 0%, rgba(255,255,255,.35), rgba(255,255,255,0) 40% );
  backdrop-filter: blur(10px);
  border: 1px solid rgba(255,255,255,.35);
  border-radius: 1rem;
}
```

### 스켈레톤 로딩(빛 번짐)

```css
.skeleton {
  position: relative; overflow: hidden; background: #e5e7eb;
}
.skeleton::after {
  content:""; position:absolute; inset:0;
  background: linear-gradient(90deg, #0000, #ffffff40, #0000);
  transform: translateX(-100%); animation: shine 1.2s infinite;
}
@keyframes shine {
  to { transform: translateX(100%); }
}
```

### 픽셀 그리드/체커보드

```css
.checker {
  background:
    linear-gradient(45deg, #0001 25%, #0000 25% 75%, #0001 75%),
    linear-gradient(45deg, #0001 25%, #0000 25% 75%, #0001 75%);
  background-position: 0 0, 8px 8px;
  background-size: 16px 16px;
}
```

### 테두리를 그라디언트로

```css
.box {
  padding: 1px; border-radius: .75rem;
  background:
    linear-gradient( to right, #8ec5fc, #e0c3fc );    /* 테두리 */
}
.box > .inner {
  background: #fff; border-radius: inherit; padding: 1rem;
}
```

### 마스크로 도형 페이드(그라디언트 마스크)

```css
.fade-mask {
  -webkit-mask-image: linear-gradient(90deg, #000, #000, #0000 90%);
  mask-image: linear-gradient(90deg, #000, #000, #0000 90%);
}
```
- 오른쪽으로 갈수록 **투명**해져 자연스러운 사라짐 효과.

## 성능·밴딩·접근성

- **성능**: 그라디언트는 보통 **벡터 렌더링**이라 가볍지만,
  **대형 고정 배경 + 스크롤**에서 페인트가 잦으면 비용이 발생. `will-change` 남발은 지양.
- **밴딩**: 두 색 간 보간에서 **줄무늬 같은 밴딩**이 보일 수 있음 →
  - 중간 스톱 추가
  - 상단에 **아주 약한 노이즈 텍스처**(data-URI) 레이어
  - LCH/OKLCH 등 컬러스페이스 사용(지원 시)
- **접근성(대비)**: 텍스트 위 그라디언트는 구간마다 대비가 달라짐.
  - 텍스트 그라디언트를 쓸 때는 **두 색 모두에서 충분 대비**가 나는 팔레트를 고르거나
  - **텍스트 외곽 그림자**/미세한 어두운 오버레이를 더해 읽기성을 확보.

## 변수·토큰으로 재사용

```css
:root {
  --grad-primary: linear-gradient(90deg, #4facfe, #00f2fe);
  --grad-accent:  linear-gradient(90deg, #ff8a00, #e52e71);
}

.btn-primary { background: var(--grad-primary); color:#fff; }
.badge-accent { background: var(--grad-accent); color:#fff; }
```
- **디자인 토큰**으로 관리하면 테마 스위치/일괄 변경이 쉬움.

## 브라우저 호환 메모

- `linear-gradient`/`radial-gradient`/반복형은 **현대 브라우저 대부분** 지원 안정.
- `background-clip: text`는 **WebKit 접두사** 필요할 수 있음.
- `lab()`/`lch()`/`oklab()`/`oklch()` 등은 **최근 브라우저**에 한함 → 폴백 색상/그라디언트 병기 권장.

## 디버깅·체크리스트

- **의도한 레이어 순서?** `background`는 **앞이 위**. 순서를 뒤집지 않았는지.
- **밴딩 발생?** 중간 스톱 추가, 노이즈 레이어, 컬러스페이스 교체(가능 시).
- **가독성 문제?** 텍스트 대비 확인, 오버레이 진하게/연하게 조정, 텍스트 그림자.
- **성능 이슈?** 스크롤 시 과도한 리페인트가 없는지 DevTools Performance로 점검.
- **반응형?** 각도/스톱을 `%`로 설계해 화면비 바뀌어도 의도 유지.

## 요약 표

| 항목 | 핵심 포인트 |
|---|---|
| `linear-gradient()` | 각도/키워드로 방향, 스톱 위치로 전환 지점 제어 |
| `radial-gradient()` | `circle/ellipse` + 사이즈 키워드(closest/farthest) + `at` 위치 |
| 하드 에지 | 동일 위치에 두 스톱으로 날카로운 경계 |
