# 1. 배경(Background)

## 1.1 배경색과 배경 이미지 기본

```html
<div class="box">이 영역은 배경색과 배경 이미지를 가집니다.</div>
```

```css
.box {
  background-color: #f5f5f5;            /* 배경색 */
  background-image: url('/images/sky.jpg');
  background-repeat: no-repeat;          /* repeat | no-repeat | repeat-x | repeat-y */
  background-position: center;           /* left top | 50% 50% | center center */
  background-size: cover;                /* cover | contain | auto */
  color: #111;
  padding: 20px;
  height: 300px;
}
```

### 자주 쓰는 조합 요약

| 속성 | 값 | 설명 |
|---|---|---|
| `background-size` | `cover` | 요소를 가득 채우되 비율 유지(잘릴 수 있음) |
|  | `contain` | 요소에 모두 보이게 비율 유지(여백 생김) |
| `background-position` | `center` | 중심 정렬 |
| `background-attachment` | `scroll` | 스크롤과 함께 이동(기본) |
|  | `fixed` | 뷰포트에 고정(모바일 성능/호환 주의) |

## 1.2 배경 shorthand와 다중 배경

```css
.hero {
  /* shorthand: color image position/size repeat attachment */
  background:
    linear-gradient(180deg, rgb(0 0 0 / .4), rgb(0 0 0 / .4)),
    url('/images/mountains.avif') center / cover no-repeat fixed,
    #0f172a;
  color: #fff;
  min-height: 40vh;
  padding: 4rem 2rem;
}
```

- 첫 번째 레이어: 반투명 오버레이(그라디언트)
- 두 번째 레이어: 사진(고정 배경)
- 마지막: 폴백 배경색

> 모바일 Safari 등에서 `background-attachment: fixed`가 성능 이슈를 유발할 수 있습니다. 필요 시 섹션 내부의 의사 고정(parallax) 기법을 고려하세요.

## 1.3 배경 오버레이(가독성 향상)

```css
.banner {
  position: relative;
  background: url('/images/city.webp') center / cover no-repeat;
  color: #fff;
  padding: 3rem 1.5rem;
}
.banner::before {
  content: "";
  position: absolute; inset: 0;
  background: linear-gradient(0deg, rgb(0 0 0 / .55), rgb(0 0 0 / .25));
}
.banner > * { position: relative; } /* 오버레이 위에 컨텐츠 배치 */
```

## 1.4 반복 패턴과 SVG 배경

```css
.pattern {
  background-image: url('/images/dot.svg');
  background-repeat: repeat;
  background-size: 16px 16px;
}
```

SVG를 배경으로 쓰면 레티나에서도 선명하며 용량이 작습니다.

## 1.5 접근성과 대비

배경 위 텍스트는 충분한 대비를 확보해야 합니다. WCAG 대비비 계산식은 다음과 같습니다.

$$
C = \frac{L_{1} + 0.05}{L_{2} + 0.05}, \quad L_{1} \ge L_{2}
$$

본문 텍스트는 최소 4.5:1을 권장합니다. 밝은 이미지 위에는 오버레이를 더해 대비를 확보하세요.

# 2. 링크(Link) — 시맨틱, 보안, 내비게이션, 상태 스타일

## 2.1 기본 구조와 속성

```html
<a href="https://www.example.com">Example로 이동</a>
```

| 속성 | 설명 |
|---|---|
| `href` | 목적지 URL(절대/상대/스킴) |
| `target="_blank"` | 새 탭(또는 새 창) |
| `rel` | 보안·관계 정보(`noopener`, `noreferrer`, `nofollow`, `ugc`) |
| `title` | 보조 설명(남용 금지, 접근성 대체 수단 아님) |
| `download` | 링크 대상 다운로드 힌트(동일 출처 또는 CORS 허용 필요) |

### 외부 링크 보안 패턴

```html
<a href="https://news.example" target="_blank" rel="noopener">뉴스 열기</a>
```

`noopener`는 새 탭 페이지가 `window.opener`로 원본 페이지를 제어하는 것을 막아 클릭재킹 위험을 줄입니다. 외부로 보내면서 referrer를 숨기려면 `noreferrer`도 함께 고려합니다.

## 2.2 내부/외부/앵커/프로토콜 링크

```html
<!-- 외부 -->
<a href="https://www.google.com" target="_blank" rel="noopener">Google</a>

<!-- 내부 -->
<a href="/about.html">회사 소개</a>

<!-- 페이지 내부 앵커 -->
<a href="#section1">1번 섹션으로 이동</a>
<h2 id="section1">1번 섹션</h2>

<!-- 이메일/전화 -->
<a href="mailto:help@site.com?subject=문의">이메일 보내기</a>
<a href="tel:+821012345678">전화 걸기</a>
```

## 2.3 링크 상태와 포커스 가시성

```css
a { color: #1d4ed8; text-underline-offset: 2px; }
a:hover { text-decoration-thickness: 2px; }
a:focus-visible { outline: 2px solid #f59e0b; outline-offset: 2px; }
a:visited { color: #6b21a8; }
```

색상만으로 상태를 구분하지 말고 밑줄 두께, 아이콘 등 시각적 보조를 추가하세요.

## 2.4 스킵 링크와 내비게이션 도움

```html
<a class="skip" href="#main">본문으로 건너뛰기</a>
<main id="main">...</main>
```

```css
.skip {
  position: absolute; left: -9999px; top: auto;
}
.skip:focus {
  left: 1rem; top: 1rem; background: #fff; padding: .5rem .75rem;
  border: 1px solid #ddd; border-radius: .4rem;
}
```

키보드 사용자를 위해 문서 첫 부분에 스킵 링크를 제공합니다.

## 2.5 성능 힌트: 프리페치·프리로드

```html
<!-- 임박한 내부 경로 예열 -->
<link rel="prefetch" href="/next-article.html" as="document">
<!-- 폰트/이미지/리소스 미리 로드 -->
<link rel="preload" href="/fonts/Inter.woff2" as="font" type="font/woff2" crossorigin>
```

# 3. 이미지(Image) — 시맨틱, 반응형, 포맷, 성능

## 3.1 기본 문법

```html
<img src="/img/cat.jpg" alt="창가에 앉아 있는 고양이" width="300" height="200" loading="lazy">
```

| 속성 | 설명 |
|---|---|
| `src` | 파일 경로/URL |
| `alt` | 대체 텍스트(의미 없는 장식이면 `alt=""`) |
| `width`, `height` | 고유 픽셀 치수(레이아웃 시프트 방지) |
| `loading="lazy"` | 지연 로드(뷰포트 근처 도달 시 로드) |
| `decoding="async"` | 비동기 디코딩 힌트 |
| `title` | 툴팁(접근성 대체 아님) |

> 의미 없는 장식 이미지라면 CSS `background-image`를 고려하거나 `<img alt="">`로 스크린리더에서 무시되도록 합니다.

## 3.2 반응형 이미지: `srcset`와 `sizes`

```html
<!-- 뷰포트 폭에 따라 다른 크기 이미지 제공 -->
<img
  src="/img/hero-1200.webp"
  srcset="/img/hero-600.webp 600w, /img/hero-900.webp 900w, /img/hero-1200.webp 1200w"
  sizes="(max-width: 640px) 90vw, (max-width: 1024px) 80vw, 1200px"
  alt="산 정상 풍경"
  width="1200" height="675" loading="lazy">
```

- `srcset`의 `w`는 이미지의 고유 너비.
- `sizes`는 CSS 레이아웃에서 실제 차지할 너비를 브라우저에 힌트로 제공.
- 결과적으로 네트워크/해상도에 맞는 소스를 자동 선택합니다.

## 3.3 밀도 대응: DPR 기반 `x` 디스크립터

```html
<img
  src="/img/logo@1x.png"
  srcset="/img/logo@1x.png 1x, /img/logo@2x.png 2x, /img/logo@3x.png 3x"
  alt="로고" width="160" height="40">
```

## 3.4 포맷 선택: AVIF/WebP/JPEG/PNG/SVG

- 사진: **AVIF**(최상 압축·품질) → **WebP** → JPEG 순으로 폴백.
- 아이콘/벡터: **SVG** 권장.
- 투명/아이콘 스프라이트: SVG 또는 PNG.
- 애니메이션: 비트맵 대신 **video** 또는 Lottie/SVG 애니메이션 고려.

## 3.5 `<picture>`로 아트 디렉션

```html
<picture>
  <source type="image/avif" srcset="/img/cover-1200.avif 1200w, /img/cover-800.avif 800w" sizes="100vw">
  <source type="image/webp" srcset="/img/cover-1200.webp 1200w, /img/cover-800.webp 800w" sizes="100vw">
  <img src="/img/cover-1200.jpg" alt="도시 야경" width="1200" height="675" loading="lazy">
</picture>
```

미디어쿼리 기반으로 가로/세로 구성을 바꾸는 것도 가능합니다.

## 3.6 스타일링과 자르기

```css
.round {
  border-radius: 50%;
  border: 3px solid #e5e7eb;
  box-shadow: 0 2px 8px rgb(0 0 0 / .2);
}
.cover {
  width: 100%; height: 240px; object-fit: cover; object-position: center;
}
```

`object-fit: cover`는 `<img>`를 배경처럼 비율 유지 자르기로 보여줍니다.

## 3.7 콘텐츠 가독성 위한 오버레이

배경 섹션에서 설명한 오버레이 패턴을 `<img>` 위에 겹치는 카드, 히어로 등에도 동일하게 적용할 수 있습니다.

## 3.8 성능 체크리스트

- [ ] 크기 지정(`width`/`height`)으로 레이아웃 시프트(CLS) 방지
- [ ] `loading="lazy"` + 중요 이미지는 eager(Above the fold)
- [ ] 적절한 포맷(AVIF/WebP)과 품질
- [ ] `srcset`/`sizes`로 낭비 없는 전송
- [ ] 캐시 헤더, CDN 사용
- [ ] 스크롤 진입 감지로 단계적 로딩(고급)

# 4. 배경 vs 이미지 선택 기준

| 항목 | `<img>` | `background-image` |
|---|---|---|
| 의미 | 문서 콘텐츠(정보 전달) | 장식(스타일) |
| 접근성 | `alt`로 대체 텍스트 제공 | 의미 없음(보조기기 비가시) |
| 반응형 | `srcset`/`sizes`/`picture` | `background-size`/미디어쿼리 |
| 상호작용 | 클릭·드래그·컨텍스트 메뉴 | 보통 비상호작용(별도 요소 필요) |

콘텐츠적 의미가 있으면 `<img>`, 장식이면 `background-image`를 사용하세요.

# 5. 실전 예제 모음

## 5.1 히어로 섹션: 다중 배경 + 오버레이 + 대비 확보

```html
<header class="hero">
  <h1>지속 가능한 탐험</h1>
  <p>자연과 도시의 경계에서 이야기를 만듭니다.</p>
  <a class="btn" href="/stories">이야기 읽기</a>
</header>
```

```css
.hero {
  min-height: 50vh; display: grid; place-content: center; gap: 1rem;
  text-align: center; color: #fff; padding: 3rem 1rem;
  background:
    linear-gradient(180deg, rgb(0 0 0 / .45), rgb(0 0 0 / .45)),
    url('/images/forest.avif') center / cover no-repeat;
}
.btn {
  display: inline-block; padding: .75rem 1.25rem; border-radius: .6rem;
  background: #2563eb; color: #fff; text-decoration: none; font-weight: 600;
}
.btn:focus-visible { outline: 2px solid #f59e0b; outline-offset: 3px; }
```

## 5.2 문서 내 목차(앵커 내비게이션 + 스크롤 스파이)

```html
<nav aria-label="문서 목차" class="toc">
  <a href="#intro">소개</a>
  <a href="#usage">사용법</a>
  <a href="#faq">FAQ</a>
</nav>

<section id="intro"><h2>소개</h2>...</section>
<section id="usage"><h2>사용법</h2>...</section>
<section id="faq"><h2>FAQ</h2>...</section>

<script>
  // 인터섹션 옵저버로 현재 섹션에 대응하는 링크 강조
  const links = [...document.querySelectorAll('.toc a')];
  const map = Object.fromEntries(links.map(a => [a.getAttribute('href'), a]));
  const obs = new IntersectionObserver(entries => {
    entries.forEach(e => {
      if (e.isIntersecting) {
        links.forEach(l => l.classList.toggle('active', l.getAttribute('href') === `#${e.target.id}`));
      }
    });
  }, { rootMargin: '-40% 0px -55% 0px', threshold: 0 });
  document.querySelectorAll('section[id]').forEach(s => obs.observe(s));
</script>

<style>
.toc { position: sticky; top: 1rem; display: flex; gap: .75rem; padding: .5rem; background: #f8fafc; border-radius: .5rem; }
.toc a { text-decoration: none; color: #334155; padding: .25rem .5rem; border-radius: .375rem; }
.toc a.active, .toc a:hover { background: #e2e8f0; }
html { scroll-behavior: smooth; }
</style>
```

## 5.3 프로필 카드: `<img>` 스타일링 + `object-fit`

```html
<article class="profile">
  <img class="avatar" src="/img/profile.jpg" alt="김도현의 프로필 사진" width="160" height="160" loading="lazy" decoding="async">
  <h3>김도현</h3>
  <p>프론트엔드 엔지니어. 시맨틱·접근성·성능에 관심.</p>

```

```css
.profile { max-width: 320px; padding: 1rem; border: 1px solid #e5e7eb; border-radius: .75rem; }
.avatar { width: 120px; height: 120px; border-radius: 50%; object-fit: cover; object-position: center; display: block; }
```

# 6. 체크리스트(배포 전)

- [ ] 배경 대비가 충분한가(본문 4.5:1 이상)
- [ ] 외부 링크에 `rel="noopener"`(필요 시 `noreferrer`) 적용
- [ ] 스킵 링크 및 포커스 가시성 제공
- [ ] 의미 있는 이미지는 `<img alt="...">`, 장식은 `background-image` 또는 `alt=""`
- [ ] 이미지에 고유 `width`/`height` 포함(CLS 방지)
- [ ] `srcset`/`sizes` 또는 `<picture>`로 반응형 제공
- [ ] 적절한 포맷(AVIF/WebP), `loading="lazy"` 도입
- [ ] 정적 자산은 캐시 및 CDN 설정

    }

    nav a { text-decoration: none; color: #1d4ed8; }

          radial-gradient(circle at 20% 20%, rgb(0 0 0 / .05), transparent 40%),
