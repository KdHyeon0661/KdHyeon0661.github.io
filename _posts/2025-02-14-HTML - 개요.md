# HTML 소개와 역사

HTML(HyperText Markup Language)은 **웹 문서의 구조와 의미(semantics)**를 기술하는 **마크업 언어**입니다.  
프로그램 로직을 수행하는 언어가 아니라 **문서 요소를 선언**하고 **브라우저가 이를 해석**해 화면에 렌더링하도록 합니다.

아래 글은 당신이 제공한 초안을 **핵심을 유지**하면서, **실무 예제와 빠진 부분을 보강**해 확장한 버전입니다.

---

## 1. HTML은 왜 ‘마크업’인가?

- **목표**: 문서의 구조(heading, paragraph, list)와 의미(article, nav, main)를 **표준 태그로 표현**  
- **역할 분리**  
  - **HTML**: 구조/의미  
  - **CSS**: 표현/레이아웃  
  - **JavaScript**: 상호작용/로직

### 간단 예시

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="utf-8">
    <title>HTML은 마크업</title>
  </head>
  <body>
    <header>
      <h1>블로그 제목</h1>
      <nav aria-label="주 메뉴">
        <a href="/">홈</a>
        <a href="/posts">글</a>
        <a href="/about">소개</a>
      </nav>
    </header>
    <main>
      <article>
        <h2>글 제목</h2>
        <p>문단은 내용을 담고, 시맨틱 태그는 역할을 명확히 합니다.</p>
      </article>
    </main>
    <footer>© 2025</footer>
  </body>
</html>
```

## 2. HTML의 역사 (정리 + 확장)

당신의 정리 내용을 기반으로, 맥락을 덧붙였습니다.

### 2.1 타임라인

- **HTML 1.0 (1993)**: 텍스트/링크 중심, 웹 문서 간 연결의 출발
- **HTML 2.0 (1995)**: 폼/테이블 추가, 브라우저 간 합의 시도
- **HTML 3.2 (1997)**: 테이블/스크립트/스타일 통합, 브라우저 기능 묶음
- **HTML 4.01 (1999)**: 구조와 표현의 분리 강조, `Transitional/Strict/Frameset`
- **XHTML 1.0 (2000)**: XML 문법의 엄격성 도입(현실 적용은 제약 많음)
- **HTML5 (2014~)**: 시맨틱 요소, `<video>`, `<audio>`, `<canvas>`, 스토리지/폼 APIs, 모바일/멀티미디어 대응

### 2.2 버전 비교 (요지 재정리)

| 버전       | 발표 | 특징 |
|------------|------|------|
| HTML 1.0   | 1993 | 텍스트/하이퍼링크 |
| HTML 2.0   | 1995 | 폼, 테이블 |
| HTML 3.2   | 1997 | 스크립팅/스타일 |
| HTML 4.01  | 1999 | 구조/표현 분리, DTD |
| XHTML 1.0  | 2000 | XML 기반, 엄격 문법 |
| **HTML5**  | 2014~| 시맨틱/미디어/API/모바일 |

## 3. Doctype과 문서 스켈레톤

당신의 예시를 유지하면서, ‘왜’까지 덧붙입니다.

```html
<!-- 현대 HTML5는 간결한 Doctype -->
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>HTML5 문서</title>
</head>
<body>
  <!-- 콘텐츠 -->
</body>
</html>
```

과거 HTML 4.01/DTD 기반 선언은 **표현/문법 검증 기준**을 브라우저에 알려주기 위함이었습니다.  
HTML5부터는 **단일하고 간결한 Doctype**으로 표준 모드 렌더링을 유도합니다.

## 4. 시맨틱(의미) 태그 총정리

시맨틱 태그는 **접근성(스크린 리더)**과 **SEO(검색엔진)**에 직접적 이점이 있습니다.

- **문서 레이아웃**: `header`, `nav`, `main`, `article`, `section`, `aside`, `footer`
- **텍스트 구조**: `h1`~`h6`, `p`, `blockquote`, `cite`, `figure`, `figcaption`
- **상태/인터랙션**: `details`, `summary`, `dialog`, `progress`, `meter`
- **데이터 표현**: `table`, `thead`, `tbody`, `tfoot`, `caption`, `th`, `td`, `colgroup`, `col`

### 시맨틱 레이아웃 예제

```html
<main id="content">
  <article>
    <header>
      <h1>HTML 역사</h1>
      <p>작성일: <time datetime="2025-11-09">2025-11-09</time></p>
    </header>

    <section aria-labelledby="evolution">
      <h2 id="evolution">진화 과정</h2>
      <p>HTML은 ...</p>
    </section>

    <aside>
      <h2>참고</h2>
      <p>W3C, WHATWG 자료</p>
    </aside>

    <footer>
      <p>태그: <a href="/tags/html">#html</a></p>
    </footer>
  </article>
</main>
```

## 5. 접근성(Accessibility, a11y)

### 5.1 기본 원칙

- 의미 있는 요소 사용(버튼은 `<button>`, 링크는 `<a>`)
- 폼/이미지 대체 텍스트 제공
- 키보드 포커스 가능한 UI
- 명확한 문서 제목/헤딩 계층

### 5.2 실무 패턴

```html
<!-- 이미지 대체 텍스트 -->
<img src="/images/diagram.png" alt="HTML5 API 간 관계를 나타낸 다이어그램" />

<!-- ARIA로 의미 강화 (필요할 때만) -->
<nav aria-label="보조 메뉴">
  <ul>
    <li><a href="/sitemap">사이트맵</a></li>
  </ul>
</nav>

<!-- 폼 레이블/설명 -->
<label for="email">이메일</label>
<input id="email" name="email" type="email" aria-describedby="email-help" required>
<small id="email-help">업데이트 수신을 원하시면 체크하세요.</small>
```

## 6. SEO 기초

- **문서 하나당 고유한 `<title>`**
- **설명 메타**: `<meta name="description" content="...">`
- **의미 있는 헤딩 계층(H1→H2→H3)**
- **링크 텍스트**는 ‘여기 클릭’ 대신 의미 있는 문구 사용
- 이미지 `alt`에 **핵심 묘사**

```html
<head>
  <title>HTML 역사 총정리</title>
  <meta name="description" content="HTML의 역사와 현대 웹 기능을 예제로 설명합니다." />
  <link rel="canonical" href="https://example.com/posts/html-history" />
</head>
```

## 7. 폼(Forms) 심화: 검증/오토필/접근성

### 7.1 입력 타입과 기본 검증

```html
<form action="/subscribe" method="post" novalidate>
  <label for="em">이메일</label>
  <input id="em" name="email" type="email" required autocomplete="email" inputmode="email" />
  <button type="submit">구독</button>
</form>

<script>
  document.querySelector('form').addEventListener('submit', (e) => {
    const email = e.target.email;
    if (!email.checkValidity()) {
      e.preventDefault();
      email.reportValidity(); // 브라우저 기본 UI로 에러 표시
    }
  });
</script>
```

### 7.2 접근 가능한 에러 표시

```html
<div role="alert" aria-live="assertive" id="error-box" hidden></div>
<script>
  const errorBox = document.getElementById('error-box');
  function showError(msg) {
    errorBox.hidden = false;
    errorBox.textContent = msg; // 스크린리더가 즉시 읽음
  }
</script>
```

## 9. 그래픽: `<canvas>` vs SVG

- **Canvas**: 픽셀 기반(게임/시뮬레이션/실시간 그래프), 스크린리더 접근성 보완 필요
- **SVG**: 벡터 기반(아이콘/도표), DOM/스타일/상호작용과 친화적

### 9.1 Canvas 기본

```html
<canvas id="c" width="400" height="240" aria-label="실시간 주가 라인 차트" role="img"></canvas>
<script>
  const ctx = c.getContext('2d');
  ctx.beginPath();
  ctx.moveTo(10, 200);
  [50,120,90,160,140,180].forEach((y, i) => {
    ctx.lineTo(10 + i * 60, 240 - y);
  });
  ctx.stroke();
</script>
```

### 9.2 SVG 기본

```html
<svg viewBox="0 0 200 100" width="400" role="img" aria-label="막대그래프">
  <rect x="10" y="20" width="20" height="60"></rect>
  <rect x="50" y="40" width="20" height="40"></rect>
  <rect x="90" y="10" width="20" height="70"></rect>
  <title>분기별 매출</title>
</svg>
```

## 10. 저장소/네트워크/웹 앱 API (개요)

> HTML 자체는 마크업이지만, 현대 웹은 **표준 API**와 결합합니다.

- **Storage**: `localStorage`, `sessionStorage`, IndexedDB(대용량/구조화)
- **Network**: `fetch`, CORS, Streams
- **PWA**: Service Worker, Manifest, 오프라인/설치형 웹앱

```html
<script>
  // 간단 캐시
  localStorage.setItem('theme', 'dark');
  const theme = localStorage.getItem('theme');

  // 네트워크 호출
  fetch('/api/posts').then(r => r.json()).then(console.log);
</script>
```

## 11. 보안 기초(콘텐츠 작성 관점)

- **신뢰할 수 없는 입력을 그대로 HTML에 삽입하지 않기(XSS)**  
  - 텍스트 노드는 `textContent`로 삽입  
  - `innerHTML` 사용 시 서버/클라이언트에서 반드시 sanitize
- **링크 보안**: 외부 탭 오픈 시 `rel="noopener noreferrer"`  
- **폼 CSRF 방지**: 서버 토큰 사용, `SameSite` 쿠키 설정

```html
<!-- 안전한 텍스트 삽입 -->
<div id="box"></div>
<script>
  const userInput = "<img src=x onerror=alert(1)>";
  document.getElementById('box').textContent = userInput; // 그대로 문자열로 렌더링
</script>
```

## 12. 성능 최적화(HTML 레벨)

- **리소스 힌트**: `preload`, `prefetch`, `dns-prefetch`
- **이미지**: `loading="lazy"`, `decoding="async"`, 적절한 `width/height`
- **스크립트**: `defer`(DOM 파싱 병행), `async`(독립 실행)

```html
<link rel="preload" href="/css/main.css" as="style">
<link rel="preload" href="/js/app.js" as="script">
<script src="/js/app.js" defer></script>
```

## 14. 테이블: 의미와 접근성

```html
<table>
  <caption>월별 방문자</caption>
  <thead>
    <tr>
      <th scope="col">월</th>
      <th scope="col">방문자</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">1월</th>
      <td>12,430</td>
    </tr>
  </tbody>
</table>
```

- `caption`/`scope`로 의미 부여  
- 복잡한 표는 `headers`/`id`로 관계 연결

## 15. 마이그레이션: 구형에서 현대 HTML로

- **프레임셋/프레임** → `<iframe>` + CSS 레이아웃
- 프리젠테이션 속성(`bgcolor`, `border`, `align`) → **CSS**로 이동
- 테이블 레이아웃 → **플렉스/그리드**

```html
<!-- 구형 -->
<table width="100%" border="0">
  <tr><td align="center"><b>제목</b></td></tr>
</table>

<!-- 현대 -->
<header class="page-header"><h1>제목</h1></header>
```

## 16. 문서 템플릿(블로그용)

```html

<html lang="ko">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>{{ page.title }}</title>

  <link rel="stylesheet" href="/assets/main.css" />

<body>
  <header>
    <a href="/" class="logo">블로그</a>
    <nav aria-label="주 메뉴">
      <a href="/posts">글</a>
      <a href="/tags">태그</a>
      <a href="/about">소개</a>
    </nav>

  <main id="content">
    <article>
      <h1>{{ page.title }}</h1>
      <p><time datetime="{{ page.date }}">{{ page.date }}</time></p>
      {{ content }}
    </article>
  </main>

  <footer>
    <small>© {{ site.time | date: "%Y" }}</small>
  </footer>
</body>
</html>
```

## 18. 체크리스트(작성 전/후)

- [ ] `<!DOCTYPE html>`와 `<meta charset="utf-8">` 포함
- [ ] `<html lang="ko">` 명시
- [ ] 유일한 `<h1>`과 논리적 헤딩 계층
- [ ] 이미지 `alt` 제공
- [ ] 폼 `label`/에러 메시지/`aria-*` 검토
- [ ] 링크 텍스트가 목적을 설명
- [ ] 외부 링크에 `rel="noopener noreferrer"`
- [ ] 이미지 지연 로딩(`loading="lazy"`)
- [ ] 스크립트 `defer`/`async` 검토
- [ ] 메타 설명/정규 URL(canonical) 지정

## 19. 흔한 함정과 대처

1. **모든 것을 `<div>`로**: 시맨틱 태그로 교체  
2. **이미지에 빈 `alt`**: 의미가 있으면 꼭 설명  
3. **인터랙션을 `<a>`로**: 동작 트리거는 `<button>`  
4. **`innerHTML` 남용**: `textContent`/`DOM API` 우선  
5. **헤딩 스킵**: H1→H2→H3 순서 유지

## 20. 마무리

HTML은 웹의 **출발점이자 기반**입니다.  
현대 웹에서는 CSS/JS, 그리고 다양한 표준 API와 함께 결합되며, **시맨틱/접근성/보안/성능**을 고려한 작성이 곧 **품질**입니다.  
이 글은 당신의 초안을 유지하면서 **실전 중심 내용**을 상당 부분 보강했습니다. 이제 HTML 문서를 **의미 있게** 작성하고, **유지보수 가능한 블로그**를 운영할 수 있을 것입니다.

---
```html
<!-- 전체 예제: 역사 섹션을 포함한 블로그 글 스켈레톤 -->
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>HTML 소개와 역사 — 확장판</title>
  <meta name="description" content="HTML의 역사부터 시맨틱/접근성/성능까지 총정리">
  <link rel="stylesheet" href="/assets/blog.css">
</head>
<body>
  <header>
    <h1>HTML 소개와 역사</h1>
    <nav aria-label="주 메뉴">
      <a href="/">홈</a>
      <a href="/posts">글</a>
      <a href="/about">소개</a>
    </nav>
  </header>

  <main id="content">
    <article>
      <header>
        <h2>왜 HTML인가</h2>
        <p><time datetime="2025-11-09">2025-11-09</time></p>
      </header>

      <section>
        <h3>핵심</h3>
        <p>HTML은 문서의 구조와 의미를 표현하는 마크업 언어입니다.</p>
      </section>

      <section aria-labelledby="history">
        <h3 id="history">역사</h3>
        <ul>
          <li><strong>1993</strong> — HTML 1.0: 링크/텍스트 중심</li>
          <li><strong>1995</strong> — HTML 2.0: 폼/테이블</li>
          <li><strong>1997</strong> — HTML 3.2: 스크립트/스타일</li>
          <li><strong>1999</strong> — HTML 4.01: 구조/표현 분리</li>
          <li><strong>2000</strong> — XHTML 1.0: XML 문법</li>
          <li><strong>2014~</strong> — HTML5: 시맨틱/미디어/API</li>
        </ul>
      </section>

      <section>
        <h3>문서 유형</h3>
        <pre><code class="language-html">&lt;!DOCTYPE html&gt;</code></pre>
      </section>
```
