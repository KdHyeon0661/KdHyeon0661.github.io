## 0) 출제 맵(한눈 정리)

- **L2 보안**: VLAN, STP/BPDU Guard, DHCP Snooping, DAI, IP Source Guard, Port Security  
- **L3/라우팅**: uRPF(Loose/Strict), ACL(표준/확장), NAT, Bogon Drop, CoPP  
- **암호화/원격접속**: TLS 1.2/1.3, HSTS, IPsec(IKEv2/ESP), SSL VPN(OpenVPN), SSH Hardening  
- **무선**: WPA3(SAE/PMF), 802.1X/EAP-TLS, 게스트/IoT 분리  
- **경계 보안**: FW/IPS/IDS/WAF, L7 정책(Reverse Proxy), DDoS 완화(레이트리밋/애니캐스트)  
- **가시성/로깅**: NetFlow/IPFIX, SPAN/TAP, SIEM(상관규칙), DNS/HTTP/Radius 로그  
- **클라우드/K8s**: SG/NACL 기본거부, S3 공개 차단, K8s NetworkPolicy 기본거부+mTLS  
- **ZTNA/SASE**: OIDC/JWT, mTLS, 디바이스 포스쳐, 앱 단위 최소권한  
- **취약점/미스컨피그**: 오픈 리졸버, 0.0.0.0/0, 기본 비번, 관리 포트 노출  
- **사고대응**: 식별→격리→근본원인→재발방지, 증거 보존/시계 동기화(NTP)

## 1) **기출 포인트 25선(정리 + 함정)**

1. **DHCP Snooping**: 바인딩 테이블 생성(맥/아이피/포트). *함정*: 업링크는 **trust** 필요.  
2. **Dynamic ARP Inspection(DAI)**: ARP 응답을 바인딩과 대조. *함정*: 스태틱 IP 호스트 예외 처리 필요.  
3. **IP Source Guard**: DHCP Snooping 바인딩과 매칭되지 않으면 L3 드롭.  
4. **uRPF**: Loose(어디로든 역경로만 존재), Strict(수신 인터페이스와 동일 경로). *함정*: 비대칭 라우팅.  
5. **ACL 평가 순서**: 상단부터 일치, default deny. *함정*: 암시적 deny 잊지 않기.  
6. **Bogon Drop**: RFC1918/예약대역 외부 인바운드 차단.  
7. **TLS 1.3 차이**: 0-RTT(재전송·replay 주의), Handshake 축소, Suite 단순화(AEAD 필수).  
8. **HSTS**: HTTPS 강제, 프리로드 목록. *함정*: 서브도메인 포함 옵션.  
9. **IPsec**: IKEv2로 SA 협상→ESP로 데이터. Transport vs Tunnel. NAT-T(UDP/4500).  
10. **OpenVPN**: UDP 권장, `tls-crypt`로 메타데이터 은닉, Split/Full Tunnel.  
11. **WPA3-Personal(SAE)** + **PMF** 필수. *함정*: 약한 패스면 여전히 위험.  
12. **WPA3-Enterprise(EAP-TLS)**: 단말 인증서, 동적 VLAN. *함정*: CRL/OCSP 운영.  
13. **WAF vs IPS**: WAF는 HTTP/앱 계층 규칙(OWASP Top10), IPS는 범용 시그니처/행위.  
14. **L7 레이트 리미트**: burst/초당 제한으로 L7 DoS 억제.  
15. **NetFlow vs IPFIX**: v9 템플릿 vs 표준 템플릿(IPFIX=IETF). 샘플링 주의.  
16. **SPAN vs TAP**: 소프트웨어 복제 vs 패시브/오류프레임 포함. 포렌식은 TAP 우선.  
17. **K8s NetworkPolicy**: **기본거부**가 핵심. *함정*: Ingress/Egress 분리 기술.  
18. **Service Mesh mTLS**: 서비스 간 식별·암호화, AuthZ로 메서드/경로 제한.  
19. **클라우드 SG**: 0.0.0.0/0 인바운드 금지, 아웃바운드 최소화. IaC 정책 게이트.  
20. **DNS 보안**: DNSSEC, 내부 리졸버 고정, 외부 DNS egress 차단.  
21. **BGP 보안**: TTL-Security, Max-Prefix, RPKI/ROA.  
22. **로그 상관**: 실패 인증 급증 + 외부 DNS 시도 + egress 증가 = 침해 의심.  
23. **DDoS 증폭계수**:  
    $$ \text{증폭} = \frac{\text{응답바이트}}{\text{요청바이트}} $$  
    NTP/CLDAP/UDP 기반 반사형 주의.  
24. **무선 WIPS**: Rogue/Evil Twin 탐지, Deauth(802.11w로 차단).  
25. **SSO/ZTNA**: OIDC/JWT, audience/exp/nbf 검증, 짧은 수명/회전.

## 3) **단답/서술형(10문항) — 모범답안(키워드)**

**S1.** DHCP Snooping–DAI–IP Source Guard의 **연계 흐름**을 설명하라.  
- *모범답안 키워드*: Snooping 바인딩(IP–MAC–Port) 생성 → DAI가 ARP 응답 검증 → IP Source Guard가 바인딩되지 않은 소스 IP의 L3 트래픽 차단.

**S2.** uRPF Strict/Loose 차이를 메리트/디메리트 포함 서술.  
- *키워드*: Strict=보안↑/비대칭 환경 차단 위험, Loose=호환성↑/보안↓.

**S3.** TLS 1.3에서 허용 Cipher와 0-RTT 주의점을 기술.  
- *키워드*: AES-GCM/ChaCha20-Poly1305, 0-RTT는 재생(replay) 위험 → 중요한 요청에는 금지.

**S4.** WPA3-Enterprise 구축 시 **PKI 운영**에서 필수 체크 3가지.  
- *키워드*: CA 신뢰사슬, CRL/OCSP, 단기 인증서/자동 갱신(MDM).

**S5.** NetFlow/IPFIX의 Active/Inactive Timeout 의미.  
- *키워드*: Inactive=무활동시 플로우 종료, Active=장수 플로우 분할→가시성/메모리 관리.

**S6.** DMZ에서 아웃바운드 Egress 최소화가 중요한 이유.  
- *키워드*: 탈취/콜백·C2 차단, 취약 시스템의 외부 접근 억제, 공격 표면 축소.

**S7.** ZTNA에서 “앱 단위 접근”을 기술적으로 실현하는 방법 2가지.  
- *키워드*: OIDC/JWT 클레임(roles/aud), mTLS로 서비스 식별.

**S8.** DDoS 증폭에서 증폭계수 정의와 UDP의 특성.  
- *키워드*: $$\text{amp}=\frac{\text{response}}{\text{request}}$$, UDP=비연결·소스 스푸핑 용이.

**S9.** RA-Guard가 차단하는 프레임 유형과 배치 위치.  
- *키워드*: IPv6 Router Advertisement, Access Switch 단 포트.

**S10.** K8s에서 NetworkPolicy **기본거부**를 켜지 않을 때의 위험.  
- *키워드*: 네임스페이스 내 포드 간 무제한 통신, 측면이동(blast radius) 확대.

## 4) **시나리오형 구성 문제(5세트) — 설정/정답 예시**

### 세트A) 사무실 VLAN10에서 ARP 스푸핑으로 장애 다발  
**요구**: L2 보호 3종(DHCP Snooping, DAI, IP Source Guard) 적용. **Cisco IOS 예**:

```cisco
ip dhcp snooping
ip dhcp snooping vlan 10
ip arp inspection vlan 10
!
interface Gi1/0/1
 switchport access vlan 10
 ip verify source
 ip dhcp snooping limit rate 30
 spanning-tree portfast
!
interface Gi1/0/48   ! 업링크
 ip dhcp snooping trust
 ip arp inspection trust
```

**정답 포인트**: 업링크 trust, 레이트리밋, 포트패스트, DAI enable.

---

### 세트B) 인터넷→DMZ의 WAF 뒤 API 서버 보호 (Nginx)  
**요구**: HTTPS 강제, L7 rate-limit 5r/s, 백엔드로 헤더 전달.

```nginx
server {
  listen 443 ssl http2;
  server_name api.example.com;
  ssl_certificate /etc/nginx/cert.pem;
  ssl_certificate_key /etc/nginx/key.pem;

  add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;

  limit_req_zone $binary_remote_addr zone=req:10m rate=5r/s;

  location /v1/ {
    limit_req zone=req burst=20 nodelay;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $remote_addr;
    proxy_pass http://app.internal:8080;
  }
}
```

**정답 포인트**: HSTS, `limit_req_zone`, `X-Forwarded-For`.

---

### 세트C) 온프레 ↔ 클라우드 IPsec S2S(StrongSwan, IPFIX 가시성 유지)  
**요구**: IKEv2/AES-GCM, NAT-T, Active Timeout 60s.

```conf
# /etc/ipsec.conf (요약)
conn s2s
  keyexchange = ikev2
  left=198.51.100.10  leftsubnet=10.0.1.0/24
  right=203.0.113.20  rightsubnet=10.0.2.0/24
  ike=aes256gcm16-prfsha384-ecp256
  esp=aes256gcm16
  dpddelay=30s dpdaction=restart
  auto=start
```

**IPFIX 수집기**
```bash
sudo nfcapd -w -l /var/flows/ipfix -p 4739 -T all -S 1
```

**정답 포인트**: IKE/ESP 제안 일치, DPD, NAT-T(UDP/4500), 플로우 수집 포트.

---

### 세트D) 무선 업무망 WPA3-Enterprise(EAP-TLS) + 동적 VLAN  
**RADIUS 응답(발췌)**

```conf
update reply {
  Tunnel-Type := VLAN
  Tunnel-Medium-Type := IEEE-802
  Tunnel-Private-Group-Id := "30"
}
```

**AP 설정 포인트**: `WPA-EAP`, `ieee80211w=2(PMF Required)`, RADIUS secret 일치.  
**정답 포인트**: 인증서/CA, CRL/OCSP, 동적 VLAN 속성 3종.

---

### 세트E) K8s 주문 서비스만 `/orders/*` POST 허용 (Service Mesh)  
```yaml
apiVersion: security.istio.io/v1beta1
kind: AuthorizationPolicy
metadata: { name: orders-only-post, namespace: prod }
spec:
  selector: { matchLabels: { app: orders } }
  rules:
  - from:
    - source:
        principals: ["spiffe://cluster.local/ns/prod/sa/web-sa"]
    to:
    - operation:
        methods: ["POST"]
        paths: ["/orders/*"]
```

**정답 포인트**: mTLS 전제(Workload 식별), 메서드/경로 제한.

## 5) **로그 해석 문제(8문항) — 정답/해설**

**L1. Suricata EVE JSON (요약)**
```json
{"event_type":"alert","src_ip":"10.10.1.50","dest_ip":"8.8.8.8","dest_port":53,
 "alert":{"signature":"ET DNS Non-Standard TTL Low","severity":2}}
```
**질문**: 의심 포인트와 조치?  
**정답**: 낮은 TTL 반복은 DNS 스푸핑·포이즈닝 징후 → egress DNS 고정, 내부 리졸버만 허용.

---

**L2. FreeRADIUS 디버그**
```
eap_ttls: Tunneled authentication will be mschapv2
WARNING: TLS Alert read:fatal:certificate unknown
```
**정답**: 클라이언트가 CA/서버 인증서 불신 → 신뢰사슬 배포, CN/SAN·시간/NTP 점검.

---

**L3. nfdump Top Talker**
```
Dst Port 4444 bytes 5.2GB from 10.20.3.44 -> 203.0.113.66
```
**정답**: 비표준 외부 포트 대량 전송 → DLP/유출 의심, 즉시 차단·포렌식.

---

**L4. NGINX Access**
```
"POST /login HTTP/1.1" 200 45 "-" "curl/8.0" xff="-" rt=0.002
```
**정답**: XFF 부재/스크립트 접근이 다량이면 크리덴셜 스터핑 가능성 → 레이트리밋/캡차/락아웃.

**L5. Windows Security 4625(로그온 실패)**  
- **정답**: 임계 초과 시 소스 IP 차단·MFA 강제·계정 잠금 정책 확인.

**L6. Linux auth.log: `Failed password for root from 45.x.x.x`**  
- **정답**: root SSH 금지, Fail2ban/Geo 차단, 배스천·SSM 사용.

**L7. K8s Audit: `user=system:anonymous resource=secrets`**  
- **정답**: 익명 접근 차단, RBAC 재검토, API 서버 방화벽.

**L8. IPsec `CHILD_SA rekey failed`**  
- **정답**: 수명/제안 미일치, 양단 암호군 동기화·시간 확인.

- DMZ 아웃바운드 **최소** / Reverse Proxy 앞세움  
- ZTNA = **OIDC/JWT + mTLS + 짧은 수명 토큰**  
- DNS egress **조직 리졸버만 허용**  
- 로그·지표는 **NTP 동기화**가 생명
