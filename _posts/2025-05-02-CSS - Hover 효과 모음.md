# Hover 효과 모음

## 0) Hover 설계 기본기

- **애니메이션 가능한 속성 중심**: `transform`, `opacity` → 합성 단계에서 처리되어 부드럽고 비용 적음  
- **레이아웃 재계산 유발 속성**(`width`, `height`, `left`, `top`, `margin`)은 지양  
- `transition: all` 대신 **명시적 속성 나열** 추천(성능·예측 가능성↑)

```css
/* 좋음 */
.card { transition: transform .25s ease, box-shadow .25s ease, opacity .2s; }

/* 지양: 불필요 속성까지 애니메이션 */
.card { transition: all .25s ease; }
```

## 1) 배경색 전환 (기본형 + 대비 자동 보정 패턴)

```css
.button {
  --bg: #2563eb;
  --bg-hover: #1e40af;
  --fg: #fff;
  background: var(--bg);
  color: var(--fg);
  padding: .9rem 1.4rem;
  border-radius: 10px;
  transition: background-color .25s ease, color .25s ease;
}
.button:hover { background: var(--bg-hover); }
```

**대비 자동 보정(옵션):** 배경이 매우 밝아지면 글자색을 자동 조정하는 전략은 JS가 필요하지만,  
디자인 시스템에서 **톤/레벨 토큰**으로 미리 정해 두면 깔끔합니다.

## 2) 크기 확대(scale) + 미세 들뜸(translateY)

```css
.card {
  transform: translateY(0) scale(1);
  transition: transform .25s ease;
  will-change: transform;
}
.card:hover { transform: translateY(-2px) scale(1.03); }
```

> 순서 주의: `transform`은 **적용 순서**에 따라 결과가 달라짐.  
> (예: `scale → translate` vs `translate → scale`)

## 7) 타이포 강조(자간/굵기/색)

```css

.thumb:hover figcaption { opacity:1; transform: translateY(0); }
```

## 17) `:has()`로 “내부 요소 호버 시 카드 강조”(서포트 최신)

```css
/* 카드 내부 버튼이나 썸네일 위에 올라가면 카드 전체를 강조 */
.card:has(:hover) {
  transform: translateY(-2px);
  box-shadow: 0 10px 24px rgba(0,0,0,.1);
}
```

> `:has()` 지원 상황(MDN) 확인. 미지원 브라우저는 JS로 `mouseenter/leave` 델리게이션 처리.

## 18) Hover 대체: 터치 디바이스 전략

- `:hover`는 터치 환경에서 **의도가 불분명**할 수 있음  
- 보편 대안:
  - **`:active`**(탭 다운 동안)로 간단한 피드백
  - **`:focus-visible`**로 키보드 사용자 시각 피드백
  - **클릭/탭 토글**(JS) → `data-open="true"` 등 상태 클래스로 스타일 분기

```css
.btn:active { transform: scale(.98); }
.btn:focus-visible { outline: 2px solid #0ea5e9; outline-offset: 2px; }
```

```html
<button class="chip" aria-pressed="false">필터</button>
```

```css
.chip[aria-pressed="true"] { background:#111; color:#fff; }
```

```js
document.querySelectorAll('.chip').forEach(ch =>
  ch.addEventListener('click', e => {
    const v = ch.getAttribute('aria-pressed') === 'true';
    ch.setAttribute('aria-pressed', String(!v));
  })
);
```

## 19) 모션 최소화: `prefers-reduced-motion`

```css
@media (prefers-reduced-motion: reduce) {
  * { transition: none !important; animation: none !important; }
}
```

> 대신 **색/굵기/밝기** 등 정적 변화로 피드백을 제공.

## 20) 성능 팁 요약

- **합성 친화 속성**만 애니메이션: `transform`, `opacity`  
- `will-change`는 **필요한 요소에 한시적**으로만  
- 대규모 리스트: **스태거**/지연 분산, 뷰포트 내에서만 애니메이션  
- DevTools → Performance/Rendering로 FPS와 페인트 확인

---

## 21) 접근성 체크리스트

- 인터랙션 요소는 **키보드 탐색 가능**: `tabindex`/네이티브 버튼/링크 사용
- 포커스 표시: `:focus-visible` 스타일 제공
- 정보 전달을 **색만**에 의존하지 않기(텍스트·아이콘 함께)
- 툴팁: `role="tooltip"`/연결(`aria-describedby`) 고려, 포커스로도 열림

## 23) 디버깅/품질 체크

- **호버가 안 먹는다**: 상위 요소에 `pointer-events: none`? z-index/stacking-context?  
- **튀는 느낌**: `transition-timing-function`을 `cubic-bezier(.2,.7,.2,1)` 등으로 튜닝  
- **레이아웃 점프**: `border` 증가 대신 `box-shadow` 사용, 공간 변화를 최소화  
- **폰트 굵기 깜빡임**: 웹폰트 가변축(Variable Font) 사용 또는 초기에 해당 weight preload  
- **접근성**: `:focus-visible` 스타일 누락 여부, 툴팁은 키보드로도 열리는지 확인

## 25) 요약

- **핵심 속성**: `transform`, `opacity` 중심 → 부드럽고 빠른 호버  
- **보편 패턴**: 배경/스케일/섀도우/밑줄/이미지 캡션/툴팁/그라디언트  
- **접근성**: `:focus-visible`, 키보드/터치, 모션 축소 모드 존중  
- **성능**: 명시적 `transition` 속성, 스태거, `will-change` 절제  
- **현대 CSS**: `:has()`/변수/그리드·플렉스와 조합해 유지보수성↑

필요에 따라 각 패턴을 **토큰화(디자인 토큰)**하고 유틸리티 클래스로 묶어  
프로젝트 전반에 **일관된 인터랙션 언어**를 구축하세요.
