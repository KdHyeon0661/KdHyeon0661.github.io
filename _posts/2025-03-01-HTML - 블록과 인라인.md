# HTML 블록 요소 vs 인라인 요소

## vs 인라인(Inline)의 핵심 차이

| 구분 | 블록 요소 | 인라인 요소 |
|---|---|---|
| 줄바꿈 | 항상 새 줄에서 시작 | 줄바꿈 없이 라인 박스(line box) 안에 배치 |
| 가로 폭 | 기본적으로 가용 너비 전체(`auto` → 부모 콘텐츠 폭) | 내용 크기만큼 |
| 세로 치수 | 콘텐츠 높이만큼 | 라인 박스의 line-height/폰트 메트릭에 영향 |
| 박스 모델 | `width/height/padding/margin` 모두 직관적으로 적용 | `width/height` 미적용(대체로), `padding/margin` 수직방향 렌더링 영향에 제약 |
| 포맷팅 컨텍스트 | **BFC** 생성 가능(조건 충족 시) | **IFC**(인라인 포맷팅 컨텍스트) 내에서 줄 단위 흐름 |
| 대표 요소 | `div`, `p`, `h1~h6`, `ul/ol/li`, `section/article/nav/footer`, `table` 등 | `span`, `a`, `em/strong`, `i/b`, `img`, `input`, `label`, `abbr` 등 |

> 인라인 요소라도 `img`, `input`, `video` 같은 **replaced 요소**는 고유의 치수 계산 규칙을 갖습니다(아래 §6).

---

## 브라우저의 레이아웃 흐름과 포맷팅 컨텍스트

### BFC(Block Formatting Context)

- **블록 박스들의 레이아웃 영역**.
- 특정 조건에서 **새로운 BFC**가 만들어지면(예: `float`, `overflow`가 `hidden/auto/scroll`, `display: flow-root` 등) **부유 요소(float) 감싸기**, **마진 병합 차단**, **컨테이닝 블록 형성** 등 레이아웃 문제가 해결됩니다.

```html
<style>
.wrap { border: 2px solid #ddd; }
.float { float: left; width: 100px; height: 100px; background: #cce; }
.fix  { background: #efe; /* BFC x → 콘텐츠가 float에 겹칠 수 있음 */ }
.fix-bfc { background: #efe; overflow: auto; /* BFC 생성 → float 감싸짐 */ }
</style>

<div class="wrap">
  <div class="float">F</div>
  <div class="fix">BFC 없음</div>
</div>

<div class="wrap" style="margin-top:1rem;">
  <div class="float">F</div>
  <div class="fix-bfc">BFC 생성으로 float 감싸기</div>
</div>
```

### IFC(Inline Formatting Context)

- **인라인 요소가 라인 박스** 안에서 흐르며, **단어 분리/줄바꿈** 규칙(`white-space`, `word-break`, `overflow-wrap`)에 따라 줄이 만들어집니다.
- **baseline 정렬**과 `vertical-align`이 작동하는 영역(§7 참고).

## 마진 병합(Margin Collapsing)과 블록 간 간격

- 인접한 **수직 마진**이 **하나로 합쳐지는 현상**. 대표적으로:
  - 부모와 첫/마지막 자식의 상하 마진
  - 인접 형제 블록 요소의 상하 마진
- **병합 방지**: 패딩/보더 추가, `overflow`로 BFC 생성, `display: flow-root` 등

```html
<style>
.box { background:#f9fafb; border:1px solid #e5e7eb; }
.box p { margin: 1rem 0; }  /* p 사이 마진 병합 → 1rem만 남는 상황 발생 */
.box.flow-root { display: flow-root; } /* 내부 BFC로 부모-자식 병합 완화 */
</style>

<div class="box">
  <p>문단 A</p>
  <p>문단 B</p>
</div>

<div class="box flow-root" style="margin-top:1rem;">
  <p>문단 A</p>
  <p>문단 B</p>
</div>
```

## 인라인 박스 모델, 라인 박스, 베이스라인

인라인 컨텐츠는 **라인 박스** 안에 배치되며, **폰트 메트릭**(ascent/descent/line-gap)과 `line-height`에 의해 줄 높이가 결정됩니다.

```html
<style>
.demo span { background: #e0f2fe; }
.demo .big { font-size: 40px; }
.demo .small { font-size: 12px; }
</style>

<p class="demo">
  기본<span class="big">큰 글자</span>와 <span class="small">작은 글자</span>의
  베이스라인 정렬 확인
</p>
```

- 같은 라인에서 폰트 크기가 다르면 베이스라인을 기준으로 상하 정렬되며 **시각적 높낮이 차이**가 생깁니다.

## `display` 전환 — block/inline/inline-block/flow-root

### 인라인 → 블록

```css
a { display: block; }
```
각 링크가 한 줄씩 차지합니다.

### 블록 → 인라인(권장: `inline-block`)

```css
div.badge { display: inline-block; padding: .25rem .5rem; border-radius: .5rem; }
```
- `inline`은 `width/height` 적용이 불가(대체로).
- **버튼/배지**처럼 크기 제어가 필요하면 `inline-block` 사용.

### `flow-root`로 미세 레이아웃 문제 해결

```css
.card { display: flow-root; } /* 자체 BFC 생성 → float 감싸기, 마진 병합 차단 */
```

## Replaced vs Non-replaced 요소

- **Replaced 요소**: 콘텐츠가 외부 리소스/UA에 의해 **치수와 렌더링이 대체**됨. 예: `img`, `video`, `canvas`, `input`, `iframe`.
  - 인라인이지만 `width/height` 지정 가능.
  - `vertical-align`으로 라인 내부 정렬 조정.
- **Non-replaced 요소**: 텍스트/자식 요소가 **직접 렌더링**. 예: `span`, `strong`, `em` 등.

```html
<style>
.icon { vertical-align: middle; }
</style>

텍스트와 <img class="icon" src="/icons/check.svg" alt="" width="16" height="16"> 아이콘의 수직 정렬
```

---

## `vertical-align`과 베이스라인 정렬(인라인 컨텍스트)

```html
<style>
.box { font-size: 16px; line-height: 1.4; }
img { width: 24px; height: 24px; }
.middle { vertical-align: middle; }
.text-top { vertical-align: text-top; }
</style>

<p class="box">
  기본<img src="/icons/star.svg" alt="">|
  middle<img class="middle" src="/icons/star.svg" alt="">|
  text-top<img class="text-top" src="/icons/star.svg" alt="">
</p>
```

- 기본값: `baseline`
- 자주 쓰는 값: `middle`, `text-top`, `text-bottom`
- 블록 요소에서의 `vertical-align`은 테이블/inline-table/특수 컨텍스트에만 의미가 있음.

---

## 인라인 요소의 padding/margin/line-height 규칙

- 인라인 요소에도 `padding`/`margin` 적용 가능하나, **수직 방향**은 라인 박스 높이에 제한되어 보이는 것에 주의.
- 클릭 영역(예: 인라인 링크)을 넓히려면 **`inline-block`** 으로 전환 후 패딩을 주는 것이 안전.

```css
a.link { display: inline-block; padding: .25rem .5rem; }
```

## 시맨틱과 접근성

- **시맨틱 태그**(`section/article/nav/footer/h1~h6`)는 기본이 블록 요소이며, 문서 구조/내비게이션/스크린리더에 이점을 줍니다.
- **인라인 태그**(예: `em`, `strong`, `abbr`)는 텍스트의 의미적 강조, 약어 등의 **어휘적 의미**를 부여.
- 레이아웃 목적만으로 시맨틱을 깨뜨리지 말고, **시맨틱은 유지 + CSS로 형태를 조절**하세요.

## 블록 vs 인라인 선택 가이드(실전)

| 목적 | 권장 |
|---|---|
| 문단/섹션 구성 | 블록(`p`, `section`, `div`) |
| 텍스트 중 특정 부분 강조/표기 | 인라인(`em/strong/span/abbr/code`) |
| 배지/버튼형 링크 | `a + display:inline-block` |
| 아이콘 삽입 | 인라인(`img`, `svg`) + `vertical-align` 조정 |
| float 겹침/마진 병합 문제 | `display: flow-root` 또는 `overflow:auto` |
| 콘텐츠 재배치/2차원 레이아웃 | `display: flex/grid` (필요 시 `inline-flex/inline-grid`) |

---

## flex/grid와 인라인/블록

- `display: flex`(블록 레벨 플렉스 컨테이너), `display: inline-flex`(인라인 레벨 플렉스 컨테이너)
- `display: grid`/`inline-grid` 도 유사.
- 이들은 **자체 포맷팅 컨텍스트**를 형성하며, 자식의 배치 규칙이 flex/grid에 의해 대체됩니다.

```html
<style>
.menu { display: inline-flex; gap: .75rem; list-style: none; padding: 0; margin: 0; }
.menu a { text-decoration: none; padding:.25rem .5rem; border-radius:.4rem; }
.menu a:hover { background: #f3f4f6; }
</style>

<ul class="menu">
  <li><a href="/">홈</a></li>
  <li><a href="/docs">문서</a></li>
  <li><a href="/about">소개</a></li>
</ul>
```

## 글흐름 제어: `white-space`, 줄바꿈, 하이픈

```html
<style>
.nowrap { white-space: nowrap; }               /* 줄바꿈 금지 */
.wrap    { white-space: normal; }              /* 기본 */
.hyphen  { hyphens: auto; }                    /* 자동 하이픈(브라우저/언어 의존) */
.break   { overflow-wrap: anywhere; }          /* 긴 토큰 강제 줄바꿈 */
</style>

<p class="nowrap">이 문장은 줄바꿈 없이 한 줄로 유지됩니다.</p>
<p class="break">supercalifragilisticexpialidocious_long_token</p>
```

인라인 흐름에서 **줄바꿈 제어**는 가독성/반응형 레이아웃에 큰 영향을 줍니다.

## 대표 패턴 예제

### 카드 목록: 블록 컨테이너 + 인라인 요소 조합

```html
<style>
.cards { display: grid; grid-template-columns: repeat(auto-fill,minmax(240px,1fr)); gap: 1rem; }
.card { border:1px solid #e5e7eb; border-radius:.75rem; padding:1rem; }
.tag { display:inline-block; padding:.1rem .5rem; background:#eef2ff; color:#3730a3; border-radius:.5rem; font-size:.875rem; }
</style>

<section class="cards">
  <article class="card">
    <h3>시맨틱 HTML</h3>
    <p>문서 구조를 명확히 하고 내비게이션과 접근성을 개선합니다.</p>
    <span class="tag">a11y</span>
    <span class="tag">semantics</span>
  </article>
  <article class="card">...</article>
</section>
```

### → inline-flex

```html
<style>
nav ul { display: inline-flex; gap: .75rem; list-style: none; padding: 0; margin: 0; }
nav a { padding: .5rem .75rem; display: inline-block; text-decoration: none; border-radius: .5rem; }
nav a:hover { background:#f3f4f6; }
</style>

<nav aria-label="주 메뉴">
  <ul>
    <li><a href="/">홈</a></li>
    <li><a href="/blog">블로그</a></li>
    <li><a href="/contact">연락처</a></li>
  </ul>
</nav>
```

### float 이슈를 BFC로 해결

```html
<style>
.media { border:1px solid #e5e7eb; border-radius:.5rem; padding: .75rem; display: flow-root; }
.media img { float: left; width: 120px; height: 120px; margin: .25rem .75rem .25rem 0; object-fit: cover; }
</style>

<article class="media">
  <img src="/img/profile.jpg" alt="">
  <h4>프로필</h4>
  <p>float된 이미지가 텍스트를 감싸되, 컨테이너가 깔끔히 높이 계산.</p>
</article>
```

## 디버깅/트러블슈팅 체크리스트

- [ ] 인라인 영역에 `width/height`가 안 먹는다 → `inline-block` 또는 `block` 전환
- [ ] 인접 블록 간 상단/하단 마진이 이상하다 → **마진 병합** 확인, `padding`/`border`/`flow-root`로 분리
- [ ] float를 컨테이너가 감싸지 못한다 → `overflow:auto` 또는 `display: flow-root`
- [ ] 인라인 아이콘이 위/아래로 어색 → `vertical-align: middle/text-bottom` 조정
- [ ] 텍스트 줄바꿈이 망가진다 → `white-space`, `overflow-wrap`, `hyphens` 설정
- [ ] 문서 구조가 혼란 → 시맨틱 태그 사용, 역할과 스타일 분리

---

## 요약

- **블록**: 레이아웃 단위(새 줄 시작, 가용 폭), **BFC**와 연관.
- **인라인**: 텍스트 흐름 단위(라인 박스), **IFC**/베이스라인/`vertical-align`과 연관.
- 인라인의 크기/패딩 제어는 **`inline-block`** 이 안전.
- 레이아웃 꼬임(마진 병합, float)은 **`flow-root`/BFC**로 해결.
- 의미(시맨틱)는 유지하고 **CSS로 형태를 전환**하되, 접근성과 가독성을 놓치지 말 것.

```html
<title>Block vs Inline Sandbox</title>

  body { font:16px/1.7 system-ui,sans-serif; max-width:820px; margin:2rem auto; padding:0 1rem; }
  h2 { margin-top:2rem; }
  .b { background:#f1f5f9; border:1px solid #e2e8f0; }
  .i { background:#fef9c3; }
  .inline span { background:#fde68a; }
  .badges a { display:inline-block; padding:.25rem .5rem; background:#e0e7ff; border-radius:.5rem; text-decoration:none; color:#111827; margin-right:.5rem; }
  .flow { display: flow-root; border:1px dashed #94a3b8; padding:.5rem; }
  .flow img { float:left; width:100px; height:100px; object-fit:cover; margin:.25rem .75rem .25rem 0; }
</style>

<h1>Block vs Inline</h1>

<h2>블록 요소</h2>
<div class="b">div(블록)</div>
<p class="b">p(블록)</p>

<h2>인라인 요소</h2>

<h2>버튼형 링크(인라인→inline-block)</h2>
<p class="badges">
  <a href="#">Docs</a>
  <a href="#">API</a>
  <a href="#">Guide</a>
</p>

<h2>float 이슈를 flow-root로 해결</h2>
<div class="flow">
  <img src="https://picsum.photos/120/120?1" alt="">

```
