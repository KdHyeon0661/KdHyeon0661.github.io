# HTML의 구조와 요소 구성 자세히 알아보기

당신이 제공한 초안의 골격(문서 기본 구조, 요소/계층, 시맨틱 태그)을 유지하면서 **실무에서 바로 쓰는 규칙·패턴·함정·리팩터링 예시**를 크게 보강했습니다.
모든 코드는 실행 가능한 최소 예제를 중심으로 제시하고, **접근성(a11y)·SEO·성능·보안** 과도 자연스럽게 연결합니다.

---

## HTML 문서의 기본 구조 (스켈레톤 + 이유)

가장 먼저 중요한 것은 **표준 모드 렌더링**과 **정확한 언어·인코딩·뷰포트 설정**입니다.

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8" />
  <title>문서 제목</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta name="description" content="이 문서에 대한 한 줄 설명" />
  <link rel="stylesheet" href="/assets/style.css" />
  <!-- 성능/리소스 힌트 예시 -->
  <link rel="preconnect" href="https://cdn.example.com" crossorigin />
  <link rel="preload" href="/assets/hero.webp" as="image" imagesrcset="/assets/hero.webp 1x" />
</head>
<body>
  <header>
    <h1>제목</h1>
    <nav aria-label="주 메뉴">
      <a href="/">홈</a>
      <a href="/posts">글</a>
      <a href="/about">소개</a>
    </nav>
  </header>

  <main id="content">
    <article>
      <h2>본문 주제</h2>
      <p>본문 내용</p>
    </article>
  </main>

  <footer>
    <small>© 2025</small>
  </footer>

  <script src="/assets/app.js" defer></script>
</body>
</html>
```

### 왜 이렇게 쓰는가?

- `<!DOCTYPE html>`: **표준 모드**를 강제해 브라우저 간 렌더링 차이를 줄입니다.
- `<html lang="ko">`: 스크린 리더와 검색엔진이 **문서 언어**를 인식합니다.
- `<meta charset="UTF-8">`: 인코딩 문제(깨짐) 예방.
- `<meta name="viewport">`: 모바일에서 **적정 스케일**로 보이게 합니다.
- `defer` 스크립트: DOM 파싱과 병행해 **렌더링 지연 최소화**.

## 요소(Element)의 기본 문법과 빈 요소

### 일반 요소 형식

```html
<태그이름 속성="값">내용</태그이름>
```

예:
```html
<a href="https://example.com">이동하기</a>
```

### 빈 요소(Self-closing처럼 보이는 HTML 빈 요소)

HTML에서 일부 요소는 **내용/종료 태그 없이** 쓰입니다.
```html
<img src="/img/logo.png" alt="사이트 로고">
<br>
<hr>
<input type="text" name="username">
<link rel="stylesheet" href="/assets/style.css">
<meta name="description" content="...">
```
> XHTML처럼 `/>`로 닫지 않아도 됩니다(HTML 파서는 빈 요소를 인지).

## DOM 계층 구조와 콘텐츠 모델

HTML은 **트리 구조(DOM)**로 표현되며, 요소는 **부모/자식/형제** 관계를 가집니다.

```html
<ul>
  <li>사과</li>
  <li>바나나</li>
</ul>
```

### DOM 트리 예시

```
html
├── head
│   └── title
└── body
    ├── h1
    └── p
```

### 콘텐츠 모델(매우 중요)

요소는 **어떤 자식을 가질 수 있는지**를 정의하는 **콘텐츠 모델**을 가집니다.

- **메타데이터 콘텐츠**: `<base>`, `<link>`, `<meta>`, `<script>`, `<style>`, `<title>` (주로 `<head>` 안)
- **플로우 콘텐츠**: 대부분의 시각적으로 보이는 요소(대부분 `<body>` 안에서 허용)
- **섹션 콘텐츠**: `<article>`, `<aside>`, `<nav>`, `<section>` (문서 구조 구분)
- **헤딩 콘텐츠**: `<h1> ~ <h6>` (제목)
- **프레이징 콘텐츠**: 텍스트와 인라인 요소(`<a>`, `<span>`, `<em>`, `<strong>`, `<img>` 등)
- **임베디드 콘텐츠**: `<img>`, `<video>`, `<audio>`, `<canvas>`, `<iframe>`, `<svg>`
- **인터랙티브 콘텐츠**: `<a>`, `<button>`, `<details>`, `<summary>`, `<input>` 등

> 올바른 콘텐츠 모델을 지키면 **접근성/SEO/브라우저 호환성**이 좋아집니다.

## 블록 vs 인라인, 그리고 현대 레이아웃

- **블록 레벨**: 라인을 차지하며 개행. 예) `<div>`, `<p>`, `<section>`, `<ul>`, `<article>`
- **인라인**: 라인 내부에 흐름. 예) `<span>`, `<a>`, `<em>`, `<strong>`, `<img>`

> 레이아웃은 **CSS(Flexbox/Grid)**로 해결하세요. HTML은 **의미**에 집중합니다.

## 시맨틱 태그의 올바른 사용

| 태그        | 의미/용도                          | 안티패턴(피하기)                      |
|-------------|------------------------------------|---------------------------------------|
| `<header>`  | 문서/섹션의 머릿말                 | 단순히 큰 로고 이미지만 넣고 끝내기  |
| `<footer>`  | 문서/섹션의 꼬릿말                 | 모든 페이지 하단 레이아웃용 div 대체 |
| `<nav>`     | 내비게이션 링크 묶음               | 모든 링크 묶음을 nav로 남발          |
| `<main>`    | 문서의 주요 콘텐츠(하나만)         | 여러 개 사용                          |
| `<article>` | 독립적으로 배포 가능한 콘텐츠 블록 | 단락마다 article로 남발               |
| `<section>` | 주제별 구획                        | 단순 스타일링용으로 사용              |
| `<aside>`   | 보조 콘텐츠                        | 주요 내용 담기                         |

### “div soup” → 시맨틱 리팩터링 예시

**Before (나쁜 예)**
```html
<div class="header">
  <div class="title">블로그</div>
```

```html
    <p>내용</p>
```

## 폼(Forms): 검증·오토필·접근성

### 기본 폼

```html
<form action="/subscribe" method="post" novalidate>
  <label for="em">이메일</label>
  <input id="em" name="email" type="email" required autocomplete="email" inputmode="email" />
  <button type="submit">구독</button>
</form>
```

### 브라우저 기본 검증 활용

```html
<script>
  document.querySelector('form').addEventListener('submit', (e) => {
    const email = e.target.email;
    if (!email.checkValidity()) {
      e.preventDefault();
      email.reportValidity();
    }
  });
</script>
```

### 접근 가능한 에러 알림

```html
<div id="form-error" role="alert" aria-live="assertive" hidden></div>
<script>
  function showError(msg) {
    const box = document.getElementById('form-error');
    box.hidden = false;
    box.textContent = msg;
  }
</script>
```

## 메타데이터·리소스 힌트·아이콘

```html
<head>
  <meta name="description" content="페이지 설명" />
  <link rel="canonical" href="https://example.com/posts/slug" />
  <link rel="icon" href="/favicon.ico" sizes="any">
  <link rel="icon" href="/icon.svg" type="image/svg+xml">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">
  <link rel="manifest" href="/site.webmanifest">
  <meta name="theme-color" content="#ffffff">
</head>
```

- `canonical`: 중복 페이지 문제 해소
- PWA `manifest`: 설치형 웹앱 지원

## 전역 속성(Global Attributes)와 데이터 속성

- **공통**: `id`, `class`, `title`, `hidden`, `style`(권장 X), `tabindex`, `lang`, `dir`, `draggable`, `contenteditable`
- **ARIA**: `role`, `aria-*` (필요할 때만, 기본 시맨틱을 우선)
- **사용자 데이터**: `data-*`

```html
<article data-post-id="42" aria-labelledby="t42">
  <h2 id="t42">글 제목</h2>
</article>
```

## 스크립트 로딩 전략(성능)

```html
<!-- 블로킹 방지 -->
<script src="/js/app.js" defer></script>

<!-- 독립 실행(의존성 없음) -->
<script src="/js/ads.js" async></script>

<!-- 모듈(ESM) -->
<script type="module">
  import { init } from '/js/init.js';
  init();
</script>
```

## 보안 기초: XSS/CSRF/외부 링크

- **XSS**: 신뢰할 수 없는 입력은 **HTML 이스케이프** 또는 안전한 API(`textContent`)로 삽입
- **외부 링크**: `target="_blank"` 시 `rel="noopener noreferrer"`
- **CSRF**: 서버에서 토큰 검증, `SameSite=Lax/Strict` 쿠키

```html
<div id="box"></div>
<script>
  const user = '<img src=x onerror=alert(1)>';
  document.getElementById('box').textContent = user; // 안전
</script>
```

## 성능 체크리스트(HTML 레벨)

- [ ] 이미지 `loading="lazy"`, `decoding="async"`, 명시적 `width/height`
- [ ] `preload`/`preconnect`/`dns-prefetch` 등 리소스 힌트 활용
- [ ] 스크립트 `defer/async/module`로 블로킹 최소화
- [ ] 중요한 CSS는 **critical CSS** 인라인, 나머지는 분리
- [ ] 과도한 DOM 깊이/노드 수 줄이기

- 루트에 `lang` 지정: `<html lang="ko">`
- 텍스트 방향이 바뀌면 `dir="rtl"` 등 사용
- **기계친화적 시간**: `<time datetime="2025-11-09">2025-11-09</time>`

## 접근성(a11y) 핵심 규칙 요약

- 인터랙션 트리거는 **`<button>`**, 링크는 **`<a>`**
- **헤딩 계층**을 유지(H1 → H2 → H3 …)
- **대체 텍스트**(의미 있을 때 `alt` 제공)
- **폼 레이블** 명시, 에러는 `role="alert"`/`aria-live`로 공지
- 색상만으로 의미 전달하지 않기(텍스트/아이콘 보강)

## 레거시 HTML 리팩터링 실제 예

**Before** (스타일/레이아웃을 HTML 속성에 직접)
```html
<table width="100%" border="0">
  <tr><td align="center"><b>제목</b></td></tr>
</table>
```

**After** (의미/표현 분리)
```html
<header class="page-header"><h1>제목</h1></header>
```

**CSS**
```css
.page-header { display: grid; place-items: center; }
.page-header h1 { font-weight: 700; }
```

## 블로그 포스트 템플릿(시맨틱 + SEO + a11y)

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="utf-8" />
  <title>{{ page.title }}</title>
  <meta name="description" content="{{ page.excerpt | strip_html | truncate: 160 }}" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <link rel="canonical" href="{{ page.url | absolute_url }}" />
  <link rel="stylesheet" href="/assets/blog.css" />
</head>
<body>
  <header>
    <a class="logo" href="/">블로그</a>
    <nav aria-label="주 메뉴">
      <a href="/posts">글</a>
      <a href="/tags">태그</a>
      <a href="/about">소개</a>
    </nav>
  </header>

  <main id="content">
    <article>
      <header>
        <h1>{{ page.title }}</h1>
        <p>
          <time datetime="{{ page.date | date_to_xmlschema }}">
            {{ page.date | date: "%Y-%m-%d" }}
          </time>
        </p>
      </header>

      {{ content }}

      <footer>
        <p>태그:
          {% for tag in page.tags %}
            <a href="/tags/{{ tag | downcase }}">#{{ tag }}</a>{% unless forloop.last %}, {% endunless %}
          {% endfor %}
        </p>
      </footer>
    </article>
  </main>

  <footer>
    <small>© {{ site.time | date: "%Y" }}</small>
  </footer>
</body>
</html>
```

## 자주 하는 실수와 대처

1. 모든 걸 `<div>`로 처리 → **시맨틱 태그로 대체**
2. 의미 있는 이미지에 빈 `alt` → **적절한 대체 텍스트 제공**
3. 클릭 이벤트를 `<a>`로 남발 → 동작 트리거는 **`<button>`**
4. `innerHTML`로 사용자 입력 삽입 → **`textContent` 우선**
5. 헤딩 레벨 점프(H1→H4) → **순차적 계층 유지**

## 최종 요약

- 문서는 항상 **표준 모드/언어/인코딩/뷰포트**부터.
- 요소는 **콘텐츠 모델**을 지키며 배치(플로우·프레이징·섹션 등).
- 시맨틱 태그로 **의미를 드러내고**, 접근성과 SEO를 확보.
- 폼/미디어/인터랙션은 **기본 기능**을 최대 활용하고 a11y 고려.
- 메타데이터/리소스 힌트/보안 헤더로 **성능·안전성** 향상.
- 레거시 HTML은 **의미/표현 분리**로 리팩터링.
