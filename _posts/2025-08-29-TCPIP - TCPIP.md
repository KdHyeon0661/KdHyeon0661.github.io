# TCP/IP 프로토콜 스위트와 아키텍처

TCP/IP는 현대 인터넷을 가능하게 하는 핵심 통신 프로토콜 집합입니다. 이 프로토콜 스위트는 서로 다른 네트워크와 시스템이 표준화된 방식으로 데이터를 교환할 수 있는 기반을 제공합니다.

## TCP/IP 개요와 역사

TCP/IP의 역사는 1970년대 초 미국 국방부의 연구 기관인 **DARPA(Defense Advanced Research Projects Agency)** 가 시작한 **ARPANET** 프로젝트로 거슬러 올라갑니다. 초기 네트워크는 신뢰성과 확장성에 한계가 있었습니다.

1974년, 빈트 서프(Vint Cerf)와 밥 칸(Bob Kahn)이 **TCP/IP의 핵심 개념을 정립한 논문**을 발표하면서 변환점이 찾아왔니다. 핵심 혁신은 다음과 같습니다:
*   **패킷 스위칭(Packet Switching)**: 데이터를 작은 패킷으로 분할하여 전송
*   **인터네트워킹(Internetworking)**: 서로 다른 네트워크를 연결하는 게이트웨이(현재의 라우터) 개념
*   **엔드-투-엔드 원칙(End-to-End Principle)**: 네트워크 코어는 단순하게, 복잡한 기능은 끝단(호스트)에서 처리

1983년 1월 1일, ARPANET이 NCP(Network Control Protocol)에서 TCP/IP로 공식적으로 전환한 **"플래그 데이(Flag Day)"** 를 통해 TCP/IP는 사실상의 표준이 되었습니다. 이 결정이 오늘날의 글로벌 인터넷으로 가는 길을 열었습니다.

## TCP/IP 서비스와 클라이언트/서버 동작 모델

TCP/IP는 다양한 네트워크 서비스를 제공하며, 대부분 **클라이언트/서버 모델**을 기반으로 동작합니다.

**주요 서비스**:
*   **원격 로그인 (SSH, Telnet)**: 원격 컴퓨터에 접속하여 제어
*   **파일 전송 (FTP, SFTP, TFTP)**: 컴퓨터 간 파일 송수신
*   **전자 메일 (SMTP, POP3, IMAP)**: 이메일 발송 및 수신
*   **웹 서비스 (HTTP, HTTPS)**: 웹 페이지 및 리소스 접근
*   **도메인 이름 시스템 (DNS)**: 도메인 이름을 IP 주소로 변환
*   **네트워크 관리 (SNMP)**: 네트워크 장치 모니터링 및 관리
*   **동적 호스트 구성 (DHCP)**: IP 주소 자동 할당

**클라이언트/서버 동작**:
```
클라이언트(요청자)        서버(제공자)
      |                       |
      |----- 요청(Request) ---->|
      |                       |
      |<---- 응답(Response) ---|
      |                       |
```
*   **서버**: 특정 포트에서 대기(listening)하며 클라이언트 요청을 기다립니다. (예: 웹 서버는 80번 포트)
*   **클라이언트**: 서버의 IP 주소와 포트 번호로 연결을 시도하고 서비스를 요청합니다.
*   **포트 번호**: 한 컴퓨터에서 여러 서비스를 구분하기 위한 논리적 주소 (0-65535)

## TCP/IP 아키텍처와 TCP/IP 모델

TCP/IP는 일반적으로 **4계층 모델**로 설명되며, 이는 OSI 7계층 모델보다 실용적이고 단순화된 접근입니다.

**TCP/IP 4계층 모델**:
```
┌─────────────────────────────────────────┐
│ 4. 애플리케이션 계층 (Application)        │
│   - HTTP, FTP, DNS, SMTP 등            │
│   - 사용자와의 인터페이스 제공           │
├─────────────────────────────────────────┤
│ 3. 전송 계층 (Transport)                 │
│   - TCP(신뢰성 있는 연결) / UDP(비연결형) │
│   - 프로세스 간 통신, 흐름 제어, 오류 복구│
├─────────────────────────────────────────┤
│ 2. 인터넷 계층 (Internet)               │
│   - IP, ICMP, ARP 등                   │
│   - 논리적 주소 지정, 경로 선택(라우팅)   │
├─────────────────────────────────────────┤
│ 1. 네트워크 접근 계층 (Network Access)   │
│   - 이더넷, Wi-Fi, PPP 등              │
│   - 물리적 전송, 하드웨어 주소 지정      │
└─────────────────────────────────────────┘
```

**캡슐화(Encapsulation) 과정**:
```
데이터 → [애플리케이션 헤더+데이터] → [TCP 헤더+세그먼트] → [IP 헤더+패킷] → [프레임 헤더+트레일러+프레임]
          (애플리케이션 계층)          (전송 계층)          (인터넷 계층)      (네트워크 접근 계층)
```

## 주요 TCP/IP 프로토콜

### 인터넷 계층 프로토콜
*   **IP (Internet Protocol)**: 비연결형(connectionless), 신뢰할 수 없는(unreliable) 패킷 전달 서비스 제공. **라우팅**과 **주소 지정**의 핵심.
*   **ICMP (Internet Control Message Protocol)**: 네트워크 진단 및 오류 보고 (ping, traceroute)
*   **ARP (Address Resolution Protocol)**: IP 주소 → MAC 주소 변환
*   **RARP (Reverse ARP)**: MAC 주소 → IP 주소 변환

### 전송 계층 프로토콜
*   **TCP (Transmission Control Protocol)**:
    *   연결 지향적(3-way handshake)
    *   신뢰성 있는 전달(순서 보장, 재전송, 흐름 제어)
    *   세그먼트 단위 전송
    ```text
    3-Way Handshake:
    클라이언트            서버
      |--- SYN ----------->|
      |<-- SYN+ACK --------|
      |--- ACK ----------->|
    (연결 수립)
    ```
*   **UDP (User Datagram Protocol)**:
    *   비연결형, 신뢰성 없음
    *   오버헤드 작음, 실시간 응용에 적합(VoIP, 스트리밍)
    *   데이터그램 단위 전송

### 애플리케이션 계층 프로토콜
*   **HTTP/HTTPS**: 웹 통신
*   **FTP**: 파일 전송
*   **DNS**: 도메인 이름 시스템
*   **SMTP/POP3/IMAP**: 이메일 전송 및 수신
*   **DHCP**: IP 주소 자동 할당
*   **SNMP**: 네트워크 관리

## 결론

TCP/IP 프로토콜 스위트는 단순한 프로토콜 집합을 넘어, **인터넷이라는 거대한 네트워크의 생명줄이자 설계 철학** 그 자체입니다. 계층적 아키텍처는 명확한 책임 분리를 통해 유연성과 확장성을 보장했으며, 이는 기술이 급변하는 수십 년 동안도 TCP/IP가 지속적으로 진화하며 중심 역할을 할 수 있었던 이유입니다. 클라이언트/서버 모델은 네트워크 서비스의 표준 패러다임이 되었고, 오늘날 클라우드 컴퓨팅과 마이크로서비스 아키텍처의 기반이 되고 있습니다.

TCP와 UDP의 선택적 조화는 신뢰성과 효율성 사이의 근본적인 트레이드오프를 보여주며, 개발자와 엔지니어가 애플리케이션의 요구사항에 맞는 최적의 통신 방식을 선택할 수 있게 합니다. 결국 TCP/IP를 이해하는 것은 단순히 패킷이 어떻게 전송되는지를 아는 것을 넘어, 현대 디지털 사회의 연결과 소통이 어떻게 구현되는지에 대한 근본적인 통찰을 제공합니다.
