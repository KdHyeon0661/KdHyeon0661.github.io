# SECTION 18 정보보안 법규 — 07. 위치정보의 보호 및 이용 등에 관한 법률

> 본 장은 **「위치정보의 보호 및 이용 등에 관한 법률」(약칭: 위치정보법)** 및 하위 규정, 방송통신위원회(KCC) 공지·점검 관행을 바탕으로, **서비스·플랫폼에서 위치정보를 처리하는 실무자**(기획/개발/보안/법무)가 바로 적용할 수 있도록 구성했다. 2021.10.19 개정(법률 제18517호)으로 정비된 체계가 **2022.04.20 시행**되었으며, 이후 KCC의 **정기점검·행정지도**가 이어졌다. 관련 조문·요건은 국가법령정보센터 및 KCC·KISA 자료를 반드시 교차 확인하라.

---

## 1. 한눈에 보는 최신 개정 포인트(요약)

- **사업자 체계**
  - **위치정보사업자/개인위치정보사업자**: *등록(등록제)*
  - **위치기반서비스사업자(LBS 제공자)**: *신고(신고제)* — 기존 허가·복잡 절차를 간소화하는 방향으로 개정되었으나 **신고 의무는 유지**된다.
- **동의 고지 항목 보강** (법 제16조): 이용약관에 **확인자료 보유근거·기간**, **개인위치정보의 보유목적·기간(신설 4의2호)** 등을 명시하고, **부분동의(유보권)** 허용.
- **확인자료 보관 의무**: *수집·이용·제공 사실 확인자료*는 **최소 6개월 이상 보관**(다수 사업자 약관·가이드 기준).
- **개인위치정보 보유기간 기본 원칙**: 목적 달성 즉시 파기. **별도 동의 시 최대 1년** 혹은 **다른 법률상 보존의무가 있는 경우**에 한해 보유 가능(개정 취지).
- **예외 제공(긴급구조)**: 급박한 위험에서 **긴급구조기관**에 제공 가능(조문 근거 명확).

> 시험·실무 포인트
> ① **6개월(확인자료)** vs **즉시 파기/최대 1년(개인위치정보 본문)** 구분
> ② **등록/신고 대상 구분**, **동의 유보권** 및 **고지 항목 추가**
> ③ **긴급구조 제공 요건**과 **처리기록** 유지

## 2. 핵심 용어 / 사업자 구분

| 용어 | 정의/예시 | 규제 요건(요지) |
|---|---|---|
| **위치정보** | 개인위치정보 외의 위치 관련 정보(기지국ID 집계 등) | 사업 유형에 따라 등록·신고 |
| **개인위치정보** | 개인과 결합되어 특정 개인의 위치를 식별·추정 가능한 정보 | 동의·고지·권리보장·파기 의무 |
| **위치정보사업자** | 위치정보의 *생성·제공*을 업(통신사/지도·위치API 제공 등) | **등록** |
| **개인위치정보사업자** | 개인의 위치를 직접 수집·제공(디바이스·앱이 직접 수집) | **등록** |
| **위치기반서비스사업자(LBS)** | 위치정보를 *이용*해 서비스를 제공(배달/모빌리티/광고 타게팅 등) | **신고** |

> 등록·신고의 세부 절차·요건은 KCC 고시·안내문을 따른다. 사업모델에 따라 **복수 지위**(예: API 제공+자서비스 이용)일 수 있어, **사전 법률검토**가 필요하다.

## 3. 동의 체계(법 제16조): 고지 항목 체크리스트

**이용약관/동의서**에 다음을 명시해야 한다(필수):

1. 사업자 **상호·주소·연락처**
2. **정보주체·법정대리인 권리** 및 행사방법(열람/정정/삭제/처리정지·철회·일시중지 등)
3. 제3자 제공 시 **서비스 내용·범위**
4. **위치정보 수집·이용·제공 사실 확인자료**의 **보유근거·보유기간(≥6개월)**
4의2. **개인위치정보 보유 목적·보유기간**(기본 즉시 파기, *별도 동의 시 최대 1년*)
5. 그 밖에 대통령령으로 정하는 보호조치 사항
→ **부분동의(유보권)** 허용: 서비스 중 일부 항목에 동의하지 않아도 이용 가능하도록 구성.

**미성년·대리 동의**
- **만 14세 미만**: **법정대리인 동의 필수**, 대리인 권리(철회/일시중지/열람 등) 보장.

**샘플: 동의 UI 문구(핵심만)**

```text
[필수] 위치기반 서비스 제공을 위한 개인위치정보 수집·이용 동의
- 수집 목적: 근거리 매칭, 경로 안내
- 보유·이용 기간: 목적 달성 즉시 파기(별도 동의 시 최대 1년)
- 확인자료 보유 기간: 6개월(법 제16조제2항)
- 권리: 열람/정정/삭제/처리정지/동의철회/일시중지
- 문의: 000-0000, privacy@company.com
```

## 4. 보관·파기 요건과 **6개월/1년 룰**(자주 틀리는 지점)

- **확인자료(“수집·이용·제공 사실 확인자료”)**: **최소 6개월** 보관(서비스·사업자 약관 일치). **시스템 접근기록**도 6개월 보존 운영이 일반적.
- **개인위치정보(본문 데이터)**: 원칙 **즉시 파기**. 다만,
  - **별도 동의**가 있으면 **최대 1년** 보유 가능, 또는
  - **타 법령 보존의무**(세법 등)가 있으면 그 범위 내 보관.

> 실무 권고: 운영상 “최근 경로 보기(최대 30~90일)” 같은 UX가 필요하다면 **추가 명시적 동의**를 별도로 구분(토글)해 받는다.

**예시: 파기 배치(SQL 스케줄)**

```sql
-- 확인자료(6개월 경과) 자동 삭제
DELETE FROM location_fact_log
 WHERE event_time < (CURRENT_DATE - INTERVAL '180 days');

-- 개인위치정보(즉시/최대 1년): 보유만료일 컬럼 기반
DELETE FROM location_personal
 WHERE retention_until < CURRENT_DATE;
```

## 5. 긴급구조·법적 요청에 대한 제공(예외)

- **긴급구조(법 제29조)**: 급박한 위험에서 **긴급구조기관**(재난·응급) 요청 시 **동의 없이 제공 가능**. 제공 후 **처리기록(누가/왜/무엇을/언제)**를 남긴다.
- **사법·행정기관** 요청: 적법한 절차(영장/요청 공문 등) 확인, **최소한 범위**로 제공, 제공내역 기록.

**대응 표준운영절차(SOP) 개요**

```text
1) 수신: 기관 자격·근거 확인(영장/조문/기관 ID)
2) 최소제공: 좌표/시각/정확도 등 최소 단위만
3) 기록: 요청 사유/범위/담당자/제공 일시
4) 사후통지: 법령·약관에 정한 바에 따라 정보주체 통지(가능 범위)
5) 정기 점검: 분기별 샘플링 감사를 통해 절차 준수 확인
```

## 6. 정보주체 권리 처리(열람/정정/삭제/처리정지·철회/일시중지)

**요청 채널**(앱 내 “내 정보 → 위치정보”)을 마련하고, SLA를 설정(예: 10영업일 이내 처리). 약관·정책에 **담당부서/연락처** 기재. 대형사업자 약관의 **권리 항목**을 참고해 구성한다.

**권리 처리 핸들러(의사코드)**

```python
def handle_rights_request(user_id, request_type, payload):
    if request_type == "ACCESS":
        return export_user_location_logs(user_id, period=payload["period"])
    if request_type == "ERASE":
        # 확인자료 제외: 본문(개인위치정보) 즉시 파기
        erase_personal_location(user_id)
        return "ERASED"
    if request_type == "WITHDRAWAL":
        revoke_all_consents(user_id, scope="location")
        stop_collecting_location(user_id)
        return "WITHDRAWN"
    if request_type == "SUSPEND":
        set_collection_pause(user_id, until=payload.get("until"))
        return "PAUSED"
```

## 7. 기술·관리 보호조치(보안 설계 안)

### 7.1 최소수집·가명처리

- **좌표 정밀도 축소**(예: 6~7 자리 geohash) 및 **시간 해상도 스냅(분 단위)**
- **가명키(사용자 내부ID ↔ 외부 토큰)** 분리 저장

```python
# 가명처리 예: 좌표를 7자리 geohash로 축약하여 정밀 추적 위험을 줄임
import geohash
def pseudonymize(lat, lon):
    return geohash.encode(lat, lon, precision=7)
```

### 7.2 암호화

- **전송 구간**: TLS 1.2+ 강제, HSTS
- **저장 구간**: 좌표·정밀 식별키에 대해 **AES-256-GCM** 등 대칭키 암호화, 키는 **KMS/HSM** 보관, 접근 **RBAC/MFA** 적용

```bash
# 예: Linux at-rest 암호화로 위치 DB 디스크 보호(LUKS)
cryptsetup luksFormat /dev/nvme1n1
cryptsetup open /dev/nvme1n1 locdb_crypt
mkfs.xfs /dev/mapper/locdb_crypt
```

### 7.3 접근통제·감사

- **업무역할 분리**: 데이터 조회는 *보안감사 계정*으로만, 개발·운영은 마스킹 데이터 사용
- **감사로그**: 위치테이블 조회 시 **레코드 단위 감사**(who/when/why)
- **확인자료**·**시스템접근기록** **≥ 6개월** 보존 정책 구성(보안장비·SIEM 연계)

```sql
-- 행 단위 감사 트리거(예시)
CREATE OR REPLACE FUNCTION audit_access()
RETURNS trigger AS $$
BEGIN
  INSERT INTO audit_log(user_id, action, table_name, row_key, reason, ts)
  VALUES (current_user, TG_OP, TG_TABLE_NAME, NEW.id, current_setting('app.reason', true), now());
  RETURN NEW;
END; $$ LANGUAGE plpgsql;
```

### 7.4 국외 이전·제3자 제공

- **PIPA(개인정보보호법) 국외이전 체계** 준수(정보·동의·보호조치), 위치정보법 **동의 항목**과 **약관 공개**를 정합성 있게 맞춘다.

## 8. 개발 체크리스트(엔드투엔드)

- [ ] **부분동의(유보권)** UX 설계

**프론트/앱**
- [ ] OS 권한 분리(iOS “WhenInUse/Always”, Android “Approximate/Precise”)
- [ ] **수집 on/off 토글**, **일시중지 스케줄러** 제공

```kotlin
// Android 12+: 근사/정밀 권한 구분
val permissions = arrayOf(

  Manifest.permission.ACCESS_FINE_LOCATION
```

**백엔드/데이터**
- [ ] **확인자료 로깅**(수집/이용/제공 이벤트+주체+시각+서비스) ≥ 6개월 보관

- [ ] **가명화/정밀도 축소** 파이프라인 적용
- [ ] **긴급구조/사법요청 SOP** 및 **제공내역 기록** 테이블

**보안/운영**
- [ ] 키관리(KMS/HSM), DB 암호화, 전송구간 TLS, HSTS
- [ ] 접근권한 RBAC/MFA, 행단위 감사, **접근기록 6개월** 이상 보존
- [ ] **분기별 자체 점검**: 동의서 문구/로그·파기 잡음/권리요청 SLA

## 9. 위치기반서비스 약관/정책 템플릿(축약 예시)

> **주의**: 아래는 구조 예시이다. 각 항목은 서비스 특성·법령에 맞게 수정하라.

```text
제1조(목적) 본 약관은 회사가 제공하는 위치기반서비스의 이용에 필요한 사항을 규정한다.

제2조(수집·이용 목적) 근거리 매칭, 경로안내, 방문 인증 등.

제3조(보유기간) 개인위치정보는 목적 달성 즉시 파기. 단, 정보주체 별도 동의 시 최대 1년 보유.

제4조(제공) 제3자 제공 시 제공받는 자·목적·항목을 사전 고지하며 매회 즉시 통보.

제5조(권리) 정보주체(및 법정대리인)는 열람·정정·삭제·처리정지·철회·일시중지를 요구할 수 있다.

제6조(확인자료 보관) 수집·이용·제공 사실 확인자료는 6개월 이상 보관(법 제16조제2항).

제7조(긴급구조) 긴급구조기관 요청 시 법령에 따라 제공할 수 있다.

제8조(보호조치) 전송구간 암호화, 저장암호화, 접근통제, 감사기록.
```

참고: 주요 사업자 약관의 **6개월 확인자료 보관**·권리 항목은 일관되게 반영됨.

## 10. 로그·확인자료 스키마(예시)

```sql
CREATE TABLE location_fact_log (
  id BIGSERIAL PRIMARY KEY,
  subject_id BIGINT NOT NULL,   -- 가명키
  event ENUM('COLLECT','USE','PROVIDE') NOT NULL,
  purpose TEXT NOT NULL,
  third_party TEXT,             -- PROVIDE 시
  lat_enc BYTEA, lon_enc BYTEA, -- 암호화 저장
  accuracy INT,
  event_time TIMESTAMP NOT NULL,
  ip TEXT,
  ua TEXT
);

CREATE INDEX ON location_fact_log (event_time);
```

운영 정책: **로그≥6개월**, **좌표는 필요 시 가명/마스킹** 후 보존.

## 11. 민감 시나리오 대응

### 11.1 출퇴근 경로 리포트(리스크)

- **기본 즉시 파기 원칙**과 충돌 소지 → **별도 동의**·**최대 1년 상한** 내 기능화
- **k-익명성** 개념 적용(동일 시간대 k명 이상 겹치는 구간만 집계 공개)

$$
k\text{-anonymity: }\forall r\in R,\ |\{x\in D: Q(x)=Q(r)\}|\ge k
$$

### 11.2 국외 클라우드 사용

- 국외 이전 고지·동의, **역외 접근 통제**(IP allowlist/VPN), **현지 법규 충돌 검토**(데이터 현지화 요구 여부).

## 12. 실무 FAQ

**Q1. “6개월”은 모든 위치 데이터에 적용?**
A. 아니다. **확인자료**는 ≥6개월. **개인위치정보 본문**은 **즉시 파기**가 원칙, **별도 동의 시 최대 1년** 보유 가능.

**Q2. 아동 서비스는?**
A. **만14세 미만 법정대리인 동의**와 대리인의 **권리 행사** 절차를 마련.

**Q3. 동의 없이 제공 가능한 경우?**
A. **긴급구조** 등 법률이 정한 경우. 이후 **제공기록**을 남기고 사후 조치.
