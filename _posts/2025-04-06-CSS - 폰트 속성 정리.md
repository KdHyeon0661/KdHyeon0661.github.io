# 폰트 관련 CSS 속성 정리

## 0. 한눈에 보는 지도

| 범주 | 핵심 속성 | 요약 |
|---|---|---|
| 글꼴 지정 | `font-family`, `@font-face` | 폰트 스택·fallback·웹폰트 로딩 |
| 크기 | `font-size`, `clamp()` | rem 기반, 유동 타이포(Fluid Type) |
| 굵기 | `font-weight` | 100–900, 가변 폰트 범위/세밀 조정 |
| 줄 간격 | `line-height` | 단위 없는 배수(본문 1.5–1.8) |
| 자간 | `letter-spacing` | 제목에 소량 양수, 본문 0에 가깝게 |
| 기울임 | `font-style` | `normal|italic|oblique` |
| 폰트 기능 | `font-kerning`, `font-feature-settings`, `font-variant-*` | 합자, 숫자, 스몰캡, 구분선 |
| 가변 폰트 | `font-variation-settings`, `font-stretch` | 가중치/폭/옵틱스 조절 |
| 로딩 | `font-display`, `unicode-range` | FOIT/FOUT 제어, 서브셋 분리 |
| 접근성/성능 | 시스템 스택, 대비/줄길이 | 60–75자, rem·clamp, 프리로드 |

---

## 1. `font-family` — 글꼴 지정(스택 전략 포함)

```css
/* 권장: 시스템 + 지역 폰트 + 일반 계열 fallback */
:root {
  --system-ui: system-ui, -apple-system, "Segoe UI", Roboto,
               "Noto Sans", "Malgun Gothic", "Apple SD Gothic Neo",
               "Helvetica Neue", Arial, sans-serif;
}
body { font-family: var(--system-ui); }

/* 특정 브랜드 폰트 우선 → 실패 시 시스템 스택으로 */
.brand {
  font-family: "Pretendard", "Inter", var(--system-ui);
}
```

- **스택**은 항상 **다국어**를 고려합니다(한·중·일, 라틴, 숫자 정렬).
- 마지막엔 `serif | sans-serif | monospace` 같은 **generic family**로 안전망을 둡니다.
- 코드/터미널 UI는 `monospace`(동일 폭) 사용:

```css
code, pre { font-family: ui-monospace, SFMono-Regular, Menlo, Consolas, "Liberation Mono", "Courier New", monospace; }
```

## 2. `font-size` — 크기(상대 단위·유동 타이포·미디어 쿼리 최소화)

### 2.1 rem 기반(권장)
```css
html { font-size: 16px; }    /* 사용자 확대 고려 시 기본값 가정 */
body { font-size: 1rem; }    /* = 16px */
h1 { font-size: 2rem; }      /* = 32px */
small { font-size: .875rem; }/* = 14px */
```

### 2.2 `clamp()`로 유동 타이포(Fluid Typography)
```css
/* 최소 1rem, 뷰포트 따라 2vw 성장, 최대 1.25rem */
body { font-size: clamp(1rem, 2vw, 1.25rem); }

/* 헤딩 스케일 – 화면 크기에 부드럽게 반응 */
h1 { font-size: clamp(2rem, 4vw + 1rem, 3.5rem); }
h2 { font-size: clamp(1.5rem, 3vw + 1rem, 2.5rem); }
```

- 수학적 직관: 뷰포트 폭 \(V\)에서 유동 크기 \(S(V)\)를
  $$ S(V)=\min\{\max(S_{\min},\ a\cdot V+b),\ S_{\max}\} $$
  형태로 제한합니다(`clamp(min, preferred, max)`에 대응).

### 2.3 em vs rem
- **`rem`**: 루트 기준(일관된 스케일) → **레이아웃/여백/폰트**에 권장
- **`em`**: 부모의 폰트 크기 기준(내부 비례) → **컴포넌트 내부 비율** 조정 시

## 3. `font-weight` — 굵기(100–900 & 가변 폰트)

```css
/* 키워드와 수치 */
strong { font-weight: bold; }     /* 보통 700 */
em     { font-weight: 400; }      /* 기본값 */

/* 가변 폰트 범위 지정 예 */
@font-face {
  font-family: "InterVar";
  src: url("/fonts/Inter-Variable.woff2") format("woff2-variations");
  font-weight: 100 900; /* 가중치 연속 범위 */
  font-style: normal;
  font-display: swap;
}
.title { font-family: "InterVar", var(--system-ui); font-weight: 650; }
```

- 정적 폰트는 400/700만 지원할 수 있습니다(숫자 임의 지정해도 **폰트가 없는 웨이트는 보간되지 않을 수 있음**).
- **가변 폰트**는 100–900 범위에서 **연속적 조정**이 가능(디자인 정밀도↑, 파일 관리↓).

---

## 4. `line-height` — 줄 간격(리듬 유지)

```css
/* 단위 없는 배수(권장) */
body { line-height: 1.6; }         /* 본문 1.5–1.8 */
h1,h2,h3 { line-height: 1.25; }    /* 제목 1.2–1.4 */
```

- 단위(예: px)를 쓰면 **하위요소 상속 시 깨질 수 있음**.
- 문단 폭은 **60–75자**(영문 기준, 한글도 55–70자 권장)에서 가장 읽기 좋습니다.

---

## 5. `letter-spacing` — 글자 간격(자간)

```css
/* 제목/버튼에서 가독성·균형 강화 */
.hero-title { letter-spacing: .02em; }
.cta        { letter-spacing: .04em; text-transform: uppercase; }
```

- **한글 본문** 자간은 보통 0에 가깝게.
- **대문자/고딕 제목**에는 소량 양수(0.02–0.06em)로 안정감.

---

## 6. `font-style` — 기울임

```css
cite   { font-style: italic; }
.normal{ font-style: normal; }
.slant { font-style: oblique 10deg; } /* 기울임 각도 지정 가능(지원 시) */
```

- `italic`은 폰트가 제공하는 진짜 이탤릭, `oblique`는 **기하학적 기울임**.
- **`font-synthesis: style weight;`**로 브라우저의 **합성 기울임/굵기** 허용 제어 가능.

```css
/* 합성 억제: 폰트가 제공하지 않으면 기울임/굵기 합성하지 않음 */
.no-synth { font-synthesis: none; }
```

## 7. 폰트 기능(마이크로타이포) — 커닝/합자/숫자/스몰캡/옵틱스

```css
/* 커닝: 글자 사이 미세 조정 활성화 */
body { font-kerning: normal; }

/* 표준/컨텍스트 합자 */
.liga { font-feature-settings: "liga" 1, "clig" 1; }

/* 숫자: 등폭(표 정렬), 라이닝/올드스타일 */
.nums {
  font-variant-numeric: tabular-nums lining-nums;
  /* Oldstyle 예: oldstyle-nums; */
}

/* 스몰캡 */
.smallcaps { font-variant-caps: small-caps; }

/* 광학 크기(가변 폰트 지원 시 자동) */
.optical { font-optical-sizing: auto; }
```

- 표의 숫자 정렬은 **`tabular-nums`** 필수.
- 합자(`fi`, `fl`)는 가독성·미려함 ↑ (폰트/언어 특성 고려).

## 8. 가변 폰트(Variable Fonts) — 폭/가중치/맞춤 축

```css
@font-face {
  font-family: "MyVF";
  src: url("/fonts/MyVF.woff2") format("woff2-variations");
  font-weight: 200 900;
  font-stretch: 75% 125%;     /* 폭 가변 */
  font-style: normal;
  font-display: swap;
}
.heading {
  font-family: "MyVF", var(--system-ui);
  font-variation-settings: "wght" 680, "wdth" 92; /* 세밀 튜닝 */
}
```

- 일반적으로 사용되는 축: `"wght"`(굵기), `"wdth"`(폭), `"opsz"`(광학 크기).
- **장점**: 파일 수 감소(경량 배포), **세밀한 디자인 조정**.

## 9. `@font-face`와 웹폰트 로딩 전략 — FOIT/FOUT 제어, 서브셋

```css
@font-face {
  font-family: "NotoSansKR-Sub";
  src: url("/fonts/NotoSansKR-Sub.woff2") format("woff2");
  font-display: swap;          /* swap|fallback|optional */
  unicode-range: U+AC00-D7A3;  /* 한글 범위만(예시) → 필요 문자만 로드 */
}
```

- `font-display`:
  - **swap**: 즉시 대체 폰트(FOUT) → 나중에 웹폰트 적용(보통 권장)
  - **fallback/optional**: 느린 네트워크에서 **아예 로드 생략** 가능(경량 전략)
- `unicode-range`로 **라틴/숫자/한글 서브셋 분리** → 최초 페인트 빠르게.
- 중요 텍스트 폰트는 `<link rel="preload" as="font" type="font/woff2" crossorigin>`으로 **사전 로딩**.

**실전 패턴(라틴 먼저, 한글 지연):**
```html
<link rel="preload" as="font" href="/fonts/Inter-Var.woff2" type="font/woff2" crossorigin>
```

## 10. 텍스트 정렬/공백·줄바꿈·말줄임(핵심만 상기)

- 정렬: `text-align: left|center|right|justify`
- 공백/줄바꿈: `white-space`, `overflow-wrap:anywhere`, `word-break:keep-all`(한글), `hyphens:auto`(영문)
- 말줄임:
```css
.ellipsis-1 { white-space:nowrap; overflow:hidden; text-overflow:ellipsis; }
.clamp-3 {
  display: -webkit-box; -webkit-line-clamp: 3; -webkit-box-orient: vertical;
  overflow: hidden;
}
```

## 11. 실전 구성 레시피

### 11.1 기본 타이포 시스템
```css
:root {
  /* 유동 바디 타입: 16px ~ 20px */
  --body-size: clamp(1rem, 1rem + 0.6vw, 1.25rem);
  /* 헤딩 스케일 */
  --h1: clamp(2rem, 4vw + 1rem, 3.5rem);
  --h2: clamp(1.5rem, 3vw + 1rem, 2.5rem);
  --h3: clamp(1.25rem, 2vw + 1rem, 1.8rem);
  --lh-body: 1.65;
  --lh-heading: 1.25;
}

html { font-size: 16px; }
body {
  font-family: var(--system-ui);
  font-size: var(--body-size);
  line-height: var(--lh-body);
  color: #0f172a;
}

h1,h2,h3 {
  font-family: "InterVar", var(--system-ui);
  line-height: var(--lh-heading);
  margin: 0 0 .5em;
}
h1 { font-size: var(--h1); font-weight: 750; }
h2 { font-size: var(--h2); font-weight: 700; }
h3 { font-size: var(--h3); font-weight: 650; }

p { margin: 0 0 1em; max-width: 70ch; }
```

### 11.2 숫자 표 정렬(재무/대시보드)
```css
.table td.num {
  font-family: "InterVar", var(--system-ui);
  font-variant-numeric: tabular-nums;
  text-align: right;
  white-space: nowrap;
}
```

### 11.3 버튼/네비 타이틀(대문자+자간)
```css
.nav a {
  font-weight: 600;
  text-transform: uppercase;
  letter-spacing: .06em;
}
```

### 11.4 본문(한/영 혼용 안정)
```css
.article p {
  line-height: 1.7;
  word-break: keep-all;           /* 한글 어절 보존 */
  overflow-wrap: anywhere;        /* 긴 URL 보호 */
  hyphens: auto;                  /* 영문 하이픈 */
  text-align: justify;            /* 필요 시 */
  text-justify: inter-word;
}
```

## 12. 성능·접근성 체크리스트

- **rem + clamp**: 기기/사용자 확대에 대응하는 탄력적 타이포.
- **대비/줄 길이**: 본문 대비 AA(4.5:1) 이상, **60–75자** 권장.
- **웹폰트 최소화**: 필요 글립만, 서브셋·`unicode-range`·가변 폰트로 파일 수 절감.
- **로딩 전략**: 핵심 폰트만 preload, 나머지는 `font-display: swap|optional`.
- **숫자 정렬**: `tabular-nums`로 표 일관성.
- **합성 억제**: `font-synthesis: none;`로 잘못된 합성 이탤릭/굵기 방지(브랜드 타이포).
- **언어 메타**: `<html lang="ko">` 정확히, 보조 언어 구간은 `lang` 속성으로 구분.
- **영역·역방향**: RTL(아랍어 등) 페이지는 `dir="rtl"` + 폰트 스택 점검.

## 14. 마무리 요약

- **`font-family`**: 다국어·숫자·OS별 렌더링 고려한 **스택 설계**가 핵심.
- **`font-size`**: **rem + clamp**로 유동형 타이포 구축, 미디어쿼리 의존도↓.
- **`font-weight`**: 가변 폰트로 파일 관리 단순화 + 정밀 굵기 조정.
- **`line-height`**: 단위 없는 배수로 리듬 유지(본문 1.5–1.8).
- **미세 타이포**: 커닝/합자/숫자/스몰캡으로 디테일 완성.
- **웹폰트 로딩**: `font-display`, `unicode-range`, preload로 **FOIT/FOUT 제어**.
- **접근성/성능**: 대비, 줄 길이, 로딩 비용, 확대 대응을 항상 함께 본다.
