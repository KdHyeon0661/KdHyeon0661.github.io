# C# 다형성과 타입 검사 (is/as/typeof)

## 소개: 유연한 설계를 위한 객체 지향 핵심 개념

객체 지향 프로그래밍에서 **다형성(Polymorphism)** 은 코드의 유연성과 재사용성을 높이는 핵심 개념입니다. C#에서 다형성은 주로 **상속**과 **가상 메서드**를 통해 구현됩니다. 또한 런타임에 객체의 실제 타입을 확인하고 안전하게 변환하기 위한 **타입 검사** 연산자(`is`, `as`, `typeof`)가 제공됩니다. 이러한 도구들을 적절히 활용하면 더 견고하고 유지보수하기 쉬운 코드를 작성할 수 있습니다.

---

## 1. 상속을 통한 다형성

### 기본 개념

다형성은 동일한 메서드 호출이 객체의 실제 타입에 따라 다른 동작을 하도록 만드는 기능입니다. C#에서는 기본 클래스의 `virtual` 메서드를 파생 클래스에서 `override`하여 다형성을 구현합니다.

```csharp
public class Animal
{
    public string Name { get; set; } = "";
    
    public virtual void MakeSound()
    {
        Console.WriteLine("동물이 소리를 냅니다.");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("멍멍!");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("야옹~");
    }
}

public class Cow : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("음메~");
    }
}
```

### 다형성의 활용

기본 클래스 타입의 참조로 파생 클래스의 인스턴스를 다룰 수 있습니다. 이를 통해 코드의 유연성이 크게 향상됩니다.

```csharp
class Program
{
    static void Main()
    {
        // 다양한 파생 객체를 기본 클래스 배열에 저장
        Animal[] animals = new Animal[]
        {
            new Dog { Name = "바둑이" },
            new Cat { Name = "나비" },
            new Cow { Name = "얼룩이" },
            new Dog { Name = "초코" }
        };
        
        // 각 동물의 소리를 다형적으로 호출
        foreach (Animal animal in animals)
        {
            Console.Write($"{animal.Name}: ");
            animal.MakeSound(); // 실제 타입의 메서드가 호출됨
        }
        
        // 메서드 매개변수로 활용
        MakeAnimalSound(new Dog());
        MakeAnimalSound(new Cat());
    }
    
    static void MakeAnimalSound(Animal animal)
    {
        // 어떤 동물이든 받아서 소리를 냄
        animal.MakeSound();
    }
}
```

### 추상 클래스와 다형성

추상 클래스를 사용하면 더 강력한 계층 구조를 만들 수 있습니다. 추상 메서드는 파생 클래스에서 반드시 구현해야 합니다.

```csharp
public abstract class Shape
{
    public abstract double GetArea();
    public abstract double GetPerimeter();
    
    public void DisplayInfo()
    {
        Console.WriteLine($"넓이: {GetArea():F2}, 둘레: {GetPerimeter():F2}");
    }
}

public class Circle : Shape
{
    public double Radius { get; set; }
    
    public override double GetArea() => Math.PI * Radius * Radius;
    public override double GetPerimeter() => 2 * Math.PI * Radius;
}

public class Rectangle : Shape
{
    public double Width { get; set; }
    public double Height { get; set; }
    
    public override double GetArea() => Width * Height;
    public override double GetPerimeter() => 2 * (Width + Height);
}
```

## 2. 타입 검사와 변환: `is`, `as`, `typeof`, 패턴 매칭

### `is` 연산자: 타입 확인

`is` 연산자는 객체가 특정 타입과 호환되는지 확인합니다. 상속 관계를 모두 고려하며, C# 7.0부터는 패턴 매칭과 결합하여 더욱 강력해졌습니다.

```csharp
public class Vehicle
{
    public void Move() => Console.WriteLine("이동 중...");
}

public class Car : Vehicle
{
    public void Honk() => Console.WriteLine("빵빵!");
}

public class Bicycle : Vehicle
{
    public void RingBell() => Console.WriteLine("따르릉~");
}

class Program
{
    static void Main()
    {
        object[] things = { new Car(), new Bicycle(), "문자열", 123, null };
        
        foreach (var thing in things)
        {
            // is 연산자를 사용한 타입 확인 및 패턴 매칭
            if (thing is Car car)
            {
                Console.Write("자동차: ");
                car.Honk();
            }
            else if (thing is Bicycle bike)
            {
                Console.Write("자전거: ");
                bike.RingBell();
            }
            else if (thing is string text)
            {
                Console.WriteLine($"문자열: {text}");
            }
            else if (thing is int number)
            {
                Console.WriteLine($"정수: {number}");
            }
            else if (thing == null)
            {
                Console.WriteLine("null 값");
            }
            else
            {
                Console.WriteLine("알 수 없는 타입");
            }
        }
    }
}
```

### 고급 패턴 매칭

C# 7.0 이상에서는 `is`와 `switch`를 활용한 다양한 패턴 매칭을 지원합니다.

```csharp
public static string DescribeObject(object obj)
{
    return obj switch
    {
        null => "null",
        int i when i < 0 => $"음수 {i}",
        int i when i == 0 => "0",
        int i when i > 0 => $"양수 {i}",
        string s when s.Length == 0 => "빈 문자열",
        string s when s.Length > 0 => $"문자열 길이: {s.Length}",
        Car c => $"자동차 (브랜드: {c.Brand ?? "알 수 없음"})",
        Bicycle b => "자전거",
        _ => $"기타 객체: {obj.GetType().Name}"
    };
}

public class Car
{
    public string? Brand { get; set; }
}
```

### `as` 연산자: 안전한 타입 변환

`as` 연산자는 타입 변환을 시도하고, 실패하면 `null`을 반환합니다. 예외를 발생시키지 않으므로 참조 타입에 안전하게 사용할 수 있습니다.

```csharp
object obj1 = new Car { Brand = "Hyundai" };
object obj2 = new Bicycle();
object obj3 = "테스트";
object obj4 = null;

// as 연산자 사용
Car? car1 = obj1 as Car;     // 성공: Car 객체
Car? car2 = obj2 as Car;     // 실패: null
Car? car3 = obj3 as Car;     // 실패: null
Car? car4 = obj4 as Car;     // 성공: null (obj4가 null이므로)

if (car1 != null)
{
    Console.WriteLine($"자동차 브랜드: {car1.Brand}");
}

if (car2 == null)
{
    Console.WriteLine("obj2는 Car로 변환할 수 없습니다.");
}

// 값 타입에는 as를 직접 사용할 수 없음 (nullable로는 가능)
object obj5 = 123;
int? number = obj5 as int?; // 가능: 123
double? d = obj5 as double?; // 실패: null (int를 double로 변환 불가)
```

### `typeof` 연산자와 `GetType()` 메서드

`typeof`와 `GetType()`은 모두 타입 정보를 얻는 데 사용되지만, 중요한 차이가 있습니다.

```csharp
public class Animal { }
public class Dog : Animal { }

class Program
{
    static void Main()
    {
        Animal animal = new Dog();
        
        // typeof: 컴파일 타임에 알려진 타입 정보
        Type animalType = typeof(Animal);
        Type dogType = typeof(Dog);
        Console.WriteLine($"typeof(Animal): {animalType.Name}"); // Animal
        Console.WriteLine($"typeof(Dog): {dogType.Name}");       // Dog
        
        // GetType(): 런타임 객체의 실제 타입 정보
        Type runtimeType = animal.GetType();
        Console.WriteLine($"animal.GetType(): {runtimeType.Name}"); // Dog
        
        // 비교
        Console.WriteLine($"animal is Animal: {animal is Animal}");           // True
        Console.WriteLine($"animal is Dog: {animal is Dog}");                 // True
        Console.WriteLine($"animal.GetType() == typeof(Animal): {runtimeType == animalType}"); // False
        Console.WriteLine($"animal.GetType() == typeof(Dog): {runtimeType == dogType}");       // True
    }
}
```

### 🔍 타입 검사는 필요할 때만: 다형성 우선

`is`와 `as`는 강력한 도구지만, **과도한 타입 검사는 코드 냄새(code smell)** 일 수 있습니다. 객체 지향 설계의 핵심은 다형성을 활용해 타입에 따라 분기하는 대신, 가상 메서드를 통해 객체가 스스로 행동하게 하는 것입니다.

**권장 원칙**: 가능하면 타입 검사를 피하고, 가상 메서드나 추상 메서드를 통한 호출을 우선 고려하세요. 타입 검사가 필요한 경우는 주로 외부 시스템과의 연동이나, 이미 설계된 레거시 코드를 다룰 때로 제한하는 것이 좋습니다.

```csharp
// 나쁜 예: 타입 검사로 분기
public void MakeNoise(Animal animal)
{
    if (animal is Dog) Console.WriteLine("멍멍");
    else if (animal is Cat) Console.WriteLine("야옹");
    else if (animal is Cow) Console.WriteLine("음메");
}

// 좋은 예: 다형성 활용
public void MakeNoise(Animal animal)
{
    animal.MakeSound(); // 각 클래스에서 override된 메서드 호출
}
```

## 3. 다형성의 실제 활용 사례

### 전략 패턴과 결합

다형성은 전략 패턴(Strategy Pattern)을 구현하는 데 이상적입니다.

```csharp
// 결제 전략 인터페이스 (실제로는 추상 클래스나 인터페이스 가능)
public abstract class PaymentStrategy
{
    public abstract void ProcessPayment(decimal amount);
}

// 신용카드 결제
public class CreditCardPayment : PaymentStrategy
{
    private string _cardNumber;
    
    public CreditCardPayment(string cardNumber)
    {
        _cardNumber = cardNumber;
    }
    
    public override void ProcessPayment(decimal amount)
    {
        Console.WriteLine($"{amount:C}를 신용카드({_cardNumber})로 결제합니다.");
    }
}

// 페이팔 결제
public class PayPalPayment : PaymentStrategy
{
    private string _email;
    
    public PayPalPayment(string email)
    {
        _email = email;
    }
    
    public override void ProcessPayment(decimal amount)
    {
        Console.WriteLine($"{amount:C}를 PayPal({_email})로 결제합니다.");
    }
}

// 결제 처리기
public class PaymentProcessor
{
    private PaymentStrategy _strategy;
    
    public PaymentProcessor(PaymentStrategy strategy)
    {
        _strategy = strategy;
    }
    
    public void SetStrategy(PaymentStrategy strategy)
    {
        _strategy = strategy;
    }
    
    public void Pay(decimal amount)
    {
        _strategy.ProcessPayment(amount);
    }
}

// 사용 예
class Program
{
    static void Main()
    {
        var processor = new PaymentProcessor(new CreditCardPayment("1234-5678"));
        processor.Pay(50000);
        
        processor.SetStrategy(new PayPalPayment("user@example.com"));
        processor.Pay(30000);
    }
}
```

### 템플릿 메서드 패턴

추상 클래스를 사용하여 알고리즘의 뼈대를 정의하고 일부 단계를 하위 클래스에서 구현하도록 할 수 있습니다.

{% raw %}
```csharp
public abstract class DataExporter
{
    // 템플릿 메서드: 전체 알고리즘 정의
    public void ExportData(string data)
    {
        var processedData = ProcessData(data);
        var formattedData = FormatData(processedData);
        SaveData(formattedData);
    }
    
    // 공통 구현
    private string ProcessData(string data)
    {
        return data.Trim().ToUpperInvariant();
    }
    
    // 하위 클래스에서 구현해야 하는 부분
    protected abstract string FormatData(string data);
    protected abstract void SaveData(string data);
}

public class CsvExporter : DataExporter
{
    protected override string FormatData(string data)
    {
        return $"CSV: {data.Replace(",", ";")}";
    }
    
    protected override void SaveData(string data)
    {
        Console.WriteLine($"CSV 파일 저장: {data}");
    }
}

public class JsonExporter : DataExporter
{
    protected override string FormatData(string data)
    {
        return $"{{\"data\": \"{data}\"}}";
    }
    
    protected override void SaveData(string data)
    {
        Console.WriteLine($"JSON 파일 저장: {data}");
    }
}
```

## 4. 주의사항: 값 타입과 참조 타입

다형성은 참조 타입(클래스)에서 자연스럽게 동작하지만, 값 타입(struct)은 상속을 지원하지 않으므로 다형성을 직접 적용할 수 없습니다. 값 타입을 인터페이스 타입으로 변환하면 **박싱**이 발생할 수 있다는 점을 기억해야 합니다. (이 글에서는 인터페이스를 다루지 않으므로 값 타입의 다형성은 고려하지 않아도 됩니다.)

## 5. 타입 검사 연산자 성능 고려사항

- `is`와 `as`는 일반적으로 빠르지만, 런타임 타입 확인이 필요하므로 약간의 오버헤드가 있습니다.
- 잦은 타입 검사가 필요한 경우 설계를 재고하거나, 더 효율적인 방법(예: 가상 메서드)을 고려하세요.
- `typeof`는 컴파일 타임에 결정되므로 런타임 비용이 거의 없습니다.
- `GetType()`은 가상 메서드 호출이므로 약간의 오버헤드가 있지만, 대부분의 경우 무시할 수준입니다.

## 결론: 적절한 추상화와 타입 안전성의 균형

다형성과 타입 검사는 C#에서 유연하고 안전한 코드를 작성하는 데 필수적인 도구입니다. 이 개념들을 효과적으로 사용하기 위한 핵심 원칙을 정리해 보겠습니다:

1. **다형성을 우선하라**: 가능하면 타입 검사보다 가상 메서드나 추상 메서드를 통한 다형성을 활용하세요. 이는 코드의 결합도를 낮추고 확장성을 높입니다.

2. **타입 검사는 필요한 경우에만**: `is`, `as`, `typeof`는 강력하지만, 과도하게 사용하면 코드가 복잡해지고 객체 지향 설계 원칙을 위반할 수 있습니다. 정말 필요한 상황에서만 사용하세요.

3. **패턴 매칭을 적극 활용하라**: C# 7.0 이상의 패턴 매칭 기능을 사용하면 타입 검사와 변환을 더 간결하고 안전하게 표현할 수 있습니다.

4. **상속 계층 설계에 주의**: 깊은 상속 계층보다는 얕고 명확한 계층을 유지하세요. 필요하다면 컴포지션을 고려하는 것도 좋은 방법입니다.

5. **런타임 타입과 컴파일 타임 타입 구분**: `typeof`는 컴파일 타임 타입, `GetType()`은 런타임 실제 타입을 반환합니다. 이 차이를 이해하고 적절히 사용하세요.

다형성과 타입 검사는 객체 지향 프로그래밍의 핵심 개념입니다. 이들을 올바르게 이해하고 활용하면 더 유연하고 유지보수하기 쉬운 소프트웨어를 구축할 수 있습니다. 다음번에 클래스 계층을 설계할 때, "어떻게 하면 다형성을 최대한 활용할 수 있을까?"를 먼저 고민해 보세요. 이 간단한 습관이 코드의 질을 획기적으로 높여줄 것입니다.
