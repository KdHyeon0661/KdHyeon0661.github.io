# HTML 테이블 완전 정복

## 1. 테이블 핵심 구조: thead / tbody / tfoot / caption / colgroup

기본 구조 예시는 유지하되, **열 그룹(colgroup)** 과 **푸터(tfoot)** 를 추가하면 스타일/요약행에 유리합니다.

```html
<table class="tbl">
  <caption>학생 성적표</caption>

  <!-- 열 그룹: 열 폭/스타일을 열 단위로 제어 -->
  <colgroup>
    <col style="width: 16ch;">
    <col span="2"> <!-- 과목/점수 두 열을 묶어 같은 스타일 적용 가능 -->
    <col style="width: 10ch;">
  </colgroup>

  <thead>
    <tr>
      <th scope="col">이름</th>
      <th scope="col">과목</th>
      <th scope="col">점수</th>
      <th scope="col">비고</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">홍길동</th>
      <td>수학</td>
      <td class="num">95</td>
      <td>우수</td>
    </tr>
    <tr>
      <th scope="row">김영희</th>
      <td>영어</td>
      <td class="num">88</td>
      <td>양호</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row" colspan="2">평균</th>
      <td class="num">91.5</td>
      <td>-</td>
    </tr>
  </tfoot>
</table>
```

### 구성요소 요약

| 태그/요소      | 역할/이점 |
|---|---|
| `<caption>`     | 표 제목. 스크린리더에 **가장 먼저** 읽혀 문맥 제공. |
| `<colgroup>/<col>` | 열 단위 폭·배경·보더 지정. **반복 스타일**을 줄임. |
| `<thead>`       | 헤더 행 그룹. 스크롤·인쇄에서 머리글 반복에 유리. |
| `<tbody>`       | 데이터 행 그룹. 여러 개의 tbody로 구역 나눌 수 있음. |
| `<tfoot>`       | 합계/평균 등 요약 행. 인쇄 시 각 페이지 하단 반복 가능(브라우저 지원 범위 참고). |
| `<th>`          | 헤더 셀. `scope`/`headers`로 데이터-헤더 관계를 명확화. |
| `<td>`          | 일반 데이터 셀. |

## 2. 병합(rowspan/colspan) — 정확한 구조가 먼저

### 2.1 열 병합 `colspan`

```html
<table class="tbl simple">
  <tr>
    <th colspan="2" scope="colgroup">사용자 정보</th>
  </tr>
  <tr>
    <th scope="row">이름</th>
    <td>홍길동</td>
  </tr>
  <tr>
    <th scope="row">이메일</th>
    <td>hong@example.com</td>
  </tr>
</table>
```

- `scope="colgroup"`: **여러 열**에 걸친 헤더라는 의미 전달.

### 2.2 행 병합 `rowspan`

```html
<table class="tbl simple">
  <tr>
    <th rowspan="2" scope="rowgroup">이름</th>
    <td>홍길동</td>
  </tr>
  <tr>
    <td>김영희</td>
  </tr>
</table>
```

- `scope="rowgroup"`: **여러 행**에 걸친 헤더.

> 병합이 많아지면 접근성이 급격히 어려워집니다. 복잡한 표는 **행/열 그룹**을 명확히 하고, 필요 시 `headers/id` 방식(§5.3)을 사용하세요.

## 3. 캡션과 표 설명

```html
<table aria-describedby="desc-sales">
  <caption>월별 매출 요약</caption>
  ...
</table>
<p id="desc-sales" class="visually-hidden">
  이 표는 2025년 1~6월 각 카테고리의 매출을 요약합니다. 단위: 백만원.
</p>
```

- `aria-describedby`로 **긴 설명**을 별도로 연결.
- 시각적으로 숨기려면 **시각적 숨김 클래스**를 사용합니다.

```css
.visually-hidden {
  position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px;
  overflow: hidden; clip: rect(0 0 0 0); white-space: nowrap; border: 0;
}
```

## 4. CSS 기본 스타일 — 가독성/정렬/숫자폰트

```css
.tbl {
  width: 100%;
  border-collapse: collapse;          /* 경계선 겹치기 */
  table-layout: auto;                 /* 내용 기반 열폭(기본). 많으면 fixed 고려 */
  margin: 1rem 0;
  font: 14px/1.6 system-ui, sans-serif;
}

.tbl :is(th, td) {
  border: 1px solid #e5e7eb;
  padding: 10px 12px;
  text-align: left;
  vertical-align: top;
}

.tbl thead th {
  background: #f8fafc;
  font-weight: 700;
}

.tbl tfoot th,
.tbl tfoot td {
  background: #f1f5f9;
  font-weight: 600;
}

.tbl.striped tbody tr:nth-of-type(odd) {
  background: #fafafa;
}

/* 숫자 정렬/숫자 모노스페이스 */
.tbl .num {
  text-align: right;
  font-variant-numeric: tabular-nums; /* 자리폭 고정 숫자 */
}
```

- `border-collapse: collapse` vs `separate`: **collapse**가 보통 깔끔. 테두리 간소화 용이.
- 숫자열에는 `text-align: right` + `tabular-nums`로 **자릿수 정렬**.

## 5. 접근성(a11y): scope / headers-id / 정렬 힌트

### 5.1 `scope` 속성

```html
<th scope="col">과목</th>
<th scope="row">홍길동</th>
<th scope="colgroup" colspan="2">2025 상반기</th>
```

- `col`, `row`, `colgroup`, `rowgroup` 4종을 상황에 맞게 사용.
- 스크린리더가 **셀→헤더 매핑**을 자동으로 파악.

### 5.2 요약·단위·약어

```html
<th scope="col" abbr="국">국어</th>
<caption>
  학생 시험 결과 <span class="unit">(점수: 0~100)</span>
</caption>
```

- `abbr`는 길고 복잡한 헤더의 **짧은 대체 텍스트**.

### 5.3 아주 복잡한 표: `headers`/`id`

```html
<table class="tbl complex">
  <caption>부서별/분기별 지표</caption>
  <thead>
    <tr>
      <th id="h-dept" scope="col">부서</th>
      <th id="h-q1" scope="col">1분기</th>
      <th id="h-q2" scope="col">2분기</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th id="r-dev" scope="row">개발</th>
      <td headers="r-dev h-q1">120</td>
      <td headers="r-dev h-q2">135</td>
    </tr>
  </tbody>
</table>
```

- 각 데이터 셀에 `headers="행헤더ID 열헤더ID"`로 **정밀 연결**.

## 6. 반응형 테이블 전략 4가지

표는 가로로 넓어지기 쉬우므로, 상황에 맞는 전략을 고릅니다.

### 6.1 스크롤 박스(가장 간단/보편)

```css
.responsive {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}
```

```html
<div class="responsive">
  <table class="tbl striped">...</table>
</div>
```

- 내용은 유지하고 **가로 스크롤**만 허용. 테이블 본연의 구조를 훼손하지 않음.

### 6.2 열 숨김(모바일에서 부차적 열 감춤)

```css
@media (max-width: 640px) {
  .tbl .col--optional { display: none; }
}
```

```html
<colgroup>
  <col>
  <col class="col--optional">
  <col>
</colgroup>
```

- 정보 손실이 생길 수 있으므로 **필수/선택 열**을 명확히 구분.

### 6.3 카드 스택(열→행 변환; 데이터 라벨 표시)

```css
@media (max-width: 640px) {
  .stacked thead { display: none; }
  .stacked tbody tr { display: grid; grid-template-columns: 10ch 1fr; border: 1px solid #e5e7eb; margin-bottom: .75rem; }
  .stacked td::before {
    content: attr(data-label);
    font-weight: 600; padding-right: .75rem;
  }
  .stacked :is(td, th[scope="row"]) { display: contents; } /* 라벨-값만 보이게 */
}
```

```html
<table class="tbl stacked">
  <thead>
    <tr><th>이름</th><th>국어</th><th>영어</th><th>수학</th></tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">홍길동</th>
      <td data-label="국어" class="num">90</td>
      <td data-label="영어" class="num">85</td>
      <td data-label="수학" class="num">95</td>
    </tr>
  </tbody>
</table>
```

- 작은 화면에서 **각 셀에 라벨을 붙여 카드처럼** 보여줌.

### 6.4 행/열 고정(Sticky)

```css
.sticky thead th {
  position: sticky; top: 0;
  background: #f8fafc; z-index: 2;
}

.sticky tbody th[scope="row"] {
  position: sticky; left: 0;
  background: #fff; z-index: 1; /* 헤더보다 낮게 */
  white-space: nowrap;
}
```

```html
<div class="responsive">
  <table class="tbl sticky">
    ...
  </table>
</div>
```

- 헤더행/행헤더를 **고정**해 대용량 스크롤에서 가독성을 높임.

## 7. 테이블 레이아웃/성능: `table-layout: fixed`

```css
.tbl.fixed {
  table-layout: fixed;  /* 첫 행/colgroup 기준으로 열폭 즉시 계산 → 페인트 안정 */
  width: 100%;
}

.tbl.fixed td, .tbl.fixed th {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

- 콘텐츠 길이에 관계 없이 **빠르게 레이아웃**. 긴 텍스트는 말줄임으로 처리.

## 8. 숫자/통계 표기와 수식

### 8.1 숫자 서식/정렬

```css
.num { text-align: right; font-variant-numeric: tabular-nums; }
.num.negative { color: #b91c1c; } /* 음수 강조 */
```

### 8.2 평균/합계 등 계산식 표현 (문서 설명용)

표 평균의 기본식:

$$
\text{평균} = \frac{\sum_{i=1}^{n} x_i}{n}
$$

문서 본문에서 수식을 명시하고, 표의 `tfoot`에 결과를 기입합니다.

## 9. 인쇄/다크모드/RTL 최적화

```css
@media print {
  .tbl { font-size: 12px; }
  thead { display: table-header-group; } /* 각 페이지마다 머리글 반복 */
  tfoot { display: table-footer-group; } /* 각 페이지마다 바닥글 반복 */
  a::after { content: " (" attr(href) ")"; font-size: .9em; color: #6b7280; }
}

@media (prefers-color-scheme: dark) {
  .tbl :is(th, td) { border-color: #334155; }
  .tbl thead th    { background: #0b1220; color: #e2e8f0; }
  .tbl tfoot :is(th, td) { background: #0f172a; color: #e5e7eb; }
}

html[dir="rtl"] .num { text-align: left; } /* RTL 환경에서 숫자 정렬 고려 */
```

- 인쇄 시 thead/tfoot 반복은 브라우저마다 차이가 있으나 **권장 설정**입니다.

## 10. 데이터 정렬/필터(프런트엔드 미니 스크립트)

### 10.1 컬럼 정렬(오름/내림 토글)

```html
<table class="tbl sortable" id="gradeTable" aria-describedby="sort-help">
  <caption>학생 시험 결과</caption>
  <p id="sort-help" class="visually-hidden">컬럼 헤더를 클릭하면 정렬됩니다.</p>
  <thead>
    <tr>
      <th data-type="text">이름</th>
      <th data-type="number">국어</th>
      <th data-type="number">영어</th>
      <th data-type="number">수학</th>
      <th data-type="number">평균</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>홍길동</td><td class="num">90</td><td class="num">85</td><td class="num">95</td><td class="num">90</td></tr>
    <tr><td>김영희</td><td class="num">88</td><td class="num">92</td><td class="num">89</td><td class="num">89.7</td></tr>
  </tbody>
</table>

<script>
(function(){
  const tbl = document.getElementById('gradeTable');
  const getVal = (td, type) => {
    const t = td.textContent.trim();
    if (type === 'number') return parseFloat(t.replace(/[^\d.-]/g,'')) || 0;
    return t.toLowerCase();
  };
  tbl.querySelectorAll('thead th').forEach((th, idx) => {
    let asc = true;
    th.tabIndex = 0;
    th.addEventListener('click', sort);
    th.addEventListener('keydown', e => { if (e.key === 'Enter' || e.key === ' ') sort(e); });
    function sort() {
      const type = th.dataset.type || 'text';
      const rows = Array.from(tbl.tBodies[0].rows);
      rows.sort((a,b)=>{
        const A = getVal(a.cells[idx], type);
        const B = getVal(b.cells[idx], type);
        return asc ? (A > B) - (A < B) : (B > A) - (B < A);
      });
      asc = !asc;
      rows.forEach(r => tbl.tBodies[0].appendChild(r));
      tbl.querySelectorAll('thead th').forEach(h => h.setAttribute('aria-sort','none'));
      th.setAttribute('aria-sort', asc ? 'ascending' : 'descending');
    }
  });
})();
</script>
```

- 헤더를 버튼처럼 사용 가능하도록 `tabIndex`/키보드 지원.
- `aria-sort`로 정렬 상태를 스크린리더에 전달.

## 11. 테이블과 Grid/Div의 구분

| 상황 | 테이블 사용 | CSS Grid/Div 사용 |
|---|---|---|
| **데이터**(정형/행열) | 예 | 아니오 |
| **레이아웃**(페이지 구성) | 아니오 | 예 |
| 행/열 헤더/요약/정렬·필터 | 예 | 보통 아니오(직접 구현해야 함) |

**의미가 테이블**이면 꼭 `<table>`을 사용하세요. 스크린리더·복사/붙여넣기·CSV 변환 등 여러 이점이 있습니다.

## 12. 흔한 실수와 교정

1) 시맨틱 없이 `<div>`로 표를 그림 → **접근성/키보드/복붙 불리**  
→ `<table>` 구조로 전환.

2) 복잡한 병합에서 `scope` 생략 → 읽기 순서/헤더 매핑 실패  
→ `scope`를 충실히, 필요 시 `headers/id`.

3) 숫자열 좌/중앙 정렬 → 읽기 어려움  
→ `.num { text-align: right; font-variant-numeric: tabular-nums; }`.

4) 모바일에서 표 깨짐 → **스크롤 박스** 또는 **스택형 카드** 전략 도입.

5) 레이아웃 시프트(CLS) → `table-layout: fixed` + 고정 폭/말줄임으로 안정화.

## 14. 체크리스트

- [ ] 표는 **데이터**일 때만 사용(레이아웃은 Grid/Flex)
- [ ] `caption`으로 표 목적 제공, 필요 시 `aria-describedby`로 보조 설명
- [ ] 헤더에는 `scope`(col/row/colgroup/rowgroup) 정확히 지정
- [ ] 복잡 표는 `headers/id`로 정밀 연결
- [ ] 숫자열은 우측 정렬 + `tabular-nums`
- [ ] 모바일은 **가로 스크롤** 또는 **스택형 카드**
- [ ] sticky 헤더/행헤더로 대용량 스크롤 가독성 확보
- [ ] 인쇄 매체 고려: `thead/tfoot` 반복, 링크 URL 출력
- [ ] 성능: `table-layout: fixed` + 말줄임, 큰 테이블은 페이징/가상 스크롤 고려
- [ ] 다크모드/RTL 대응

## 15. 요약

- **의미가 표**라면 반드시 `<table>`: 시맨틱·a11y·복붙/CSV·정렬/요약에 유리.
- 구조는 `caption`/`colgroup`/`thead`/`tbody`/`tfoot`을 적극 활용.
- 병합은 최소화하고, 필요 시 `scope`/`headers`로 **관계**를 명확히.
- 반응형은 **스크롤·스택·열숨김·sticky** 중 목적에 맞게 선택.
- 숫자/인쇄/다크모드/성능까지 챙기면 **전문적인 데이터 테이블** 완성.
