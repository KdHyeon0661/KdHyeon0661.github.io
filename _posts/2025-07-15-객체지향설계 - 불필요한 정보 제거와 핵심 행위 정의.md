# ✂️ 불필요한 정보 제거와 핵심 행위 정의

---

## 1. 개념 개요

**"불필요한 정보 제거(Irrelevant Information Removal)"**와 **"핵심 행위 정의(Core Behavior Definition)"**는 추상화(Abstraction)의 두 기둥으로, **객체의 설계와 표현을 단순화하여 효과적인 모델링을 가능하게 하는 방법론**입니다.

이 두 개념은 함께 작용하여 다음을 가능하게 합니다:

- 복잡한 시스템을 이해하기 쉽게 만들고,
- 설계에서 중요한 것에만 집중하게 하며,
- 코드 유지보수와 확장을 용이하게 합니다.

---

## 2. 불필요한 정보 제거 (Information Hiding by Abstraction)

### ✅ 의미

객체가 가지고 있는 **모든 데이터와 동작 중에서 외부에서 사용할 필요가 없는 것들은 감추고 제거**하는 것을 의미합니다. 

이는 단순히 `private`로 숨긴다는 의미가 아니라, **모델링 단계에서부터 신중하게 속성과 기능을 선택**한다는 뜻입니다.

### ✅ 예시: 커피 자판기

- 사용자가 자판기를 사용할 때는 버튼을 누르고 돈을 넣는 행위만 필요합니다.
- 내부적으로 어떤 회로가 작동하고, 물이 몇 도로 끓고 있는지는 **사용자가 알 필요도, 알 수 있는 방법도 없습니다.**

```java
class CoffeeMachine {
    public void makeCoffee() {
        heatWater();
        grindBeans();
        brew();
        serve();
    }

    private void heatWater() { ... }
    private void grindBeans() { ... }
    private void brew() { ... }
    private void serve() { ... }
}
```

- `makeCoffee()`만 외부에 공개
- 내부 단계들은 숨겨진 "불필요한 정보"

### ✅ 목적

- 사용자는 복잡한 내부 과정을 몰라도 객체를 사용할 수 있음
- 변경이 발생해도 외부에 영향을 주지 않음 (은닉성)
- 인터페이스 간결화 → **이해도 향상**

---

## 3. 핵심 행위 정의 (Defining Core Behavior)

### ✅ 의미

**객체가 본질적으로 수행해야 하는 주요 동작만을 정의하는 것**입니다. 객체가 어떤 "역할"을 맡는지를 파악하고, 그 역할에 적절한 최소한의 기능만을 제공하는 것이 중요합니다.

### ✅ 예시: 차량 객체

- 자동차가 해야 할 핵심 행위는 `운전`, `정지`, `회전` 등입니다.
- `오디오 음량 조절`, `에어컨 설정` 등은 보조 행위이며, **핵심이 아닐 수 있습니다.**

```java
interface Vehicle {
    void accelerate();
    void brake();
    void turnLeft();
    void turnRight();
}
```

- 최소한의 동작만 정의한 **핵심 인터페이스**
- 구체 차량(Bike, Car, Truck)은 이 핵심 동작을 구현

### ✅ 목적

- 중요한 동작만 드러내어 사용성 향상
- 지나치게 많은 책임을 객체에 부여하지 않음
- 객체의 **단일 책임 원칙(SRP)** 유지에 도움

---

## 4. 실전 활용: ATM 예제

### 추상화 설계

```java
abstract class ATM {
    public final void processTransaction() {
        authenticateUser();
        selectTransactionType();
        performTransaction();
        printReceipt();
    }

    protected abstract void authenticateUser();
    protected abstract void performTransaction();
    
    private void selectTransactionType() { ... }
    private void printReceipt() { ... }
}
```

- `authenticateUser()`와 `performTransaction()`만 하위 클래스에서 정의하도록 함
- 나머지는 불필요하거나 고정된 흐름이므로 추상화 수준에서 숨김
- **핵심 행위만 정의**, 나머지는 제거 또는 은닉

---

## 5. 효과 및 장점 요약

| 항목 | 효과 |
|------|------|
| 🎯 관심 집중 | 사용자 또는 설계자가 중요한 동작에 집중할 수 있음 |
| 🔐 정보 은닉 | 내부 변경이 외부에 영향을 주지 않도록 구조화 가능 |
| 📦 인터페이스 최소화 | 외부 노출 범위를 줄여 오류 가능성도 감소 |
| 🔁 유지보수 용이 | 중요한 기능에 집중한 코드 구조는 더 쉽게 수정 가능 |
| 🚀 확장성 향상 | 핵심만 정의하므로 다른 부분과의 결합도 낮음 |

---

## 6. 정리

- **불필요한 정보 제거**는 복잡한 세부사항을 숨겨 **인터페이스를 단순화**합니다.
- **핵심 행위 정의**는 객체가 반드시 수행해야 할 행위에만 집중하여 **책임을 명확히** 합니다.
- 이 두 원칙은 **추상화 수준을 높이고**, 소프트웨어 시스템의 **품질, 유지보수성, 확장성**을 모두 향상시킵니다.

이 두 가지는 객체지향 설계의 진정한 **추상화 능력**을 발휘하기 위한 실천적인 방법입니다.
