# HTML 리스트(List)

## 리스트 한눈에 보기(요약)

| 쓰임 | 태그 | 핵심 |
|---|---|---|
| 순서 없는 목록 | `<ul>` + `<li>` | 불릿(•) 기반. 순서 중요 X |
| 순서 있는 목록 | `<ol>` + `<li>` | 번호/문자/로마숫자. 절차/순서 중요 O |
| 설명 목록 | `<dl>` + `<dt>` + `<dd>` | 용어–설명, 키–값, FAQ 등 |

---

## 순서 없는 리스트(UL)

### 기본

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

### 기본 스타일 바꾸기

```css
ul { list-style-type: square; } /* disc | circle | square | none */
```

### 커스텀 마커: `::marker` (권장)

```html
<ul class="skills">
  <li>Semantic HTML</li>
  <li>Accessible Components</li>
  <li>CSS Architecture</li>
</ul>
```
```css
.skills { padding-left: 1.25rem; }
.skills li::marker {
  content: "▹ ";           /* 이모지/문자/유니코드 가능 */
  color: oklch(0.7 0.16 230);
  font-weight: 700;
}
```
> `::marker`는 실제 **목록 마커**를 바꾸므로 스크린리더/선택 영역과의 **일관성**이 좋습니다.
> `::before`로 불릿을 만드는 방법은 **의미적 마커가 아니므로** 가능한 `::marker`를 쓰세요.

### 이미지 마커(대안)

```css
.features { list-style-image: url('/icons/dot.svg'); }
```
> `list-style-image`는 **크기/정렬 제어가 제한**적입니다. 최근에는 `::marker`나 `::before + background`를 혼합합니다.

## 순서 있는 리스트(OL)

### 기본

```html
<ol>
  <li>회원가입</li>
  <li>이메일 인증</li>
  <li>로그인</li>
</ol>
```

### 속성: `type`, `start`, `reversed`, `value`

```html
<ol type="A" start="3">
  <li>Alpha</li>
  <li>Bravo</li>
</ol>

<ol reversed>
  <li>Step 3</li>
  <li>Step 2</li>
  <li>Step 1</li>
</ol>

<ol>
  <li value="10">열 번째부터</li>
  <li>열한 번째</li>
</ol>
```

- `type`: `1`(숫자), `A`/`a`(알파벳), `I`/`i`(로마숫자)
- `start`: 시작 번호 지정
- `reversed`: 역순 카운트
- `value`: 해당 항목의 번호를 **강제로** 지정(부분 재번호 매기기)

### 커스텀 번호: CSS 카운터(고급)

```html
<ol class="steps">
  <li>요구사항 수집</li>
  <li>설계</li>
  <li>구현</li>
  <li>테스트</li>
</ol>
```
```css
.steps {
  list-style: none;
  counter-reset: step;         /* 카운터 초기화 */
  padding-left: 0;
}
.steps > li {
  counter-increment: step;     /* 항목마다 증가 */
  margin: .5rem 0 .5rem 3.2rem;
  position: relative;
}
.steps > li::before {
  content: counter(step);
  position: absolute; left: 0; top: 0;
  width: 2.2rem; height: 2.2rem; display: grid; place-items: center;
  border-radius: 50%;
  color: white;
  background: oklch(0.68 0.13 230);
  font-weight: 700;
}
```
> `counter()`/`counters()` 조합으로 **중첩 번호(1.1, 1.2 …)** 도 구현 가능합니다.

## — 용어·정의·키·값

### 기본

```html
<dl>
  <dt>HTML</dt>
  <dd>웹 페이지의 구조를 만드는 언어</dd>
  <dt>CSS</dt>
  <dd>웹 페이지를 꾸미는 스타일 언어</dd>
</dl>
```

### 하나의 용어에 다중 정의 / 하나의 정의에 다중 용어

```html
<dl>
  <dt>API</dt>
  <dd>응용 프로그램 인터페이스</dd>
  <dd>소프트웨어 간 상호작용 명세</dd>

  <dt>색 공간</dt><dt>Color Space</dt>
  <dd>색을 수학적으로 표현하는 체계</dd>
</dl>
```

### 2열 그리드로 “정의표”처럼 보이게

```html
<dl class="grid">
  <dt>이름</dt><dd>김도현</dd>
  <dt>역할</dt><dd>프론트엔드 엔지니어</dd>
  <dt>관심사</dt><dd>시맨틱·접근성·성능</dd>
</dl>
```
```css
.grid {
  display: grid;
  grid-template-columns: 10ch 1fr; /* 왼쪽 용어 폭 고정 */
  gap: .5rem 1rem;
}
.grid dt { font-weight: 700; }
.grid dd { margin: 0; }
```

### FAQ에 DL 쓰기(접기/펼치기)

```html
<dl class="faq">
  <dt><button aria-expanded="false" aria-controls="a1">Q. HTML과 XHTML 차이는?</button></dt>
  <dd id="a1" hidden>HTML5는 SGML 비종속, XHTML은 XML 문법 준수 등…</dd>
</dl>
<script>
document.querySelectorAll('.faq button').forEach(btn=>{
  btn.addEventListener('click', ()=>{
    const id = btn.getAttribute('aria-controls');
    const dd = document.getElementById(id);
    const ex = btn.getAttribute('aria-expanded') === 'true';
    btn.setAttribute('aria-expanded', String(!ex));
    dd.hidden = ex;
  });
});
</script>
```

## 네비게이션 메뉴와 리스트

### 시맨틱 구조

```html
<nav aria-label="주 메뉴">
  <ul class="menu">
    <li><a href="/">홈</a></li>
    <li><a href="/about">소개</a></li>
    <li>
      <a href="/docs">문서</a>
      <ul>
        <li><a href="/docs/getting-started">시작하기</a></li>
        <li><a href="/docs/api">API</a></li>
      </ul>
    </li>
  </ul>
</nav>
```

### 수평 메뉴/드롭다운(간단 버전)

```css
.menu { display: flex; gap: 1rem; list-style: none; padding: 0; margin: 0; }
.menu > li { position: relative; }
.menu > li > ul {
  position: absolute; inset: auto auto auto 0;
  list-style: none; padding: .5rem 0; margin: 0;
  background: white; border: 1px solid #e5e7eb; border-radius: .5rem;
  display: none; min-width: 12rem;
}
.menu > li:hover > ul, .menu > li:focus-within > ul { display: block; }
.menu a { display: block; padding: .5rem .75rem; text-decoration: none; color: #111827; }
.menu a:hover { background: #f3f4f6; }
```
> **접근성**: 키보드 포커스로도 열리도록 `:focus-within` 사용.
> 메뉴 위젯 역할을 엄격히 적용하려면 WAI-ARIA 메뉴 패턴을 따르는 **고급 스크립트**가 필요합니다.

## 리스트 마커를 디자이너처럼 다루기

### `::marker`로 색/내용 바꾸기

```css
ul.checklist li::marker { content: "✓ "; color: #16a34a; font-weight: 700; }
```

### 목록 들여쓰기 제어: `list-style-position`

```css
li { list-style-position: outside; }  /* 기본: 마커 밖 */
.note li { list-style-position: inside; } /* 마커를 텍스트 블록 안으로 */
```

### 긴 줄바꿈과 마커 정렬

```css
li {
  text-wrap: pretty;     /* 줄바꿈 품질 개선(지원 브라우저에서) */
  hanging-punctuation: first; /* 마커/구두점 시각 정렬(실험적) */
}
```

## CSS 카운터로 **다단계 번호** 만들기

```html
<ol class="chapters">
  <li>소개
    <ol>
      <li>배경</li>
      <li>목표</li>
    </ol>
  </li>
  <li>설계</li>
</ol>
```
```css
.chapters { counter-reset: c1; list-style: none; padding-left: 0; }
.chapters > li { counter-increment: c1; margin-left: 2rem; position: relative; }
.chapters > li::before {
  content: counter(c1) ". ";
  position: absolute; left: -2rem;
  font-weight: 700; color: #111827;
}

/* 2단계 */
.chapters ol { counter-reset: c2; list-style: none; padding-left: 0; }
.chapters ol > li { counter-increment: c2; margin-left: 2rem; position: relative; }
.chapters ol > li::before {
  content: counter(c1) "." counter(c2) " ";
  position: absolute; left: -2rem; color: #374151;
}
```

## 접근성(a11y)과 리스트

- `<li>`는 반드시 `<ul>`/`<ol>` **안에서만** 사용합니다.
- 순서가 중요한 절차는 `<ol>`, 단순 묶음은 `<ul>`이 더 적합합니다.
- 네비게이션은 `<nav>` 안에 `<ul>`을 넣으면 스크린리더가 **메뉴 영역**으로 파악합니다.
- 커스텀 마커는 가능하면 `::marker`를 써서 **의미 있는 마커**를 유지하세요.
- `<dl>`은 **질문–답변**, **키–값** 표현에 자연스러우며, 표(`<table>`)가 아닙니다.
- 링크/상태를 **색상만**으로 구분하지 말고 **밑줄/아이콘/텍스트**를 함께 사용하세요.

## 흔한 실수와 교정

1) `div`로 목록을 만들고 CSS로 불릿 흉내 → **스크린리더 목록 인식 실패**
→ **반드시** `<ul>/<ol>/<li>` 사용.

2) 메뉴를 `<div>`만으로 구성 → **탐색 구조 미노출**
→ `<nav><ul>…</ul></nav>`로 시맨틱 추가.

3) `<li>`를 단독 사용 → **유효하지 않은 마크업**
→ `<ul>` 또는 `<ol>` 안에 배치.

4) 불릿을 `::before`로만 생성 → **실제 마커가 아님**
→ `::marker` 사용(가능 브라우저), 불가 시 접근성 대안을 고려.

5) `<dl>`을 표처럼 쓰면서 `dt/dd`를 1:1로 강제
→ **한 용어 다중 정의/다중 용어 하나의 정의**도 허용(스펙상 OK).

## 성능과 유지보수 팁

- 리스트 스타일은 **클래스 기반**으로 재사용(UL/OL에 직접 인라인 스타일 지양).
- 레이아웃은 **`display: grid|flex`** 와 조합해 **카드/갤러리** 등으로 확장.
- 아이콘 불릿은 **SVG 스프라이트** 또는 **웹폰트**보다 **`::marker` + 이모지/SVG**가 단순·경량.
- 디자인 시스템에서는 **의미 기반 유틸리티 클래스**(예: `.list--check`, `.list--timeline`)로 표준화.

## 체크리스트

- [ ] 적절한 리스트 유형 선택(순서 유무/설명 구조)
- [ ] `<li>`는 반드시 `<ul>/<ol>` 내부
- [ ] 네비게이션은 `<nav>` + `<ul>` 시맨틱
- [ ] 커스텀 마커는 `::marker` 우선(불가 시 대안)
- [ ] 번호 커스텀은 **CSS 카운터**로 구현
- [ ] `<dl>`은 다중 `dt`/`dd` 패턴도 지원
- [ ] 키보드/스크린리더 상호작용 점검(드롭다운/메뉴)
- [ ] 프린트/다크모드/모바일까지 시나리오 테스트
