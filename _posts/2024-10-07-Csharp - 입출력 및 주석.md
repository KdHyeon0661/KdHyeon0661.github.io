# C# 기본 입출력과 주석 활용 완벽 가이드

프로그래밍을 시작할 때 가장 먼저 마주하는 화면은 검은 콘솔 창입니다. 사용자와 프로그램이 소통하기 위해서는 데이터를 입력받고 결과를 출력하는 과정이 필수적입니다. 또한, 작성한 코드를 나 자신 혹은 동료가 쉽게 이해할 수 있도록 기록을 남기는 주석 작성법은 개발자의 기본 소양입니다.

이 글에서는 C# 환경에서 콘솔 입출력을 다루는 방법부터 안전한 데이터 변환, 그리고 실무에서 활용하는 다양한 주석 작성 기법까지 상세히 알아봅니다.

## 콘솔 출력 (Output)

C#에서 화면에 데이터를 출력할 때는 `Console` 클래스가 제공하는 메서드를 사용합니다. 가장 기본이 되는 두 가지 메서드는 출력 후 줄바꿈을 할 것인지 여부에 따라 나뉩니다.

```csharp
// 줄바꿈 없이 이어서 출력합니다.
Console.Write("Hello "); 
// 문자열을 출력한 뒤 커서를 다음 줄로 이동시킵니다.
Console.WriteLine("World!"); 

```

단순히 고정된 문자열만 출력하는 경우는 드뭅니다. 변수에 저장된 값을 문장 속에 자연스럽게 섞어서 출력해야 할 때, C#은 **문자열 보간법(String Interpolation)**이라는 매우 편리한 기능을 제공합니다. 문자열 앞에 `$` 기호를 붙이고, 변수나 수식을 중괄호 `{}` 안에 직접 입력하면 됩니다.

```csharp
string name = "김철수";
int age = 25;

// 문자열 보간법을 사용한 직관적인 출력
Console.WriteLine($"이름: {name}, 나이: {age}");

// 기존의 서식 문자열 방식 (인덱스 사용)
Console.WriteLine("이름: {0}, 나이: {1}", name, age);

```

이러한 보간법 내부에서는 단순한 변수 출력뿐만 아니라 데이터의 형식을 지정할 수도 있습니다. 예를 들어, 통화 기호를 붙이거나 소수점 자리를 제한할 때 서식 지정자를 활용합니다.

```csharp
double price = 12345.6789;

// C는 통화(Currency) 형식을 의미하며 시스템 설정에 맞는 기호가 붙습니다.
Console.WriteLine($"가격: {price:C}");  
// N2는 숫자(Number) 형식이며 소수점 둘째 자리까지 반올림하여 표현합니다.
Console.WriteLine($"가격: {price:N2}"); 

DateTime now = DateTime.Now;
// 날짜와 시간도 원하는 형태로 자유롭게 포맷팅할 수 있습니다.
Console.WriteLine($"현재: {now:yyyy-MM-dd HH:mm:ss}");

```

## 콘솔 입력 (Input)과 데이터 변환

사용자로부터 데이터를 입력받을 때는 `Console.ReadLine()` 메서드를 사용합니다. 이 메서드는 사용자가 키보드로 값을 입력하고 Enter 키를 누를 때까지 대기하다가, 입력된 전체 라인을 문자열로 반환합니다.

```csharp
Console.Write("이름을 입력하세요: ");
string? inputName = Console.ReadLine();
Console.WriteLine($"안녕하세요, {inputName}님!");

```

여기서 주의할 점이 있습니다. `Console.ReadLine()`은 사용자가 숫자를 입력하더라도 항상 문자열(String) 형태로 데이터를 반환합니다. 만약 입력받은 값을 이용해 

$$X + Y = Z$$

 와 같은 수학적 사칙연산을 수행해야 한다면, 반드시 문자열을 숫자형(Integer, Double 등)으로 변환하는 과정이 필요합니다.

초보자들이 흔히 사용하는 `int.Parse()` 메서드는 사용자가 숫자가 아닌 문자를 입력했을 때 프로그램이 곧바로 튕겨버리는 예외(Exception)를 발생시킵니다. 실무에서는 이러한 런타임 에러를 방지하기 위해 **안전한 형변환 메서드인 `TryParse()`**를 사용합니다.

`TryParse`는 변환 시도를 하고 성공 여부를 불리언(Boolean) 값으로 반환하며, 성공했을 경우에만 `out` 키워드를 통해 결과를 변수에 담아줍니다.

```csharp
Console.Write("나이를 입력하세요: ");
string? ageInput = Console.ReadLine();

// 입력값이 안전하게 숫자로 변환될 수 있는지 검사합니다.
if (int.TryParse(ageInput, out int age))
{
    Console.WriteLine($"입력된 나이: {age}세");
}
else
{
    Console.WriteLine("경고: 올바른 숫자를 입력해주세요.");
}

```

## 프로그램의 시작점, Main 메서드

입출력 코드를 작성할 때, C# 버전에 따라 프로그램의 진입점을 작성하는 형태가 크게 다릅니다. 이는 코드를 읽고 쓸 때 혼동을 줄 수 있으므로 반드시 짚고 넘어가야 합니다.

### 레거시 방식: 명시적인 Main 메서드 (C# 8 이하)

과거의 C#이나 현재 유지보수 중인 구형 .NET Framework 프로젝트에서는 프로그램의 시작점인 `Main` 메서드가 명시적으로 존재해야 합니다. 모든 실행 코드는 반드시 클래스 내부의 정적 메서드 안에 위치해야 했습니다.

```csharp
using System;

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Hello, World!");
    }
}

```

### 최신 방식: Top-level Statements (C# 9 이상)

.NET 5 (C# 9.0)부터는 불필요한 보일러플레이트 코드를 줄이기 위해 **Top-level statements(최상위 문)** 기능이 도입되었습니다. 이제 별도의 클래스나 `Main` 메서드를 명시하지 않아도, 파일 최상단에 작성한 코드가 곧바로 프로그램의 진입점으로 인식됩니다.

```csharp
using System;

Console.WriteLine("Hello, World!");

```

컴파일러가 내부적으로 기존의 `Main` 메서드 구조를 자동으로 생성해주기 때문에, 학습을 진행하거나 간단한 콘솔 도구를 만들 때 코드를 훨씬 간결하게 유지할 수 있습니다.

## 주석(Comments) 작성 기법

주석은 컴파일러에 의해 무시되는 텍스트로, 코드의 의도나 부가적인 설명을 사람에게 전달하기 위해 작성합니다. 좋은 코드는 코드 자체로 읽히는 것이 가장 이상적이지만, 복잡한 비즈니스 로직이나 수식, 예외적인 처리의 경우 반드시 주석이 필요합니다.

### 기본 주석과 작업 태그

가장 널리 쓰이는 한 줄 주석 `//`과 여러 줄 주석 `/* */` 외에도, 실무에서는 특정 패턴의 키워드를 사용하여 작업 상태를 팀원들에게 공유합니다.

```csharp
// 한 줄 주석: 학생의 수학 시험 점수를 담는 변수입니다.
int mathScore = 100; 

/*
 * 여러 줄 주석
 * 여기에 함수가 수행하는 복잡한 수식의 원리나,
 * 특정 라이브러리를 사용한 이유 등을 상세히 적습니다.
 */

public int CalculatePhysicsScore(int mass, int acceleration)
{
    // TODO: 중력 가속도를 상수가 아닌 환경 변수에서 가져오도록 개선해야 함
    // FIXME: 질량이 음수로 들어오는 예외 처리가 누락됨
    // HACK: 시연을 위해 임시로 결과값에 10을 더함. 릴리스 전 반드시 수정할 것.

    int force = mass * acceleration;
    return force + 10;
}

```

대대부분의 현대적인 IDE(Visual Studio, VS Code 등)는 `TODO`나 `FIXME` 같은 특수 주석을 감지하여 작업 목록 창에 별도로 모아 보여주는 기능을 제공합니다.

### XML 문서 주석 (API 문서화)

클래스나 메서드 바로 위에 `///` (슬래시 세 개)를 입력하면 IDE가 자동으로 **XML 문서 주석** 템플릿을 생성해줍니다. 이 주석은 단순히 코드를 설명하는 것을 넘어, 다른 개발자가 해당 메서드를 호출하려고 할 때 툴팁(IntelliSense)으로 설명을 띄워주는 매우 강력한 기능을 합니다.

```csharp
/// <summary>
/// 두 숫자를 입력받아 덧셈을 수행합니다.
/// 수식: $$Result = a + b$$
/// </summary>
/// <param name="a">더해질 첫 번째 정수</param>
/// <param name="b">더해질 두 번째 정수</param>
/// <returns>두 정수의 합</returns>
public int Add(int a, int b)
{
    return a + b;
}

```

프로젝트 설정(`.csproj`) 파일에 `<GenerateDocumentationFile>true</GenerateDocumentationFile>` 구문을 추가하면, 프로젝트를 빌드할 때 이 XML 주석들이 모여 실제 API 문서 파일로 추출됩니다.

## 실전 응용: 안전한 콘솔 계산기 만들기

지금까지 배운 콘솔 출력, 안전한 입력 변환, 그리고 주석 기법을 모두 종합하여 간단하지만 견고한 콘솔 계산기 프로그램을 만들어 보겠습니다. 아래 코드는 C# 9 이상(Top-level statements)을 기준으로 작성되었습니다.

```csharp
using System;

// 프로그램 시작
Calculator calc = new Calculator();
calc.Run();

/// <summary>
/// 기본적인 사칙연산을 처리하는 계산기 클래스입니다.
/// </summary>
class Calculator
{
    /// <summary>
    /// 두 정수를 더합니다.
    /// </summary>
    public int Add(int a, int b)
    {
        return a + b;
    }

    /// <summary>
    /// 사용자 인터페이스를 담당하며 계산기를 실행합니다.
    /// </summary>
    public void Run()
    {
        Console.WriteLine("간단 계산기");
        Console.WriteLine("============");

        Console.Write("첫 번째 숫자를 입력하세요: ");
        string? input1 = Console.ReadLine();

        Console.Write("두 번째 숫자를 입력하세요: ");
        string? input2 = Console.ReadLine();

        // 입력받은 두 개의 문자열이 모두 안전하게 정수로 변환되는지 확인합니다.
        if (int.TryParse(input1, out int num1) && int.TryParse(input2, out int num2))
        {
            int result = Add(num1, num2);
            Console.WriteLine($"\n계산 결과: {num1} + {num2} = {result}");
        }
        else
        {
            // FIXME: 숫자가 아닌 값이 입력되었을 때 다시 입력을 요구하도록 루프(Loop) 구조로 개선 필요
            Console.WriteLine("\n[에러] 올바른 숫자를 입력해주세요. 프로그램이 종료됩니다.");
        }
    }
}

```

## 마무리

화면에 글자를 띄우고 사용자의 키보드 입력을 받는 것은 프로그래밍의 가장 원초적인 상호작용입니다. `TryParse()`를 통해 예외 상황을 방어하는 습관과, XML 주석을 통해 나의 코드에 친절한 설명서를 붙이는 습관은 여러분의 코드를 한 차원 더 높은 수준으로 끌어올려 줍니다.

기본기가 탄탄할수록 이후에 배우게 될 복잡한 객체지향 설계나 비동기 프로그래밍에서도 흔들리지 않을 수 있습니다. 오늘 작성해본 계산기 코드에 반복문을 추가해 보거나 다른 연산 기능을 확장해 보며 감각을 익혀보시길 바랍니다.
