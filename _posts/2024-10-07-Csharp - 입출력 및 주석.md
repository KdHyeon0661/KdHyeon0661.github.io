# C# 기본 입출력과 주석 활용

C#에서 콘솔 입출력과 주석 작성은 프로그래밍의 가장 기본이 되는 요소입니다.  
이 글에서는 실무에서도 바로 사용할 수 있는 예제를 통해 핵심 개념을 정리합니다.

참고: C# 버전에 따라 `Main` 메서드 작성 방식이 달라질 수 있습니다. 아래에서 버전별 차이를 함께 설명합니다.

---

## 콘솔 입출력

### 기본 출력

```csharp
// 기본적인 콘솔 출력
Console.Write("Hello ");          // 줄바꿈 없이 출력
Console.WriteLine("World!");      // 줄바꿈 포함 출력

// 문자열 보간법 (String Interpolation)
string name = "김철수";
int age = 25;
Console.WriteLine($"이름: {name}, 나이: {age}");

// 서식 문자열 사용
Console.WriteLine("이름: {0}, 나이: {1}", name, age);
````

### 다양한 형식의 출력

```csharp
// 숫자 서식화
double price = 12345.6789;
Console.WriteLine($"가격: {price:C}");   // 통화 형식
Console.WriteLine($"가격: {price:N2}");  // 숫자 형식 (소수점 2자리)

// 날짜 서식화
DateTime now = DateTime.Now;
Console.WriteLine($"현재: {now:yyyy-MM-dd}");
Console.WriteLine($"현재: {now:HH:mm:ss}");
```

## 콘솔 입력

### 기본 입력

```csharp
Console.Write("이름을 입력하세요: ");
string? inputName = Console.ReadLine();  // nullable string
Console.WriteLine($"안녕하세요, {inputName}님!");
```

### 안전한 숫자 입력 처리

```csharp
Console.Write("나이를 입력하세요: ");
string? ageInput = Console.ReadLine();

if (int.TryParse(ageInput, out int age))
{
    Console.WriteLine($"입력된 나이: {age}세");
}
else
{
    Console.WriteLine("올바른 숫자를 입력해주세요.");
}
```

`TryParse()`를 사용하면 예외 없이 안전하게 입력값을 검증할 수 있습니다.

## 여러 값 입력받기

```csharp
Console.WriteLine("학생 정보 입력");

Console.Write("이름: ");
string? studentName = Console.ReadLine();

Console.Write("학번: ");
string? studentId = Console.ReadLine();

Console.Write("학점: ");
if (double.TryParse(Console.ReadLine(), out double grade))
{
    Console.WriteLine("\n입력된 정보:");
    Console.WriteLine($"이름: {studentName}");
    Console.WriteLine($"학번: {studentId}");
    Console.WriteLine($"학점: {grade:F2}");
}
```

## 프로그램 시작점(Main 메서드) 차이

C#은 **버전에 따라 프로그램 시작 방식이 달라집니다.**

---

### C# 9 이상 (.NET 5 이상)

Top-level statements 기능이 도입되어 `Main` 메서드를 생략할 수 있습니다.

```csharp
Console.WriteLine("Hello, World!");
```

컴파일러가 내부적으로 `Main` 메서드를 자동 생성합니다.

이 방식은 학습용 코드나 간단한 콘솔 프로그램에서 많이 사용됩니다.

---

### C# 8 이하 또는 .NET Framework

이전 버전에서는 반드시 `Main` 메서드가 필요합니다.

```csharp
class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Hello, World!");
    }
}
```

실무에서는 여전히 레거시 프로젝트(.NET Framework 4.x 등)가 많기 때문에
전통적인 `Main` 구조를 이해하는 것이 중요합니다.

## XML 문서 주석 (API 문서화)

```csharp
/// <summary>
/// 학생 정보를 나타내는 클래스입니다.
/// </summary>
public class Student
{
    /// <summary>
    /// 학생의 이름
    /// </summary>
    public string Name { get; set; }

    /// <summary>
    /// 학생의 나이
    /// </summary>
    public int Age { get; set; }

    /// <summary>
    /// 학생 정보를 문자열로 반환합니다.
    /// </summary>
    public string GetInfo()
    {
        return $"이름: {Name}, 나이: {Age}";
    }
}
```

### XML 문서 파일 생성

`.csproj`에 아래 설정을 추가하면 XML 문서가 생성됩니다.

```xml
<PropertyGroup>
  <GenerateDocumentationFile>true</GenerateDocumentationFile>
</PropertyGroup>
```

## 실용 예제: 간단 계산기

### C# 9 이상

```csharp
Calculator calc = new Calculator();
calc.Run();
```

### C# 8 이하

```csharp
class Program
{
    static void Main(string[] args)
    {
        Calculator calc = new Calculator();
        calc.Run();
    }
}
```

```csharp
class Calculator
{
    public int Add(int a, int b)
    {
        return a + b;
    }

    public void Run()
    {
        Console.WriteLine("간단 계산기");
        Console.WriteLine("============");

        Console.Write("첫 번째 숫자: ");
        string? input1 = Console.ReadLine();

        Console.Write("두 번째 숫자: ");
        string? input2 = Console.ReadLine();

        if (int.TryParse(input1, out int num1) &&
            int.TryParse(input2, out int num2))
        {
            Console.WriteLine($"결과: {num1} + {num2} = {Add(num1, num2)}");
        }
        else
        {
            Console.WriteLine("올바른 숫자를 입력해주세요.");
        }
    }
}
```

## 핵심 요약

1. `Console.Write()` / `WriteLine()` → 출력
2. `Console.ReadLine()` → 입력
3. `$"문자열 보간법"` → 가독성 높은 출력
4. `TryParse()` → 안전한 입력 처리
5. `//`, `/* */` → 기본 주석
6. `///` → XML 문서 주석
7. C# 9 이상 → `Main` 생략 가능
8. C# 8 이하 → `Main` 반드시 필요

## 마무리

기본적인 입출력과 주석 작성 습관은
코드의 **가독성, 안정성, 유지보수성**을 크게 향상시킵니다.

버전에 따른 차이까지 이해하면
레거시 프로젝트와 최신 프로젝트 모두 대응할 수 있는 개발자가 될 수 있습니다.
