# VS Code에서 C# 개발 환경 설정하기 (입문 가이드)

과거에는 C# 개발이라고 하면 무겁고 방대한 Visual Studio IDE를 설치하는 것이 당연하게 여겨졌습니다. 하지만 이제는 가볍고 확장성이 뛰어난 Visual Studio Code(VS Code)만으로도 훌륭한 C# 개발 환경을 구축할 수 있습니다.

이 가이드는 Visual Studio Code에서 C# 개발을 처음 시작하는 분들을 위해 복잡한 테스트나 배포 설정은 과감히 생략하고, 코드를 작성하고 실행하며 디버깅하는 가장 핵심적인 기본 흐름에 집중하여 상세히 안내합니다.

---

## 필수 구성 요소 설치

C# 코드를 작성하고 실행하기 위해서는 편집기인 VS Code 외에도 코드를 해석하고 실행해 줄 런타임과 도구 모음이 필요합니다. 이를 위해 가장 먼저 설치해야 할 것이 바로 **.NET SDK**입니다.

* **.NET SDK 다운로드**: [공식 다운로드 페이지](https://dotnet.microsoft.com/ko-kr/download)에 접속하여 운영체제에 맞는 버전을 다운로드합니다.
* **주의사항**: 다운로드 페이지를 보면 'SDK'와 '런타임(Runtime)'이 나뉘어 있습니다. 개발자는 코드를 빌드하고 실행하기 위한 모든 도구가 포함된 **SDK**를 설치해야 합니다.

설치를 마쳤다면 터미널(Windows의 경우 명령 프롬프트나 PowerShell)을 열고 다음 명령어를 실행하여 정상적으로 설치되었는지 확인합니다.

```bash
dotnet --version

```

명령어를 입력했을 때 설치한 .NET의 버전 숫자가 화면에 출력된다면 개발을 위한 첫 번째 준비가 완벽하게 끝난 것입니다.

---

## VS Code 확장 프로그램 설치

VS Code 자체는 아주 가벼운 텍스트 에디터일 뿐이므로, C#의 문법을 이해하고 자동 완성 기능(IntelliSense)이나 디버깅 기능을 제공하려면 전용 확장 프로그램을 설치해야 합니다.

1. VS Code를 실행합니다.
2. 화면 왼쪽의 활동 모음에서 네모난 블록 모양의 **확장(Extensions)** 아이콘을 클릭하거나, 단축키 `Ctrl+Shift+X`를 누릅니다.
3. 검색창에 `C#`을 입력합니다.
4. Microsoft에서 공식적으로 제공하는 **C#** 확장 프로그램을 찾아 설치합니다.

추가로 Microsoft에서 제공하는 **C# Dev Kit**이라는 확장 프로그램을 함께 설치하는 것을 적극적으로 권장합니다. 이 확장을 설치하면 기존 Visual Studio처럼 솔루션 탐색기를 제공하여 여러 프로젝트를 관리하거나 파일들을 탐색하는 과정이 훨씬 편리해집니다.

---

## 첫 번째 프로젝트 생성 및 실행

개발 환경 구축이 끝났으니, 이제 터미널 명령어를 통해 직접 C# 프로젝트를 생성해 보겠습니다. VS Code의 내장 터미널(`Ctrl + ~`)을 열거나 외부 터미널을 사용하여 다음 명령어들을 순서대로 입력합니다.

```bash
# 프로젝트를 저장할 폴더를 생성하고 해당 폴더로 이동합니다.
mkdir HelloCSharp
cd HelloCSharp

# 콘솔 애플리케이션 템플릿을 기반으로 새 프로젝트를 생성합니다.
dotnet new console

# 현재 폴더를 기준으로 VS Code를 새롭게 엽니다.
code .

```

`dotnet new console` 명령어는 C#으로 콘솔(터미널)에서 동작하는 프로그램을 만들기 위한 기본 뼈대 코드와 설정 파일(`.csproj`)을 자동으로 생성해 주는 역할을 합니다.

명령어를 통해 VS Code가 다시 열리면 탐색기에 `Program.cs` 파일이 생성된 것을 볼 수 있습니다. 이 파일을 클릭해 보면 아래와 같이 아주 간결한 코드가 작성되어 있습니다.

```csharp
// Program.cs
Console.WriteLine("Hello, World!");

```

이제 이 코드를 실행해 볼 차례입니다. VS Code 상단 메뉴에서 **터미널 -> 새 터미널**을 클릭하거나 단축키 `Ctrl + ~`를 눌러 내장 터미널을 엽니다. 그리고 아래의 실행 명령어를 입력합니다.

```bash
dotnet run

```

터미널 화면에 `Hello, World!`라는 문구가 출력되었다면 여러분의 첫 C# 프로그램이 성공적으로 동작한 것입니다.

## 디버깅 환경 구성 (launch.json)

코드를 작성하다 보면 논리적인 오류를 찾기 위해 코드의 실행을 중간에 멈추고 변수의 값을 확인해야 하는 순간이 옵니다. 이를 '디버깅(Debugging)'이라고 하며, VS Code에서 디버깅을 시작하려면 디버그 설정 파일인 `launch.json`을 만들어야 합니다.

1. VS Code 왼쪽 활동 모음에서 벌레 모양의 **실행 및 디버그(Run and Debug)** 아이콘을 클릭하거나 단축키 `Ctrl+Shift+D`를 누릅니다.
2. 패널 화면에 보이는 파란색 글씨의 **"launch.json 파일을 만듭니다."** 링크를 클릭합니다.
3. 화면 상단에 환경을 선택하는 드롭다운 창이 나타나면 **".NET"** (또는 .NET Core)을 선택합니다.

이 과정을 거치면 VS Code가 자동으로 `.vscode` 폴더를 만들고 그 안에 `launch.json` 파일을 생성하여 디버깅에 필요한 기본 설정을 채워 넣습니다.

이제 코드 편집기 창에서 줄 번호 왼쪽의 빈 공간을 클릭하여 빨간색 점(중단점, Breakpoint)을 찍은 뒤, 키보드의 `F5` 키를 누르면 프로그램이 실행되다가 해당 줄에서 일시 정지하는 것을 확인할 수 있습니다.

---

## 빌드 작업 단축키 설정 (tasks.json)

코드를 수정할 때마다 터미널에 매번 `dotnet build`를 타이핑하는 것은 번거로운 일입니다. VS Code에서는 `tasks.json` 파일을 구성하여 이러한 반복 작업을 단축키로 쉽게 해결할 수 있습니다.

1. 키보드의 `Ctrl+Shift+P`를 눌러 명령 팔레트를 엽니다.
2. 검색창에 **"Tasks: Configure Task"**(작업: 작업 구성)를 입력하고 선택합니다.
3. 이어서 나오는 목록에서 **"Create tasks.json file from template"**(템플릿에서 tasks.json 파일 만들기)을 선택한 뒤, **".NET"** (또는 .NET Core) 항목을 고릅니다.

VS Code가 `.vscode` 폴더 안에 `tasks.json` 파일을 생성해 줍니다. 이제 터미널에 일일이 명령어를 칠 필요 없이, 키보드에서 `Ctrl+Shift+B`를 누르기만 하면 프로젝트 전체가 자동으로 빌드됩니다.

---

## 마무리

여기까지 따라오셨다면 VS Code를 활용하여 C# 프로젝트를 생성하고, 코드를 실행하며, 디버깅과 빌드 자동화까지 처리하는 가장 뼈대가 되는 개발 워크플로를 모두 익히신 것입니다.

* 코드를 작성하고 실행하는 기본 흐름: `dotnet new console` ➔ `code .` ➔ `dotnet run`
* 디버깅과 빌드 자동화 흐름: `launch.json` 설정 후 `F5` 디버깅 ➔ `tasks.json` 설정 후 `Ctrl+Shift+B` 빌드

이 기본 뼈대만 확실히 잡아두면, 이후에 여러 개의 파일로 프로젝트를 분리하거나 외부 라이브러리(NuGet 패키지)를 가져와 사용하는 등 더 복잡한 개발을 진행할 때도 흔들림 없이 환경을 제어할 수 있습니다. 완벽하게 준비된 환경에서 자유롭게 C# 코드를 작성해 보시기 바랍니다!
