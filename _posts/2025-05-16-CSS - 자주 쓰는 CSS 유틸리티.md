# 자주 쓰는 CSS 유틸리티 클래스 모음

## 빠른 사용법

- **붙여넣기**: 아래 `utility.css`를 `link`로 추가 후 바로 클래스 조합
- **확장**: 토큰(`:root` 변수)만 바꾸면 테마/스케일 전체가 바뀜
- **원칙**: “UI는 유틸로 배치/여백/색/타이포를 결정하고, 의미/상태는 컴포넌트 클래스가 가진다”

```html
<link rel="stylesheet" href="/css/utility.css">
<button class="btn bg-brand text-on-brand px-4 py-2 rounded hover:bg-brand-600 focus:outline">로그인</button>
```

---

## 유틸 클래스가 주는 가치

| 항목 | 요점 |
|---|---|
| 반복 제거 | 여백·정렬·색상 반복 선언 제거, HTML에서 조합만으로 스타일 완성 |
| 속도 | 디자인 시안 맞추며 **즉시 미세 조정** 가능 |
| 일관성 | 토큰/스케일 강제 → 임의 값 난립 방지 |
| 충돌 최소화 | BEM 없이도 특이도 낮은 단일 클래스 조합으로 충돌 줄임 |

---

## 설계 원칙(실무 지침)

1. **토큰 우선**: 임의 값 대신 `--space-2`, `--brand-500` 같은 시스템 값 고정
2. **규모 스케일**: 4/8 기반 spacing, 1.125·1.2 타이포 모듈 스케일
3. **특이도 최소**: `.u` 한 번으로 끝, 조합만으로 우선순위 충돌 방지 (`:where()`/`@layer` 활용)
4. **계층화**: `@layer tokens, utilities, components` 순으로 충돌 제어
5. **반응형 접두어**: `sm: md: lg: xl:`(min-width) 전략, 모바일 퍼스트
6. **상태/접근성**: `hover: focus: active: disabled: aria-[]:` 변형 제공
7. **국제화**: `:dir(rtl)` 가변 유틸 제공(좌우 반전)
8. **다크모드**: `[data-theme="dark"]`로 색 토큰 교체

## 상태·변형 유틸 확장

- `hover:`/`focus:`/`active:`/`disabled:`/`aria-[expanded=true]:` 등 상태 변형은 **의미/상태만** 표현하고, 색/여백은 기본 유틸로 결합.

```css
@layer utilities {
  .active\:translate-y-0_5:active{ transform:translateY(2px); }
  .disabled\:opacity-50:disabled{ opacity:.5; cursor:not-allowed; }
  .aria-expanded\:rotate-180[aria-expanded="true"]{ transform:rotate(180deg); }
}
```

```html
<button class="inline-flex items-center gap-2 px-4 py-2 bg-surface border rounded transition aria-expanded:rotate-180">
  <span class="inline-block transition">▼</span> 더 보기
</button>
```

## 다크 모드와 유틸 결합

- 본문에 `[data-theme="dark"]`를 적용하면 토큰만 바뀌므로 **유틸은 수정 불필요**.

```html
<html data-theme="dark">
<body class="font-sans text">
  <div class="bg-surface border rounded p-4">다크 테마 카드</div>
</body>
</html>
```

- 초기 FOUC 방지를 위해 `<head>` 인라인 스니펫으로 `data-theme` 선적용(이전 글 참고).

## 반응형 패턴과 컨테이너 사고방식

- 모바일 퍼스트로 기본 유틸 작성 → `sm: md: lg:`로 점진 확장
- “컴포넌트가 **자기 컨테이너 너비**를 기준으로 반응”하도록 설계하면 재사용 유리(컨테이너 쿼리가 필요하면 프레임워크 단계에서 도입).

```html
<div class="grid gap-4 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4">
  <!-- 동적 카드 -->
</div>
```

## 프린트/접근성/국제화 보강 유틸

```css
@layer utilities {
  /* 프린트 최소화 */
  @media print {
    .print\:hidden{ display:none !important; }
    .print\:black{ color:#000 !important; }
    .print\:bg-white{ background:#fff !important; }
  }
  /* 고대비/Forced Colors */
  @media (forced-colors: active) {
    .border{ border-color: CanvasText; }
    .focus\:outline:focus-visible{ outline-color: Highlight; }
  }
  /* RTL/LTR logical margin/padding 예시 */
  .ps-4{ padding-inline-start:var(--space-4); }
  .pe-4{ padding-inline-end:var(--space-4); }
}
```

## 성능·특이도·품질 관리

- **특이도 최소화**: `.class` 한 단계, `!important` 회피
- **@layer**: `tokens < utilities < components` 순서로 충돌 제어
- **번들 크기**: 실제 쓰는 유틸만 유지(빌드 단계에서 **Purge**/**Content-aware** 도입)
- **테스트**: iOS Safari, Samsung Internet, Firefox, Chrome에서 스크롤·포커스·다크/라이트 전환 확인
- **접근성**: 포커스 링 제거 금지(`outline: none` 금지), 대비비 4.5:1 이상 확인

## Tailwind 등 프레임워크와의 관계

- 본 글의 유틸 세트는 **경량·직관**에 방점. 대규모 시스템/디자인 토큰 고도화/플러그인 생태계가 필요하면 **Tailwind CSS**로 전환 권장.
- 다만 토큰·스케일·접근성·다크모드 사고방식은 **프레임워크를 바꿔도 동일하게 유효**.

---

## 확장 아이디어

- **유동 타이포**: `font-size: clamp(1rem, 1vw + .8rem, 1.25rem);` 유틸 추가
- **애니메이션 유틸**: `.animate-spin`, `.animate-pulse` 등
- **컨테이너 쿼리 유틸**: `@container` 기반 접두어(실험 가능)
- **데이터 속성 상태 유틸**: `[data-state=open]` 변형

```css
@layer utilities {
  .text-fluid{ font-size:clamp(1rem, 1vw + .8rem, 1.25rem); }
  .animate-spin{ animation:spin 1s linear infinite; }
  @keyframes spin{ to { transform:rotate(360deg); } }

  /* 예: data-state */
  .data-open\:block[data-state="open"]{ display:block; }
  .data-open\:hidden[data-state="open"]{ display:none; }
}
```

## 결론

- **유틸 = 빠른 납품과 일관성**의 기반.
- **토큰/스케일/레이어/접두어 규약**으로 팀 합의를 만들면, 프레임워크 없이도 Tailwind에 준하는 생산성을 얻게 됩니다.
- 다크모드·접근성·국제화·프린트까지 포함한 **작동하는 유틸 템플릿**을 바탕으로, 프로젝트에서 바로 복제·확장하세요.
