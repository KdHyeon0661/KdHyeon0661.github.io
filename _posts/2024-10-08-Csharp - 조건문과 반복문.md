# C# 조건문과 반복문: 제어 흐름의 기초와 고급 패턴

프로그래밍에서 조건문과 반복문은 컴퓨터가 스스로 결정을 내리고 동일한 작업을 효율적으로 수행하도록 만드는 가장 핵심적인 제어 구조입니다. 단순히 코드를 위에서 아래로 순차 실행하는 것을 넘어, 데이터의 상태와 사용자의 입력에 따라 복잡한 비즈니스 로직을 명확하게 표현하는 도구가 됩니다.

이 글에서는 C#이 제공하는 기본적인 제어문부터 시작하여, 최신 C# 버전에 도입된 강력한 패턴 매칭, 지연 실행(Lazy Evaluation), 실무에서 자주 쓰이는 디자인 패턴, 그리고 성능 최적화 기법까지 상세히 파헤쳐 보겠습니다.

## 조건문: 코드의 결정 지점

조건문은 주어진 논리식의 참과 거짓을 평가하여 프로그램의 실행 경로를 나눕니다. 조건문이 얼마나 직관적으로 작성되었느냐에 따라 코드의 가독성과 유지보수성이 크게 달라집니다.

### if 문과 조건 분기

`if` 문은 프로그래밍의 가장 원초적인 결정 구조입니다. 조건이 참(`true`)일 때 특정 코드 블록을 실행하며, `else if`와 `else`를 통해 다양한 상황에 대응할 수 있습니다.

```csharp
int temperature = 25;

if (temperature > 30)
{
    Console.WriteLine("날씨가 덥습니다.");
}
else if (temperature > 20)
{
    Console.WriteLine("날씨가 따뜻합니다.");
}
else if (temperature > 10)
{
    Console.WriteLine("날씨가 선선합니다.");
}
else
{
    Console.WriteLine("날씨가 춥습니다.");
}

```

단순한 분기 처리는 직관적이지만, 실제 비즈니스 로직에서는 예외 처리와 검증 로직이 겹치면서 `if` 문이 깊게 중첩되는 현상이 발생하기 쉽습니다.

### 가드 클로즈 (Guard Clauses): 중첩의 늪에서 벗어나기

메서드 내부에서 `if` 문이 여러 겹으로 중첩되면, 코드를 읽는 개발자는 머릿속에 너무 많은 문맥을 유지해야 합니다. 이를 해결하는 가장 우아한 방법이 바로 가드 클로즈(Guard Clauses) 패턴입니다.

이 패턴은 본격적인 로직을 실행하기 전에 유효하지 않은 조건을 미리 검사하여 메서드를 일찍 종료(`return`)시킵니다. 이를 통해 본문 코드의 들여쓰기 깊이를 줄이고 가독성을 획기적으로 높일 수 있습니다.

```csharp
// 중첩된 조건문 (가독성이 떨어지는 안 좋은 예)
public string GetUserStatus(User user)
{
    if (user != null)
    {
        if (user.IsActive)
        {
            if (user.Subscription != null)
            {
                if (user.Subscription.IsValid)
                {
                    return "정상 사용자";
                }
                else
                {
                    return "구독이 만료됨";
                }
            }
            else
            {
                return "구독 정보 없음";
            }
        }
        else
        {
            return "비활성 사용자";
        }
    }
    else
    {
        return "사용자 정보 없음";
    }
}

// 가드 클로즈 사용 (개선된 예)
public string GetUserStatusOptimized(User user)
{
    // 비정상적인 상황이나 예외 조건을 먼저 필터링하여 일찍 반환합니다.
    if (user is null) return "사용자 정보 없음";
    if (!user.IsActive) return "비활성 사용자";
    if (user.Subscription is null) return "구독 정보 없음";
    if (!user.Subscription.IsValid) return "구독이 만료됨";
    
    // 모든 검증을 통과한 핵심 로직은 중첩 없이 작성됩니다.
    return "정상 사용자";
}

```

## switch 문과 패턴 매칭

하나의 변수가 가질 수 있는 여러 값에 따라 분기해야 할 때 `if-else if` 체인을 길게 늘어놓는 것보다 `switch` 문을 사용하는 것이 훨씬 명확합니다.

### 전통적인 switch 문과 Fall-through 규칙

C#의 `switch` 문은 C나 C++과 유사해 보이지만, 치명적인 버그를 막기 위한 엄격한 규칙이 존재합니다. 바로 의도치 않은 Fall-through(아래로 떨어짐)를 허용하지 않는다는 점입니다. 각 `case` 블록 끝에는 반드시 `break`, `return`, 또는 `goto` 문이 있어야 컴파일이 통과됩니다.

```csharp
int score = 85;
string grade;

switch (score / 10)
{
    case 10:
    case 9:
        // C#에서는 실행할 코드가 없는 빈 case 레이블을 연속해서 작성하는 것만 Fall-through를 허용합니다.
        grade = "A";
        break;
    case 8:
    case 7:
        grade = "B";
        break;
    case 6:
        grade = "C";
        break;
    default:
        grade = "F";
        break;
}

```

### 패턴 매칭과 switch 식 (Switch Expression)

최신 C# 버전(8.0 이상)에서는 `switch`가 단순한 값 비교를 넘어 객체의 형태와 내부 속성까지 검사할 수 있는 패턴 매칭(Pattern Matching)의 중심으로 진화했습니다.

명령문(Statement)이었던 `switch`를 값으로 평가되는 식(Expression)으로 작성할 수 있게 되면서, 코드가 극적으로 간결해졌습니다. 또한 `when` 키워드를 통해 패턴 내에 추가적인 논리 조건을 부여할 수 있습니다.

```csharp
// 1. 객체의 속성을 기반으로 한 패턴 매칭 (속성 패턴)
public string GetWeatherDescription(WeatherData data)
{
    return data switch
    {
        // 온도와 습도를 동시에 평가합니다.
        { Temperature: > 30, Humidity: > 80 } => "불쾌지수가 높은 무더운 날씨",
        { Temperature: > 25 } => "따뜻한 날씨",
        { Temperature: > 15 } => "선선한 날씨",
        { Temperature: <= 0 } => "영하의 추운 날씨",
        _ => "보통 날씨" // _ (버리기 패턴)는 전통적인 default와 같습니다.
    };
}

// 2. 튜플 패턴 매칭 (여러 변수를 동시에 평가)
public string GetQuadrant(int x, int y)
{
    return (x, y) switch
    {
        (> 0, > 0) => "제1사분면",
        (< 0, > 0) => "제2사분면",
        (< 0, < 0) => "제3사분면",
        (> 0, < 0) => "제4사분면",
        (0, 0) => "원점",
        _ => "축 위에 있음"
    };
}

// 3. when 가드 조건을 활용한 복합 패턴
public string DescribeNumber(object obj)
{
    return obj switch
    {
        int n when n > 0 && n < 10 => "한 자리 양수",
        int n when n >= 10 && n < 100 => "두 자리 양수",
        int n when n >= 100 => "큰 양수",
        int n when n < 0 => "음수",
        string s when s.Length > 0 => $"문자열: {s}",
        null => "null 입력",
        _ => "알 수 없는 형식"
    };
}

```

이러한 패턴 매칭은 다형성을 다루거나 복잡한 데이터 구조를 분해하여 로직을 처리할 때 매우 우아한 해결책을 제공합니다.

## 반복문: 작업의 반복 실행

동일한 연산을 컬렉션의 모든 요소에 적용하거나 특정 조건을 만족할 때까지 작업을 수행해야 할 때 반복문을 사용합니다. 알고리즘의 시간 복잡도를 결정짓는 가장 중요한 요소이기도 합니다.

### for 문: 정해진 횟수의 반복

`for` 문은 반복 횟수가 명확하거나 인덱스가 필요할 때 가장 적합합니다. 초기화, 조건 검사, 증감식이 한 줄에 명시되므로 루프의 수명 주기를 파악하기 쉽습니다.

```csharp
// 기본적인 for 문
for (int i = 0; i < 10; i++)
{
    Console.WriteLine($"현재 값: {i}");
}

// 감소하는 for 문
for (int i = 10; i > 0; i--)
{
    Console.WriteLine($"카운트다운: {i}");
}

// 중첩된 for 문 (구구단 출력)
for (int i = 1; i <= 9; i++)
{
    for (int j = 1; j <= 9; j++)
    {
        Console.WriteLine($"{i} × {j} = {i * j}");
    }
}

```

만약 크기가 

$$N$$

인 배열을 중첩 `for` 문으로 순회한다면, 알고리즘의 시간 복잡도는 

$$O(N^2)$$

이 됩니다. 따라서 루프가 깊어질수록 성능에 미치는 영향을 주의 깊게 살펴야 합니다.

### while 문과 무한 루프 제어

`while` 문은 반복 횟수보다 '특정 조건이 유지되는 동안'이라는 논리가 중요할 때 사용합니다. 파일의 끝에 도달할 때까지 읽거나, 사용자가 종료 명령을 내릴 때까지 대기하는 시나리오에 적합합니다.

```csharp
// 파일 읽기 예제
int lineCount = 0;
using StreamReader reader = new StreamReader("data.txt");

while (!reader.EndOfStream)
{
    string? line = reader.ReadLine();
    if (line != null)
    {
        lineCount++;
        Console.WriteLine($"줄 {lineCount}: {line}");
    }
}

```

서버 프로세스나 백그라운드 워커에서는 `while (true)` 형태의 무한 루프를 자주 사용합니다. 이러한 루프는 반드시 안전하게 탈출할 수 있는 설계가 동반되어야 합니다.

```csharp
// CancellationToken을 이용한 취소 가능 루프 패턴
using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(5)); // 5초 후 자동 취소 상태로 변경

while (!cts.Token.IsCancellationRequested)
{
    Console.WriteLine("작업 실행 중...");
    Task.Delay(1000).Wait(); // 실제 코드에서는 비동기(await) 처리 권장
}

Console.WriteLine("작업이 안전하게 취소되었습니다.");

```

### do-while 문: 최소 1회 실행 보장

`do-while` 문은 조건 검사를 루프의 끝에서 수행합니다. 즉, 조건의 참/거짓 여부와 상관없이 본문 코드가 최소한 한 번은 무조건 실행됨을 보장합니다.

```csharp
int choice;
bool validChoice;

do
{
    Console.WriteLine("메뉴를 선택하세요 (1. 시작 / 2. 종료):");
    Console.Write("선택: ");
    
    validChoice = int.TryParse(Console.ReadLine(), out choice);
    
    if (!validChoice || choice < 1 || choice > 2)
    {
        Console.WriteLine("잘못된 선택입니다. 다시 시도하세요.\n");
        validChoice = false;
    }
    
} while (!validChoice);

```

이 패턴은 사용자에게 입력을 먼저 받고, 그 입력의 유효성을 검사하여 실패 시 다시 입력을 요구하는 로직에 완벽하게 들어맞습니다.

### foreach 문: 컬렉션 순회와 내부 원리

`foreach` 문은 배열, 리스트, 딕셔너리 등 컬렉션의 요소를 처음부터 끝까지 순회할 때 사용합니다. 인덱스 범위를 벗어날 위험이 없어 코드가 안전하고 간결해집니다.

```csharp
// 딕셔너리 순회 예제
Dictionary<string, int> scores = new Dictionary<string, int>
{
    ["Alice"] = 95,
    ["Bob"] = 87,
    ["Charlie"] = 92
};

foreach (KeyValuePair<string, int> entry in scores)
{
    Console.WriteLine($"{entry.Key}: {entry.Value}점");
}

```

`foreach`는 마법처럼 동작하는 것이 아니라, 컴파일러가 이터레이터 패턴(Iterator Pattern)으로 코드를 자동 변환해 주는 문법적 편의입니다.

```text
[foreach의 내부 동작 구조]

IEnumerable 객체 ── GetEnumerator() 요청 ──> IEnumerator (열거자 객체 반환)
                                                  │
   ┌──────────────────────────────────────────────┘
   │
   ├── 1. MoveNext() 호출 (다음 요소로 이동, 요소가 있으면 true 반환)
   ├── 2. Current 속성 접근 (현재 가리키고 있는 데이터 반환)
   └── 3. MoveNext()가 false를 반환할 때까지 1~2 반복

```

즉, 어떠한 커스텀 클래스라도 `GetEnumerator()` 메서드를 노출하여 열거자를 반환하도록 구현하면 `foreach` 문을 그대로 사용할 수 있습니다.

### 지연 실행과 yield return

메모리 효율을 극대화하기 위해 C#은 `yield return`이라는 강력한 기능을 제공합니다. 대용량 데이터를 처리할 때 모든 데이터를 한 번에 메모리(리스트 등)에 올려두고 반환하는 대신, 호출자가 값을 하나 요청할 때마다 계산해서 넘겨주고 함수의 상태를 일시 정지시킵니다.

```csharp
public IEnumerable<int> GetEvenNumbers(int max)
{
    for (int i = 0; i <= max; i++)
    {
        if (i % 2 == 0)
        {
            // 값을 호출자에게 반환한 뒤, 현재 지역 변수 상태를 기억한 채로 일시 중단(Pause)됩니다.
            yield return i; 
        }
    }
}

// 사용 예 (데이터가 한꺼번에 생성되지 않고, 루프가 도는 순간순간 생성됩니다)
foreach (int num in GetEvenNumbers(1000000))
{
    if (num > 10) break;
    Console.WriteLine(num); // 0, 2, 4, 6, 8, 10 출력 후 종료
}

```

이러한 방식을 지연 실행(Lazy Evaluation)이라고 합니다. 위 코드에서 `yield return` 없이 100만 개의 짝수를 리스트에 담아 반환했다면 막대한 메모리가 낭비되었을 것입니다. 하지만 `yield return`을 사용하면 단 하나씩만 값을 평가하므로 메모리 사용량이 획기적으로 줄어듭니다.

## 반복문 제어: break, continue, goto

반복문 내부에서 실행 흐름을 강제로 비틀어야 할 때 제어문을 사용합니다.

* **break**: 루프의 실행을 즉시 중단하고 반복문을 완전히 빠져나갑니다. 데이터를 검색하다가 원하는 값을 찾았을 때 불필요한 추가 연산을 막기 위해 사용합니다.
* **continue**: 루프의 현재 회차만 건너뛰고, 바로 다음 회차의 조건 검사나 증감식으로 이동합니다.
* **goto**: 지정한 레이블(Label) 위치로 코드 실행 지점을 순간 이동시킵니다. (중첩 루프 탈출 등 제한적 상황에서만 사용 권장)

## 실무에서 빛을 발하는 제어문 설계 패턴

앞서 배운 조건문과 반복문, 그리고 제어 키워드들을 조합하면 실무에서 마주하는 복잡한 시스템들을 우아하게 설계할 수 있습니다.

### 1. 데이터 처리 파이프라인 패턴

대량의 데이터를 정제하고 가공할 때, `continue` 문을 활용하여 예외적인 데이터를 빠르게 필터링하는 파이프라인을 구축할 수 있습니다.

```csharp
public class DataProcessor
{
    public List<double> ProcessData(List<double> rawData)
    {
        List<double> processedData = new List<double>();
        
        foreach (double value in rawData)
        {
            // 1단계 필터링: 유효하지 않은 음수 값은 즉시 건너뜀 (continue 활용)
            if (value < 0)
            {
                continue; 
            }
            
            // 2단계 가공: 지나치게 큰 값의 상한선 제한
            double limitedValue = Math.Min(value, 1000);
            
            // 3단계 변환: 로그 변환 적용
            double transformedValue = Math.Log(limitedValue + 1);
            
            processedData.Add(transformedValue);
        }
        
        return processedData;
    }
}

```

### 2. 상태 머신 (State Machine) 패턴

게임 개발이나 복잡한 주문 처리 시스템에서는 객체가 현재 어떤 '상태(State)'인지에 따라 다르게 동작해야 합니다. 이때 열거형(`enum`)과 `switch` 문을 결합하면 훌륭한 상태 머신을 구현할 수 있습니다.

```csharp
public class GameStateMachine
{
    public enum GameState
    {
        MainMenu,
        Playing,
        Paused,
        GameOver
    }
    
    private GameState _currentState = GameState.MainMenu;
    
    // 사용자 입력이 들어올 때마다 현재 상태에 따라 흐름을 분기합니다.
    public void ProcessInput(string input)
    {
        switch (_currentState)
        {
            case GameState.MainMenu:
                HandleMainMenuInput(input);
                break;
            case GameState.Playing:
                HandlePlayingInput(input);
                break;
            case GameState.Paused:
                HandlePausedInput(input);
                break;
            case GameState.GameOver:
                HandleGameOverInput(input);
                break;
        }
    }
    
    private void HandleMainMenuInput(string input)
    {
        if (input.ToLower() == "start")
        {
            Console.WriteLine("게임 시작!");
            _currentState = GameState.Playing; // 상태 전이
        }
        else if (input.ToLower() == "quit")
        {
            Environment.Exit(0);
        }
    }
    
    private void HandlePlayingInput(string input)
    {
        if (input.ToLower() == "pause")
        {
            Console.WriteLine("게임 일시정지");
            _currentState = GameState.Paused; // 상태 전이
        }
        else if (input.ToLower() == "die")
        {
            Console.WriteLine("플레이어 사망, 게임 오버.");
            _currentState = GameState.GameOver; // 상태 전이
        }
    }
}

```

## 성능 최적화와 모범 사례

### 반복 중 컬렉션 수정 금지

`foreach` 문을 사용하여 리스트를 순회하는 도중에 요소를 추가하거나 삭제하면 `InvalidOperationException`이 발생합니다. 이는 내부의 열거자가 컬렉션이 변경되었음을 감지하고 순회 무결성이 깨졌다고 판단하기 때문입니다.

요소를 조건에 따라 삭제해야 한다면, **끝 인덱스에서부터 역순으로 `for` 문을 사용하여 순회**해야 합니다. 역순으로 지우면 리스트의 요소들이 앞으로 당겨지더라도 아직 검사하지 않은 인덱스에는 영향을 미치지 않기 때문입니다.

```csharp
List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };

// 안전한 삭제를 위한 역순 순회 패턴
for (int i = numbers.Count - 1; i >= 0; i--)
{
    if (numbers[i] % 2 == 0) // 짝수라면
    {
        numbers.RemoveAt(i); // 안전하게 삭제 완료
    }
}

```

### Span<T>를 활용한 고성능 순회

성능이 극도로 중요한 애플리케이션에서는 연속된 메모리 블록을 매우 빠르게 순회할 수 있는 `Span<T>`나 `ReadOnlySpan<T>`를 사용하는 것이 트렌드입니다.

```csharp
public int SumSpan(ReadOnlySpan<int> span)
{
    int total = 0;
    // Span<T>에 대한 for 순회는 JIT 컴파일러가 배열 경계 검사(Bounds Check)를 생략하도록 
    // 최적화하여 C++ 포인터 연산에 버금가는 속도를 냅니다.
    for (int i = 0; i < span.Length; i++)
    {
        total += span[i];
    }
    return total;
}

// 사용 예
int[] array = { 1, 2, 3, 4, 5 };
int sum = SumSpan(array); // 배열은 암시적으로 Span<T>로 변환되어 전달됩니다.

```

## 결론

C#의 조건문과 반복문은 단순한 문법을 넘어, 개발자의 논리적 사고를 컴퓨터의 언어로 번역하는 가장 중요한 도구입니다.

`if` 문의 중첩을 피하기 위해 가드 클로즈를 활용하고, 복잡한 데이터 검증은 최신 패턴 매칭 `switch` 식으로 리팩토링하세요. 또한 `yield return`을 활용한 지연 실행, 안전한 상태 머신 구현, 그리고 역순 순회를 통한 컬렉션 제어 등 실무 패턴들을 코드에 녹여낸다면 유지보수하기 좋고 성능까지 뛰어난 C# 애플리케이션을 완성할 수 있을 것입니다.
