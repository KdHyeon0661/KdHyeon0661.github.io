# 텍스트 스타일링 정리 — `text-align`, `line-height`, `letter-spacing` 중심 확장(＋타이포그래피 핵심, 마이크로타입, 다국어, 웹폰트, 줄바꿈·말줄임, 접근성)

당신이 준 초안을 그대로 확장해, 실무에서 반드시 쓰이는 **줄바꿈 규칙**, **말줄임/클램프**, **하이픈/워드랩**,  
**폰트 관련 속성(가변 폰트·합자·커닝·라틴/한글 차이)**, **다국어/세로쓰기**, **웹폰트 로딩 전략**,  
**가독성·대비·리듬**까지 한 번에 정리합니다. 모든 코드는 복사-붙여넣기 즉시 동작하도록 구성했습니다.

---

## 0) 빠른 개요(요약표)

| 카테고리 | 핵심 속성 | 기억 포인트 |
|---|---|---|
| 정렬 | `text-align` | 본문은 `left/justify`, 표/수치열은 `right` 고려 |
| 줄 간격 | `line-height` | 본문 1.5~1.8, 제목 1.2~1.4 권장(단위 없는 배수) |
| 글자 간격 | `letter-spacing` | 제목(대문자·고딕)에 약간의 양수, 본문은 보통 0 |
| 단어 간격 | `word-spacing` | `justify` 보정, 한글에선 보통 사용 빈도 낮음 |
| 들여쓰기 | `text-indent` | 첫 줄만, `margin-left`와 역할 다름 |
| 공백·줄바꿈 | `white-space`, `overflow-wrap`, `word-break`, `hyphens` | 언어/콘텐츠 특성 맞춰 조합 |
| 말줄임 | `text-overflow`, `-webkit-line-clamp` | 1줄/멀티라인 트렁케이션 패턴 |
| 변환·장식 | `text-transform`, `text-decoration*` | 링크 밑줄 커스텀, 케이스 변환 |
| 고급 타이포 | `font-kerning`, `font-feature-settings`, `font-variant-*`, 가변 폰트 | 합자·옛한글·스몰캡·가변 축 |
| 다국어 | `direction`, `unicode-bidi`, `writing-mode` | RTL/세로쓰기 |
| 폰트 로딩 | `@font-face`, `font-display`, `font-family` 스택 | FOUT/FOIT 제어, 시스템 폰트 스택 |
| 접근성 | 대비·크기·문단폭·리듬 | 색 대비, 최대 줄 길이 60–75자 등 |

## 1) `text-align` — 텍스트 정렬

```css
/* 기본 */
p { text-align: left; }

/* 가운데 정렬(영문 제목·배너) */
.hero-title { text-align: center; }

/* 양쪽 정렬(신문 스타일) */
.article p { text-align: justify; text-justify: inter-word; }

/* 표/금액 열 정렬 */
.table td.amount { text-align: right; }
```

- `justify`는 **영문**에서 특히 효과적. **한글**에서는 자간/어간이 어색해질 수 있어 **문단 폭/하이픈/워드랩** 보조가 중요합니다.
- 추가: `text-justify: inter-word | inter-character`(브라우저 지원 범위 확인)로 양쪽정렬 알고리즘 힌트.

---

## 2) `line-height` — 줄 간격(리듬)

```css
/* 단위 없는 배수(권장): 폰트 크기 변화에도 리듬 유지 */
body { line-height: 1.6; }

/* 제목은 촘촘하게 */
h1, h2, h3 { line-height: 1.25; }
```

권장 범위(경험칙):
- 본문: **1.5–1.8**
- 제목: **1.2–1.4**
- 캡션/부연: **1.4–1.6**

수학적 직관: 폰트 크기 \(f\)일 때 줄 간격 \(L\)은
$$
L = \alpha \cdot f \quad (\alpha \in [1.4, 1.8])
$$
위와 같이 **배수(line-height multiplier)** \(\alpha\)로 정의하면, 반응형 · 접근성 확대 시에도 안정적 리듬을 유지할 수 있습니다.

---

## 3) `letter-spacing` — 글자 간격(자간)

```css
/* 제목(대문자/고딕)에 소량의 자간 */
h1.title { letter-spacing: .02em; }

/* 타이트한 로고/히어로(주의: 가독성 훼손 유의) */
.hero .brand { letter-spacing: .06em; }
```

- 단위는 보통 **`em` 선호**(폰트 크기에 상대적).  
- 한글 본문은 보통 0에 가깝게 두며, **장문의 자간 증가는 피로**를 유발할 수 있음.

---

## 4) `word-spacing` — 단어 간격

```css
/* 양쪽정렬에서 단어간 과도 압축/늘림 보정 */
.article p.tight { word-spacing: .05em; }
```
- 영문 기사에서 `justify`와 함께 미세 조정.
- 한글은 단어 경계가 띄어쓰기여도 **음절 단위 인지**가 강하므로 사용 빈도 낮음.

---

## 5) `text-indent` — 첫 줄 들여쓰기

```css
/* 전자책/에디터 풍 문단 */
.article p { text-indent: 1.5em; }

/* 제목 다음 첫 문단은 들여쓰기 제거 패턴 */
.article h2 + p { text-indent: 0; }
```

- `margin-left`는 **전체 문단** 이동, `text-indent`는 **첫 줄**만.

---

## 6) `text-decoration` — 밑줄/취소선/두께·색상·스타일

```css
/* 링크 밑줄 커스텀 */
a {
  color: #0a58ca;
  text-decoration-line: underline;
  text-decoration-color: color-mix(in oklab, currentColor 60%, black 15%);
  text-decoration-thickness: from-font; /* 또는 1px/0.08em */
  text-underline-offset: .12em;         /* 밑줄과 글자 사이 거리 */
}

/* 취소선 */
.del { text-decoration: line-through; }
```

- `text-decoration-skip-ink: auto;`로 **글자 하행부와 밑줄 충돌** 최소화(지원 엔진에서).

---

## 7) `text-transform` — 대/소문자·단어별

```css
.title { text-transform: uppercase; }
.teaser { text-transform: capitalize; }
```

- 실제 데이터는 **원본 유지**, 표시만 변환.  
- 지역화 케이스(터키어 i 등)에 주의.

## 8) 공백/줄바꿈 처리 — `white-space` + 래핑 3형제

### 8.1 `white-space`
```css
code    { white-space: pre; }        /* 공백/줄바꿈 그대로 */
pre     { white-space: pre-wrap; }   /* 보존 + 필요 시 래핑 */
.nowrap { white-space: nowrap; }     /* 한 줄 유지 */
```

### 8.2 래핑/하이픈(언어별 조합)
```css
.readable {
  overflow-wrap: anywhere; /* 긴 URL/토큰도 어디서든 줄바꿈 */
  word-break: normal;      /* CJK엔 필요 시 'keep-all' 검토 */
  hyphens: auto;           /* 영문 하이픈 자동 삽입(사전 기반) */
}
```

- **영문 기사**: `hyphens:auto` + `overflow-wrap:anywhere`  
- **한글**: `word-break: keep-all;`로 **어절 단위**를 유지(문장 자연스러움 ↑).  
  긴 토큰이 많은 UI(테이블/카드)는 `overflow-wrap:anywhere`로 **레이아웃 파손 방지**.

```css
.korean {
  word-break: keep-all;         /* 한글에서 권장 */
  overflow-wrap: anywhere;      /* 긴 이메일/URL 대응 */
}
```

## 9) 말줄임 — 한 줄 / 여러 줄

### 9.1 한 줄 말줄임(트렁케이션)
```css
.ellipsis-1 {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
  max-width: 48ch; /* 줄 길이 컨트롤(권장) */
}
```

### 9.2 여러 줄 말줄임(라인 클램프)
```css
.clamp-3 {
  display: -webkit-box;
  -webkit-line-clamp: 3;   /* 3줄 제한 */
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

- 접근성: 중요한 정보가 잘리지 않도록, **title 속성/토글/더보기** 제공 고려.

## 10) 텍스트 넘침·정렬 디테일

```css
/* 숫자 열 정렬(등폭 수직 맞춤) */
.tabular {
  font-variant-numeric: tabular-nums; /* 012345 동일 폭 */
  text-align: right;
}

/* 코드/토큰 강제 줄바꿈 */
.break-all {
  word-break: break-all;     /* CJK/토큰 섞인 테이블에 한정 사용 */
  overflow-wrap: anywhere;
}
```

## 11) 고급 타이포그래피 — 커닝/합자/숫자/가변 폰트

```css
/* 커닝(글자 사이 미세 조정) */
body { font-kerning: normal; }

/* 합자/문자 기능(브라우저·폰트 지원 필요) */
.f-lig {
  font-feature-settings: "liga" 1, "clig" 1; /* 표준/컨텍스트 합자 */
}

/* 숫자: 등폭·옛형/현대형 */
.nums {
  font-variant-numeric: tabular-nums lining-nums; /* 표 높이 맞춤에 유리 */
}

/* 스몰캡, 옛한글 등(폰트 지원 필요) */
.smallcaps { font-variant-caps: small-caps; }
```

### 11.1 가변 폰트(Variable Fonts)
```css
@font-face {
  font-family: 'InterVar';
  src: url('Inter-Variable.woff2') format('woff2-variations');
  font-weight: 100 900;           /* 가중치 범위 */
  font-stretch: 75% 125%;         /* 폭 범위 */
  font-style: normal;
  font-display: swap;
}

.h1 {
  font-family: 'InterVar', system-ui, -apple-system, 'Segoe UI', Roboto, 'Malgun Gothic', sans-serif;
  font-weight: 700;
}
.h1--tuned {
  font-variation-settings: "wght" 680, "wdth" 95; /* 세밀 조정 */
}
```

## 12) 다국어/쓰기 방향 — RTL & 세로쓰기

```css
/* 아랍어/히브리어와 같은 RTL */
[dir="rtl"] .panel {
  direction: rtl;
  text-align: right;
}

/* 세로쓰기(일본어 레이아웃 예시, 브라우저/폰트 의존) */
.tate {
  writing-mode: vertical-rl;  /* 위→아래, 우→좌 */
  text-orientation: mixed;    /* 영문은 세로 회전 */
  line-height: 1.8;
}
```

- LTR/RTL **혼합 콘텐츠**는 `unicode-bidi: bidi-override | isolate` 등으로 섬 격리.
- 세로쓰기는 폰트 지원/브라우저 렌더링 차이를 반드시 사전 확인.

---

## 13) 웹폰트 로딩 전략 — FOUT/FOIT 제어

```css
@font-face {
  font-family: 'Pretendard';
  src: url('/fonts/Pretendard.woff2') format('woff2');
  font-display: swap; /* swap | fallback | optional */
}

/* 시스템 폰트 스택(로딩 지연 최소) */
:root {
  --system-ui: system-ui, -apple-system, 'Segoe UI', Roboto, 'Noto Sans', 'Malgun Gothic', sans-serif;
}
body { font-family: var(--system-ui); }
```

- `font-display: swap`으로 **FOIT(깜빡이며 안 보임)** 방지, 초기엔 대체 폰트 표시.
- 핵심 페이지 성능 위해 대제목은 **시스템 폰트**, 본문만 웹폰트 등 선택적 적용.

## 15) 접근성/가독성 체크리스트

- **색 대비**: 본문 텍스트는 WCAG AA(4.5:1) 이상 권장.
- **문단 폭**: 한 줄 **60–75자**(영문 기준, 한글도 55–70자 권장).
- **줄 간격**: 본문 1.5–1.8, 제목 1.2–1.4.
- **링크 식별성**: 색만으로 구분하지 말고 밑줄/굵기/패턴 병행.
- **키보드 포커스**: 포커스시 `outline` 제거 금지(대체 제공 필수).
- **말줄임**: 중요 정보는 잘리지 않도록 **접근 가능한 더보기** 제공.
- **폰트 크기**: 루트 기준 `rem` 사용, 사용자 확대에 잘 반응.
- **언어 속성**: `<html lang="ko">` 등 **언어 메타** 정확히 지정(하이픈·음성합성·맞춤법 도우미에 영향).

## 17) 마무리 요약

- **정렬**은 맥락에 맞게(`left/justify/center/right`),  
- **줄 간격**은 배수(line-height)로 리듬을 고정,  
- **간격·들여쓰기**는 목적에 맞는 속성으로(`letter/word-spacing`, `text-indent`),  
- **공백/줄바꿈**은 언어 특성에 맞춰(`keep-all`, `hyphens`, `overflow-wrap`),  
- **말줄임**은 1줄/다줄 패턴으로 접근성까지 고려,  
- **폰트 기능**(커닝·합자·숫자·가변 폰트)으로 디테일을 완성,  
- **웹폰트 로딩**은 `font-display`·시스템 스택으로 FOUT/FOIT 제어,  
- **접근성**은 대비·줄 길이·라인하이트·포커스가 핵심입니다.
