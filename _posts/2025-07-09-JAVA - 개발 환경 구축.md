# Java 개발 환경 구축 (IntelliJ, Eclipse, VSCode 등)

## 무엇을 설치하고 왜 필요한가

- **JDK(Java Development Kit)**: 컴파일러(`javac`), 런타임(`java`) 등 개발 도구 모음. 개발/실행 모두 JDK만 있으면 됩니다.
- **IDE**: 코드 작성·디버깅·리팩토링·빌드 자동화를 돕는 통합 환경
  - **IntelliJ IDEA**: 생산성·리팩토링·프레임워크 통합이 강력
  - **Eclipse**: 전통적인 Java IDE, 풍부한 플러그인, 경량 감각
  - **VSCode**: 경량 에디터 + 확장팩으로 Java 개발 지원

> 권장 JDK 버전: **LTS(장기 지원)** 라인을 권장(예: 21, 25). 팀 표준과 빌드/런타임 호환을 우선합니다.

---

## JDK 설치 (Windows / macOS / Linux)

### 설치 원칙 요약

- **패키지 관리자**(winget/choco/brew/apt/dnf 등)로 설치하면 업데이트가 편리합니다.
- **여러 JDK 버전**을 쓰려면 **버전 관리자**(예: SDKMAN, jEnv) 또는 OS의 **대안 선택(update-alternatives)** 를 사용합니다.
- **JAVA_HOME**과 **PATH**를 정확히 잡아야 IDE/빌드 도구가 JDK를 인식합니다.

### Windows

#### 패키지 관리자 이용

```powershell
# winget (권장)

winget search OpenJDK
winget install EclipseAdoptium.Temurin.21.JDK     # Temurin 21 예시
winget install Oracle.JDK.25                      # Oracle JDK 25 예시

# chocolatey

choco install temurin21         # 또는 temurin-25-jdk 패키지 검색
choco install ojdkbuild11

# scoop

scoop bucket add java
scoop install temurin-lts-jdk
```

#### 환경 변수 설정

```powershell
# - 예: Temurin 21

setx JAVA_HOME "C:\Program Files\Eclipse Adoptium\jdk-21"
setx PATH "%JAVA_HOME%\bin;%PATH%"
```

#### 확인

```powershell
java -version
javac -version
```

### macOS

#### Homebrew

```bash
brew update
brew install temurin@21     # LTS 예시
# 또는 최신 장기 지원 외 버전도 별도 cask 존재

```

#### SDKMAN (멀티 JDK 관리에 편리)

```bash
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
sdk list java
sdk install java 21-tem     # Temurin 21
sdk use java 21-tem
```

#### 확인 및 JAVA_HOME

```bash
/usr/libexec/java_home -V
export JAVA_HOME=$(/usr/libexec/java_home -v 21)
echo $JAVA_HOME
java -version
```

### Linux (Debian/Ubuntu, RHEL/Fedora, Arch, SUSE)

```bash
# Debian/Ubuntu

sudo apt update
sudo apt install -y openjdk-21-jdk
sudo update-alternatives --config java
sudo update-alternatives --config javac

# RHEL/Fedora

sudo dnf install -y java-21-openjdk java-21-openjdk-devel

# Arch

sudo pacman -S jdk21-openjdk

# openSUSE

sudo zypper install java-21-openjdk java-21-openjdk-devel
```

#### JAVA_HOME 설정

```bash
# 예: Debian/Ubuntu

echo 'export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
java -version && javac -version
```

### 멀티 JDK 버전 운용 팁

- **SDKMAN**(macOS/Linux): `sdk install/use/java`로 프로젝트별 전환
- **jEnv**(macOS/Linux): `jenv add`, `jenv local 21`로 디렉터리 단위 고정
- **Windows**: IDE의 **Project SDK**를 프로젝트별로 지정하거나, **Gradle Toolchain**을 사용

## IntelliJ IDEA

### 설치

- **JetBrains Toolbox**로 설치/업데이트 관리(권장) 또는 설치 프로그램으로 직접 설치
- Community(무료) / Ultimate(유료) 중 선택

### 최초 설정

- **File > New Project > Java**
  - **JDK**: 설치한 JDK를 **Project SDK**로 지정
  - **Build System**: Gradle 또는 Maven 선택
- **Project Structure(⌘; / Ctrl+Alt+Shift+S)**
  - Project SDK/Language level 일치 여부 확인
  - Modules/Dependencies에서 라이브러리 확인

### 실행/디버깅: HelloWorld

```java
public class HelloIntelliJ {
    public static void main(String[] args) {
        System.out.println("Hello, IntelliJ!");
    }
}
```
- **Run > Run 'HelloIntelliJ'**
- 브레이크포인트(에디터 왼쪽 여백 클릭) → **Debug** 실행 → Variables/Frames 관찰

### Maven/Gradle 템플릿

**Maven `pom.xml`**
```xml
<project xmlns="http://maven.apache.org/POM/4.0.0" ...>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>demo</artifactId>
  <version>1.0.0</version>
  <properties>
    <maven.compiler.source>21</maven.compiler.source>
    <maven.compiler.target>21</maven.compiler.target>
  </properties>
  <dependencies><!-- 필요시 추가 --></dependencies>
</project>
```

**Gradle `build.gradle`**
```gradle
plugins { id 'java' }
java { toolchain { languageVersion = JavaLanguageVersion.of(21) } }
repositories { mavenCentral() }
dependencies { /* 필요시 추가 */ }
```

### 유용 설정

- **Settings > Build, Execution, Deployment > Gradle**: Gradle JDK를 **프로젝트 JDK**로 맞추기
- **Editor > Code Style/Inspections**: 팀 컨벤션 반영
- **Version Control**: Git 연동, Code Review 툴 체인(예: GitHub) 연결

### 자주 겪는 문제

- **Project SDK 미설정**: `File > Project Structure`에서 SDK 지정
- **Gradle JDK 불일치**: Gradle JVM을 프로젝트 JDK로 변경
- **인덱싱 느림**: 불필요 플러그인 비활성화, 메모리 할당 증가(Help > Change Memory Settings)

## Eclipse IDE

### 설치/워크스페이스

- 패키지(“Eclipse IDE for Java Developers”) 설치 후 첫 실행 시 **워크스페이스 폴더** 선택
- 워크스페이스는 프로젝트/설정들이 저장되는 **작업 디렉터리**

### 새 프로젝트/실행

- **File > New > Java Project** → JRE/JDK 선택
- **New > Class** → `public static void main` 체크
```java
public class HelloEclipse {
    public static void main(String[] args) {
        System.out.println("Hello from Eclipse!");
    }
}
```
- **Run > Run As > Java Application**

### 빌드 도구 통합

- **Maven(m2e)**: `File > New > Maven Project`
- **Gradle(Buildship)**: `File > New > Gradle Project`

### 디버깅/성능 팁

- **Debug Perspective**에서 Variables/Expressions 활용
- **Content Assist(자동완성)**, **Organize Imports**(Ctrl+Shift+O)
- 느릴 때: 불필요 플러그인 제거, 메모리 옵션 조정(`eclipse.ini`의 `-Xms/-Xmx`)

## Visual Studio Code (VSCode)

### 설치 & 확장팩

- 설치 후 **Extension Pack for Java**(언어 지원/디버거/테스트/Maven 등 포함) 설치

### 최소 예제/실행

```java
public class HelloVSCode {
    public static void main(String[] args) {
        System.out.println("Hello from VSCode!");
    }
}
```
- 에디터 상단 **Run** 또는 **Ctrl+F5**로 실행

### 디버깅 설정(`.vscode/launch.json`)

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "java",
      "name": "Launch Current File",
      "request": "launch",
      "mainClass": "${file}"
    }
  ]
}
```

### JDK 지정(`settings.json`)

```json
{
  "java.configuration.runtimes": [
    {
      "name": "JavaSE-21",
      "path": "/Library/Java/JavaVirtualMachines/temurin-21.jdk/Contents/Home"
    },
    {
      "name": "JavaSE-25",
      "path": "C:\\Program Files\\Java\\jdk-25"
    }
  ]
}
```

### Maven/Gradle 통합

- **Maven for Java**: 사이드바에서 POM 탐색/빌드/테스트
- **Gradle for Java**: 태스크 탐색/실행, 래퍼 감지

### WSL/Remote 개발

- **Remote - WSL**: Windows VSCode ↔ Ubuntu(WSL)의 JDK/툴체인 사용
- **Remote - SSH** 또는 **Dev Containers**: 원격 서버/컨테이너 내부에서 빌드·디버깅

## 빌드 도구: Maven/Gradle 빠른 시작

### Maven 프로젝트 구조

```
project
 ├─ pom.xml
 └─ src
    ├─ main
    │   └─ java
    └─ test
        └─ java
```
```bash
mvn -q -DskipTests package
java -jar target/<artifact>-<version>.jar
```

### Gradle 프로젝트 구조

```
project
 ├─ build.gradle
 ├─ settings.gradle
 └─ src
    ├─ main/java
    └─ test/java
```
```bash
gradle build
java -jar build/libs/<name>-<version>.jar
```

> **Toolchain(Gradle)** 을 사용하면 빌드 시 필요한 JDK 버전을 자동 설치/선택해 **“내 로컬 JDK와 달라도”** 재현 가능한 빌드를 만들 수 있습니다.

**.devcontainer/devcontainer.json**
```json
{
  "name": "Java Dev",
  "image": "mcr.microsoft.com/devcontainers/java:1-21-bullseye",
  "customizations": {
    "vscode": {
```

```dockerfile

WORKDIR /workspace
COPY . .

```

## 자주 겪는 오류와 해결

- **`'java' is not recognized` / `javac: command not found`**
  → JAVA_HOME/PATH 점검, 터미널 재시작, IDE의 Project SDK/Gradle JVM 재지정
- **JDK 버전 미스매치(예: 21로 컴파일, 17로 실행)**
  → `--release`/`target` 일치, 런타임 JDK 경로 확인
- **VSCode가 JDK를 못 찾음**
  → `settings.json`의 `java.configuration.runtimes` 지정
- **Eclipse 빌드 경로 오류**
  → Project > Properties > Java Build Path에서 JRE System Library 교체
- **Gradle 메모리 부족**
  → `gradle.properties`: `org.gradle.jvmargs=-Xmx2g -Dfile.encoding=UTF-8`
- **인증서/HTTPS 오류**
  → 사내 프록시·인증서 체인 설정(Gradle/Maven에 CA 추가), JDK cacerts 업데이트
