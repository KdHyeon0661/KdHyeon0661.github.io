# HTML에서 스타일과 색상 다루기 — CSS와 함께하는 웹 꾸미기

## CSS로 스타일 지정하는 3가지 방식과 권장 패턴

### 인라인 스타일 (빠른 실험, 장기 유지보수에는 비권장)

```html
<p style="color: red; font-size: 18px;">이 문장은 빨간 글자입니다.</p>
<h1 style="background-color:white; color:maroon; font-size:150%; text-align:center">
  style 속성으로 한 번에 스타일링
</h1>
```
- 장점: 빠른 적용, 특정 요소 오버라이드에 유용
- 단점: **재사용/검색/일관성/캐시/특이성 관리**에 취약 → 규모가 커질수록 유지보수 비용 급증

### 내부 스타일 시트 (페이지 한정 스타일)

```html
<head>
  <style>
    p { color: blue; font-size: 20px; }
  </style>
</head>
```
- 장점: 한 문서 안에서 빠른 관리
- 단점: 페이지가 늘면 중복과 불일치 발생

### 외부 스타일 시트 (실무 기본)

```html
<!-- HTML -->
<link rel="stylesheet" href="/assets/style.css">
```
```css
/* /assets/style.css */
p { color: green; font-weight: bold; }
```
- 장점: **재사용, 캐시, 일관성, 버전 관리** 모두 유리
- 팁: **Critical CSS 인라인 + 나머지 외부 로드** 전략으로 초기 렌더 성능 개선

## CSS 기본기: 박스 모델, 리셋, 특이성, 계단식

### 리셋/노멀라이즈와 박스 모델 통일

```css
/* 권장: 모든 요소에 border-box */
*,
*::before,
*::after { box-sizing: border-box; }

/* 간단 리셋 */
html, body, h1, h2, h3, p, ul, ol, li, figure { margin: 0; padding: 0; }
img { max-width: 100%; display: block; }
```

### 특이성(Specificity)과 계단식(Cascade)

- 인라인 > ID > 클래스/속성/가상클래스 > 태그/가상요소
- **유지보수 팁**: BEM, 유틸리티 클래스, 레이어(`@layer`)로 특이성 폭주 방지

```css
@layer reset, base, components, utilities;

@layer base {
  body { font: 16px/1.6 system-ui, sans-serif; color: var(--fg); background: var(--bg); }
}
@layer components {
  .button { padding: .5rem 1rem; border-radius: .5rem; }
}
@layer utilities {
  .text-center { text-align: center; }
}
```

## 색상 지정 방법 총정리 — 고전부터 최신까지

아래 표는 **지원 범위가 넓은 순서**로 나열했습니다. 최신 스펙은 폴백 전략을 함께 제시합니다.

| 방식 | 예시 | 특징/비고 |
|---|---|---|
| 키워드 | `red`, `rebeccapurple` | 의미는 불명확, 대략적 색만 필요할 때 |
| Hex | `#ff0000`, `#f00`, `#ffaa00cc` | 마지막 2자리 알파(8자리 hex) 지원 |
| `rgb()` | `rgb(255 0 0)`, `rgb(255,0,0)` | 공백 구분 문법 권장, 범위 0–255 |
| `rgba()` | `rgba(0,0,0,.5)` | 최근엔 `rgb(0 0 0 / 0.5)` 형태 선호 |
| `hsl()` | `hsl(0 100% 50%)` | 색조-채도-명도, 테마 조절에 유리 |
| `hwb()` | `hwb(0deg 0% 0% / 0.7)` | Hue-Whiteness-Blackness, 직관적(신규) |
| `lab()` | `lab(60% 20 30)` | 지각 균등, 컬러 레벨4 |
| `lch()` | `lch(60% 50 30)` | 명도-채도-색상, 색공간 균일 |
| `oklab()` | `oklab(60% 0.1 0.05)` | 최신 지각 균등 |
| `oklch()` | `oklch(0.7 0.16 30)` | **현업 추천**(자연스러운 대비/톤) |
| `color()` | `color(display-p3 1 0.3 0.1 / 0.8)` | P3 등 넓은 색역, 폴백 필요 |

### 실전 예시: 동일 톤을 다양한 표기법으로

```css
.box-hex  { background: #0ea5e9; }
.box-rgb  { background: rgb(14 165 233); }
.box-hsl  { background: hsl(198 84% 48%); }
.box-hwb  { background: hwb(198 6% 9% / 1); }
.box-lch  { background: lch(65% 55 230); }
.box-oklch{ background: oklch(0.68 0.13 230); } /* 추천 */
```

### 알파 채널 비교: `opacity` vs 컬러 알파

```css
/* 요소 전체 투명도 → 자식까지 영향 */
.card { opacity: .7; }

/* 컬러만 반투명 → 텍스트/자식은 영향 적음 */
.card-bg { background: rgb(0 0 0 / 0.7); }
```

## 색상 토큰과 CSS 변수(커스텀 프로퍼티)

### 디자인 토큰 선언

```css
:root {
  /* 의미 기반 토큰 */
  --color-bg: oklch(0.985 0.003 255);
  --color-fg: oklch(0.23 0.02 255);
  --color-primary: oklch(0.7 0.16 230);
  --color-primary-contrast: oklch(0.18 0.02 230);
  --color-muted: oklch(0.95 0.01 255);
}
```

### 사용

```css
body { color: var(--color-fg); background: var(--color-bg); }
.button {
  color: var(--color-primary-contrast);
  background: var(--color-primary);
  border: 1px solid color-mix(in oklch, var(--color-primary), black 20%);
}
```

### 상대 색상(Relative Color Syntax, 최신)

```css
/* 기준 색을 살짝 어둡게 변주 */
.card {
  background: oklch(from var(--color-primary) l calc(c - 0.03) h);
}
```

## 다크 모드, 자동 테마, 시스템 색상

### 다크/라이트 테마 전환

```css
:root {
  color-scheme: light dark; /* UA 스크롤바/폼 등 네이티브 요소 테마화 */
  --bg: oklch(0.98 0.003 255);
  --fg: oklch(0.20 0.02 255);
}
@media (prefers-color-scheme: dark) {
  :root {
    --bg: oklch(0.14 0.02 255);
    --fg: oklch(0.92 0.02 255);
  }
}
body { background: var(--bg); color: var(--fg); }
```

### 사용자 토글 + 저장

```html
<button id="themeToggle" aria-pressed="false">테마 전환</button>
<script>
  const key = 'theme';
  const stored = localStorage.getItem(key);
  if (stored) document.documentElement.dataset.theme = stored;

  themeToggle.addEventListener('click', () => {
    const cur = document.documentElement.dataset.theme === 'dark' ? 'light' : 'dark';
    document.documentElement.dataset.theme = cur;
    themeToggle.setAttribute('aria-pressed', String(cur === 'dark'));
    localStorage.setItem(key, cur);
  });
</script>
```
```css
:root[data-theme="dark"] { --bg: oklch(0.14 0.02 255); --fg: oklch(0.92 0.02 255); }
```

## 접근성: 대비, 색각 이상, 의미 전달

### 대비 비율(WCAG 2.x)

본문 텍스트 대비 기준:
- 일반 텍스트: **최소 4.5:1**
- 굵은 대형 텍스트(18.66px bold 이상 또는 24px 이상): **3:1**

상대 휘도 \(L\)과 대비비 \(C\):
$$
C = \frac{L_1 + 0.05}{L_2 + 0.05},\quad L_1 \ge L_2
$$

### 실제 검사 워크플로

- 색상 선택 시 **명도 중심 공간(OKLCH/LCH)** 에서 조정 → 더 예측 가능한 대비
- 디자인 시안 단계에서 **시뮬레이션(색각 이상 필터)** 및 **자동 테스트** 사용
- 링크/상태는 **색상만**으로 구분하지 말고, **밑줄, 아이콘, 패턴**을 병행

```css
/* 링크: 색 + 밑줄 */
a { color: var(--color-primary); text-decoration: underline; }
a:hover { text-decoration-thickness: 2px; }
```

## 그라디언트, 배경, 경계, 그림자, 블렌딩

### 그라디언트

```css
.hero {
  /* OKLCH 베이스와 mix로 자연스러운 그라디언트 */
  --c1: oklch(0.85 0.1 200);
  --c2: oklch(0.7 0.16 230);
  background: linear-gradient(135deg, var(--c1), var(--c2));
}
```

### 다중 배경과 패턴

```css
.card {
  background:
    radial-gradient(circle at 20% 20%, rgb(0 0 0 / 0.06), transparent 40%),
    linear-gradient(180deg, white, #f8fafc);
}
```

### 테두리와 그림자

```css
.panel {
  border: 1px solid color-mix(in oklch, var(--fg), transparent 85%);
  box-shadow: 0 10px 20px rgb(0 0 0 / 0.08);
  border-radius: 12px;
}
```

### 블렌딩/필터

```css
.overlay { mix-blend-mode: multiply; }
.photo { filter: saturate(1.1) contrast(1.05); }
```

## 타이포와 색의 조합: 가독성 강화

```css
:root {
  --fg: oklch(0.22 0.03 250);
  --muted-fg: oklch(0.45 0.02 255);
}
p { color: var(--fg); }
small, .meta { color: var(--muted-fg); }
```
- 본문은 충분한 대비, 보조 정보는 덜 강조된 색
- 줄 간격 `line-height: 1.6~1.8`, 최대 줄 길이 60–75자 권장

## 실무 색상 설계 워크플로

1) **기준 색상 공간 선택**: OKLCH 또는 LCH
2) **기본 팔레트 정의**: 배경/본문/강조/경고/성공/보조
3) **스케일 생성**: `lightness`를 기준으로 단계화(예: 95, 90, 80, 70, 60…)
4) **대비 검사**: 핵심 텍스트 조합은 4.5:1 이상
5) **토큰화**: `--color-primary`, `--color-bg` 등 의미 기반
6) **다크 모드**: 명도 반전이 아니라 **대비 유지**를 목표로 재조정
7) **컴포넌트 매핑**: 버튼/배지/알림/표/폼 상태에 토큰 연결
8) **자동화**: 컬러 제너레이터 스크립트(디자인 툴/빌드 단계) 도입

## 성능과 유지보수

- **Critical CSS 인라인** + 나머지 지연/비동기 로드
- **레이어(@layer)** 로 계층 제어, **특이성 최소화**
- **변수(var)** 중심 설계로 테마 전환/브랜딩 변경 비용 감소
- 재페인트/리플로우 비용이 큰 **필터/블러/대형 그림자**는 최소화

## 색 관련 주요 속성 요약

| 속성 | 설명 | 예시 |
|---|---|---|
| `color` | 글자색 | `color: oklch(0.22 0.03 250);` |
| `background`, `background-color`, `background-image` | 배경 | `background: linear-gradient(...)` |
| `border`, `outline` | 경계선 | `border: 1px solid color-mix(...);` |
| `box-shadow`, `text-shadow` | 그림자 | `box-shadow: 0 10px 26px rgb(0 0 0 / 0.08);` |
| `opacity` | 요소 전체 투명도 | `opacity: .8;` |
| `mix-blend-mode`, `background-blend-mode` | 블렌딩 | `mix-blend-mode: multiply;` |
| `filter` | 색/명암/블러 | `filter: contrast(1.05);` |
| `color-scheme` | 네이티브 UI 테마 힌트 | `color-scheme: light dark;` |

## 자주 하는 실수와 교정

1) **인라인 스타일 남발** → 외부 CSS + 토큰/레이어로 구조화
2) **색상만으로 상태 구분** → 아이콘/패턴/텍스트 병행
3) **명도/대비 고려 없이 색만 바꿈** → OKLCH로 명도·채도·각도 조절, 대비 검사
4) **opacity로 배경 반투명** → 텍스트까지 투명해짐 → 컬러 알파 사용
5) **다크 모드에 단순 반전** → 대비 유지, 링크/경계/휴먼 스킨톤 재조정
