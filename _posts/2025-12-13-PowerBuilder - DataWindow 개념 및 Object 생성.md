# DataWindow 개념 및 Object 생성

PowerBuilder의 가장 강력하고 독특한 기능은 단연 **DataWindow**입니다. DataWindow는 데이터베이스 연동, 화면 표시, 데이터 조작, 보고서 출력까지 하나의 객체에서 통합적으로 처리하는 기능입니다. 일반적인 개발 도구에서는 데이터 그리드, 입력 폼, 보고서를 각각 따로 만들어야 하지만, PowerBuilder의 DataWindow는 이 모든 것을 하나로 통합하여 개발 생산성을 획기적으로 높여줍니다.

---

## DataWindow란 무엇인가?

DataWindow는 다음과 같은 기능을 하나로 묶은 **지능형 데이터 객체**입니다.

- **데이터 소스(Data Source)**: 데이터를 가져올 위치(테이블, 저장 프로시저, 외부 파일 등) 정의
- **프레젠테이션 스타일(Presentation Style)**: 데이터를 화면에 어떻게 표시할지 결정 (Grid, Freeform, Tabular 등)
- **데이터 조작(Data Manipulation)**: 삽입, 갱신, 삭제, 필터링, 정렬 등의 기능 내장
- **데이터 유효성 검사(Validation)**: 컬럼 단위 입력 규칙, 드롭다운 리스트, 체크박스 등 정의 가능
- **출력(Printing)**: 보고서 형태로 인쇄, PDF 변환 등

DataWindow는 크게 두 부분으로 나뉩니다.

1. **DataWindow 객체**: 디자인 타임에 정의하는 실제 객체 (라이브러리 .pbl에 저장)  
   예: `d_customer_list`, `d_order_entry`
2. **DataWindow 컨트롤**: 윈도우에 배치하는 컨테이너로, DataWindow 객체를 연결하여 화면에 표시  
   예: `dw_list`, `dw_detail`

---

## DataWindow의 구성 요소

DataWindow는 다음과 같은 계층적 구조를 가집니다.

```
[DataWindow 객체]
   ├── [데이터 소스] : SQL SELECT, Stored Procedure 등
   ├── [프레젠테이션 스타일] : Grid, Freeform 등
   ├── [밴드(Band)] : 헤더, 상세, 요약, 푸터 등
   │     ├── [컨트롤] : 컬럼, 텍스트, 그림, 버튼 등
   │     └── [속성] : 폰트, 색상, 테두리 등
   └── [데이터 조작 규칙] : 정렬, 필터, 유효성 검사 등
```

### 밴드(Band)의 종류

| 밴드 | 설명 |
|------|------|
| **Header** | 각 페이지 상단에 반복 표시 (컬럼 제목 등) |
| **Detail** | 데이터 행이 반복되는 영역 (실제 데이터 표시) |
| **Footer** | 각 페이지 하단에 반복 표시 (페이지 번호 등) |
| **Summary** | 마지막 데이터 뒤에 한 번 표시 (합계 등) |
| **Group Header/Footer** | 그룹화 시 각 그룹의 시작/끝에 표시 |

## DataWindow 생성 단계

DataWindow 객체를 생성하는 과정을 단계별로 살펴보겠습니다.

### 1. DataWindow Painter 실행

- **File → New** (또는 도구 모음 New 아이콘)
- **DataWindow** 탭에서 원하는 프레젠테이션 스타일 선택 (예: Grid, Freeform 등)

### 2. 데이터 소스 선택

DataWindow가 데이터를 가져올 방식을 선택합니다.

| 데이터 소스 | 설명 |
|------------|------|
| **Quick Select** | 단일 테이블에서 간단히 컬럼 선택 (가장 쉬움) |
| **SQL Select** | 여러 테이블 조인, WHERE 조건, 그룹화 등을 그래픽으로 설정 |
| **Query** | 미리 저장된 Query 객체 사용 |
| **Stored Procedure** | 데이터베이스 저장 프로시저 실행 결과 사용 |
| **External** | 데이터베이스와 무관한 외부 데이터 (예: 텍스트 파일, 수동 입력) |

**SQL Select**를 선택하면 그래픽 쿼리 빌더가 열립니다.

```
[SQL Select 화면]
   ┌─────────────────────────────┐
   │ 테이블: customer, orders     │
   │ 조인 조건: customer.id = orders.cust_id │
   │ 선택 컬럼: customer.name,    │
   │           orders.order_date, │
   │           orders.amount      │
   │ WHERE: orders.amount > 1000  │
   │ ORDER BY: order_date DESC    │
   └─────────────────────────────┘
```

### 3. 컬럼 및 스타일 설정

데이터 소스를 정의한 후, DataWindow Painter로 넘어갑니다. 여기서 각 컬럼의 위치, 폰트, 배경색, 정렬 등을 조정하고, 추가 컨트롤(텍스트, 그림, 버튼 등)을 배치할 수 있습니다.

### 4. DataWindow 객체 저장

이름을 지정하여 저장합니다. 보통 `d_` 접두사를 사용합니다. 예: `d_customer_list`, `d_order_entry`.

## DataWindow 컨트롤과 연결 (껍데기와 알맹이)

생성한 DataWindow 객체를 윈도우에 표시하려면 DataWindow 컨트롤(껍데기)이 필요합니다. 이 과정을 정확히 이해해야 "DataObject를 설정하지 않아 데이터가 안 보이는" 오류를 방지할 수 있습니다.

### 단계 5: 윈도우에 DataWindow 컨트롤(껍데기) 배치

1. 윈도우를 열고 컨트롤 팔레트에서 **DataWindow** 아이콘을 선택합니다.
2. 윈도우에 적당한 크기로 컨트롤을 그립니다. (이름을 예를 들어 `dw_list`로 지정합니다.)
3. 속성 시트(Properties)에서 **DataObject** 항목을 찾습니다. 여기에 앞서 만든 DataWindow 객체(알맹이)인 `d_customer_grid`를 선택하여 연결(바인딩)합니다.

> 이 단계에서 DataObject를 설정하지 않으면, 아무리 코드를 작성해도 화면에 데이터가 표시되지 않습니다.

### 단계 6: 코드 작성 (컨트롤을 통해 객체 사용)

이제 윈도우의 스크립트에서 DataWindow 컨트롤을 통해 DataWindow 객체의 기능을 사용합니다. 컨트롤 이름(`dw_list`)을 사용하여 메서드를 호출합니다.

```powerbuilder
// 윈도우 Open 이벤트
// 껍데기(dw_list)에 DB 통신 객체(SQLCA)를 연결하고 데이터를 조회합니다.
dw_list.SetTransObject(SQLCA)
dw_list.Retrieve()
```

# 주요 프레젠테이션 스타일

프레젠테이션 스타일은 데이터를 어떤 형태로 화면에 표시할지 결정합니다. 각 스타일은 용도에 따라 장단점이 있으므로, 애플리케이션의 요구사항에 맞게 선택해야 합니다.

## 1. Grid 스타일

엑셀과 같은 스프레드시트 형태로 데이터를 표시합니다. **모든 셀에 테두리 선이 기본적으로 그어져 있고**, 사용자가 열 너비를 조절할 수 있는 바둑판 형태입니다.

### 특징
- 모든 컬럼이 고정된 열 너비를 가짐
- 사용자가 열 너비 조절 가능
- 컬럼 제목(헤더)이 항상 표시됨
- 스크롤 시 헤더는 고정되고 데이터만 스크롤됨

### 장점
- 많은 양의 데이터를 한눈에 보기 좋음
- 사용자에게 익숙한 인터페이스
- 열 정렬, 이동 등이 자유로움

### 단점
- 입력 폼으로는 부적합 (한 행에 너무 많은 컬럼이 있으면 가독성 떨어짐)
- 복잡한 레이아웃(예: 여러 줄 입력) 표현 어려움

### 사용 예
- 고객 목록, 주문 내역, 재고 현황 등 조회 화면
- 단순 데이터 그리드 형태의 리포트

### Grid 스타일 예시 다이어그램 (테두리 있음)

```
┌───────────┬────────────┬─────────────┬──────────┐
│ 고객 ID   │ 고객명     │ 전화번호    │ 등급     │
├───────────┼────────────┼─────────────┼──────────┤
│ C001      │ 홍길동     │ 02-1234-5678│ 골드     │
├───────────┼────────────┼─────────────┼──────────┤
│ C002      │ 김철수     │ 031-234-5678│ 실버     │
├───────────┼────────────┼─────────────┼──────────┤
│ C003      │ 이영희     │ 051-345-6789│ 골드     │
└───────────┴────────────┴─────────────┴──────────┘
```

## 2. Freeform 스타일

폼 형태로 데이터를 표시합니다. 각 컬럼이 자유롭게 배치되며, 마치 종이 서식처럼 디자인할 수 있습니다.

### 특징
- 컬럼 레이블과 데이터 필드가 쌍을 이루어 배치됨
- 하나의 행(레코드)만 화면에 표시 (상세 보기)
- 컬럼 위치를 자유롭게 조정 가능
- 여러 줄 텍스트, 그림, 버튼 등 다양한 컨트롤 추가 가능

### 장점
- 입력 화면에 최적화
- 시각적으로 보기 좋은 폼 디자인 가능
- 사용자 가이드에 적합

### 단점
- 한 번에 하나의 레코드만 보여줌 (여러 레코드 탐색은 별도 기능 필요)
- 데이터 조회용으로는 비효율적

### 사용 예
- 고객 정보 등록/수정 화면
- 계약서, 신청서 등 입력 폼
- 상세 정보 조회 화면

### Freeform 스타일 예시 다이어그램

```
┌─────────────────────────────────┐
│   [고객 정보]                    │
│                                   │
│   고객 ID  : C001                │
│   고객명   : 홍길동               │
│   전화번호 : 02-1234-5678         │
│   이메일   : hong@example.com     │
│   등급     : [●] 골드  [ ] 실버   │
│                                   │
│   [저장] [취소]                   │
└─────────────────────────────────┘
```

## 3. Tabular 스타일

Grid와 Freeform의 중간 형태로, **선 없이 데이터가 일정 간격으로 띄워져서** 행 단위로 나열되는 리포트 형태입니다. 컬럼 제목이 위에 있고 데이터 행이 세로로 나열됩니다.

### 특징
- 컬럼 제목(헤더)이 상단에 위치
- 데이터는 행 단위로 반복 표시
- 셀 테두리 없음 (텍스트만 나열)
- 행 높이를 자유롭게 조절 가능 (여러 줄 텍스트 표시 가능)

### 장점
- 여러 레코드를 보여주면서도 Grid보다 유연한 디자인 가능
- 복잡한 행 구조(예: 여러 줄 주소) 표현 가능
- 간단한 목록형 리포트에 적합

### 단점
- 열 간격 조정이 Grid만큼 자유롭지 않음
- 많은 컬럼을 표시하기 어려움

### 사용 예
- 간단한 명단, 제품 목록
- 주문 내역 요약
- 입력과 조회가 혼합된 화면

### Tabular 스타일 예시 다이어그램 (선 없음)

```
   고객 ID    고객명         전화번호
   ─────────────────────────────────
   C001       홍길동        02-1234-5678
   C002       김철수        031-234-5678
   C003       이영희        051-345-6789
```

## 4. 기타 프레젠테이션 스타일

### Label 스타일
- 우편물 라벨, 명함, 카드 형태로 데이터를 출력할 때 사용
- 여러 개의 레코드를 라벨 형식으로 배열

### N-Up 스타일
- 여러 개의 컬럼으로 데이터를 배열 (신문 칼럼처럼)
- 적은 수의 컬럼을 가진 데이터를 공간 효율적으로 표시

### Graph 스타일
- 데이터를 차트(막대, 선, 원형 등)로 표시
- 데이터 분석, 요약 보고서에 유용

### Crosstab 스타일
- 피벗 테이블 형태로 데이터를 교차 분석
- 행과 열이 동적으로 생성되는 보고서

### Composite 스타일
- 여러 DataWindow를 하나로 결합
- 마스터-디테일 리포트 등 복합 문서 생성

---

## 프레젠테이션 스타일 선택 가이드

| 스타일 | 주요 용도 | 특징 |
|--------|----------|------|
| **Grid** | 다중 레코드 조회, 단순 리스트 | 열 기반, 헤더 고정, 사용자 열 조절 가능, 테두리 있음 |
| **Freeform** | 단일 레코드 입력/상세 | 자유로운 배치, 폼 디자인 |
| **Tabular** | 간단한 목록, 입력/조회 혼합 | 행 기반, 유연한 행 디자인, 테두리 없음 |
| **Label** | 라벨, 명함 출력 | 정형화된 반복 출력 |
| **N-Up** | 다중 컬럼 목록 | 공간 효율적 |
| **Graph** | 데이터 시각화 | 차트 표시 |
| **Crosstab** | 교차 분석, 피벗 테이블 | 동적 행/열 |
| **Composite** | 복합 보고서 | 여러 DataWindow 조합 |

## DataWindow 생성 실전 예제

간단한 고객 목록을 Grid 스타일로 생성해 보겠습니다.

### 단계 1: 새 DataWindow 생성 (알맹이 만들기)
- New → DataWindow 탭 → Grid 선택
- 데이터 소스: Quick Select 선택

### 단계 2: 테이블과 컬럼 선택
- 테이블: `customer`
- 컬럼: `cust_id`, `cust_name`, `phone`, `grade`

### 단계 3: DataWindow 디자인
- 각 컬럼 제목을 한글로 변경 (예: cust_id → 고객ID)
- 폰트, 배경색, 정렬 등을 조정
- 필요하다면 추가 컨트롤(예: 합계 행) 배치

### 단계 4: 저장
- 이름: `d_customer_grid`

### 단계 5: 윈도우에 DataWindow 컨트롤(껍데기) 배치
- 윈도우를 열고 DataWindow 컨트롤을 화면에 그립니다. (이름: `dw_list`)
- `dw_list`의 속성 창에서 **DataObject** 항목에 앞서 만든 `d_customer_grid`를 선택하여 연결(바인딩)합니다.

### 단계 6: 코드 작성
```powerbuilder
// 윈도우 Open 이벤트
// 껍데기(dw_list)에 DB 통신 객체(SQLCA)를 연결하고 데이터를 조회합니다.
dw_list.SetTransObject(SQLCA)
dw_list.Retrieve()
```

이제 실행하면 고객 목록이 Grid 형태로 표시됩니다.

## DataWindow 프레젠테이션 스타일 변경

한 번 생성한 DataWindow 객체의 프레젠테이션 스타일은 변경할 수 없습니다. 만약 다른 스타일이 필요하면 새로 만들어야 합니다. 하지만, 디자인 화면에서 컨트롤을 자유롭게 추가/삭제/이동할 수 있으므로 Grid를 Tabular처럼 보이게 조정하는 것은 가능합니다.

---

## 결론

DataWindow는 PowerBuilder 개발의 핵심이며, 프레젠테이션 스타일을 통해 다양한 형태의 데이터 표현이 가능합니다. Grid는 목록형 조회에, Freeform은 입력 폼에, Tabular는 그 중간 형태에 적합합니다. 각 스타일의 특성을 이해하고 용도에 맞게 선택하면 효율적이고 사용자 친화적인 인터페이스를 구축할 수 있습니다.

**중요한 점**: DataWindow 객체(알맹이)와 DataWindow 컨트롤(껍데기)의 역할을 정확히 구분하고, 컨트롤의 DataObject 속성에 객체를 연결하는 과정을 잊지 않아야 합니다.
