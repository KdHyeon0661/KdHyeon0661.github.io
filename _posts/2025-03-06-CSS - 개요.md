# CSS란?

## 0. 개요: CSS란 무엇이며 왜 쓰는가

**CSS (Cascading Style Sheets)** 는 HTML 문서의 표시(스타일)를 정의하는 언어입니다.
HTML이 구조(문서의 의미/계층)를 담당한다면, CSS는 **색상, 여백, 레이아웃, 타이포그래피, 애니메이션** 등 **보이는 모든 것**을 통제합니다.

핵심 포인트:
- **Cascading(캐스캐이딩)**: 여러 규칙이 충돌할 때 **우선순위**(특이성과 선언 순서)로 결정
- **Separation of Concerns(관심사 분리)**: HTML(의미)와 CSS(표현)를 분리하면 유지보수성 증가
- **Responsive(반응형)**: 하나의 코드로 다양한 뷰포트/환경에 대응
- **Progressive Enhancement(점진적 향상)**: 핵심 경험은 단순하게, 가능한 곳에서만 고급 기능 적용

## 1. CSS 기본 문법과 선택자

### 1.1 룰셋의 구조

```css
선택자 {
  속성: 값;
}
```

예시:
```css
h1 {
  color: blue;
  font-size: 24px;
}
```

### 1.2 선택자(Selectors) 확장

- **기본 선택자**: 요소, 클래스, 아이디
```css
/* 요소 */
p { line-height: 1.7; }

/* 클래스 */
.card { border: 1px solid #ddd; }

/* ID (과사용 금지 권장) */
#site-header { position: sticky; top: 0; }
```

- **복합/관계 선택자**
```css
/* 자식(>) vs 후손(공백) */
ul.menu > li { list-style: none; }
ul.menu li a { text-decoration: none; }

/* 인접 형제(+) / 일반 형제(~) */
h2 + p { margin-top: 0; }
h2 ~ p.note { color: #666; }
```

- **속성 선택자**
```css
/* data-* 속성을 활용한 상태 스타일링 */
button[data-variant="danger"] { background: #c53030; color: white; }
input[type="email"] { border: 1px solid #aaa; }
a[href^="https://"]::after { content: "↗"; font-size: 0.875em; }
```

- **의사클래스/의사요소**
```css
/* 상태 */
a:hover { text-decoration: underline; }
input:focus-visible { outline: 2px solid #0ea5e9; }

/* 구조 */
li:nth-child(odd) { background: #fafafa; }
section:has(> h2) { padding-top: 2rem; }       /* 현대 CSS :has */

/* 의사요소 */
p::first-line { font-variant: small-caps; }
button::before { content: ""; display: inline-block; width: .5rem; height: .5rem; background: currentColor; border-radius: 50%; margin-right: .5rem; }
```

- **스코프와 레이어(현대 CSS)**
```css
/* @layer으로 CSS 충돌 제어(기본 → 컴포넌트 → 유틸리티 순) */
@layer base, components, utilities;

@layer base {
  html { color-scheme: light dark; }
  body { margin: 0; font-family: system-ui, sans-serif; }
}

@layer components {
  .btn { padding: .5rem 1rem; border-radius: .5rem; }
}

@layer utilities {
  .text-center { text-align: center; }
}

/* @scope : 특정 범위에만 스타일 한정 (실험/현대 구현 환경 확인) */
@scope (.article) {
  h2 { margin-top: 2rem; }
}
```

## 2. 캐스캐이딩, 특이성, 상속

### 2.1 캐스캐이딩(Cascade)

동일한 속성에 여러 규칙이 적용될 때, 다음 순서로 결정:
1) 사용자 에이전트(브라우저 기본) < 2) 사용자 스타일 < 3) 저자 스타일
그리고 `!important` > 인라인 스타일 > 일반 규칙, 마지막에 **선언 순서**가 결정적.

### 2.2 특이성(Specificity)

대략적 우선순위:
- ID 선택자 > 클래스/속성/의사클래스 > 요소/의사요소
```css
/* 같은 대상일 때 */
#notice { color: red; }         /* 더 강함 */
.notice { color: blue; }        /* 약함 */
p { color: green; }             /* 가장 약함 */
```

### 2.3 상속(Inheritance)

일부 속성(예: `color`, `font-family`, `line-height`)은 상속됨.
상속 여부는 문서화 참고. 필요 시 `inherit`, `initial`, `revert`, `unset` 활용.
```css
.article { color: #222; font-family: ui-serif, Georgia, serif; }
.article code { color: inherit; font-family: ui-monospace, SFMono-Regular, monospace; }
```

## 3. 박스 모델(Box Model)과 레이아웃 기초

### 3.1 박스 모델

요소는 **content + padding + border + margin**으로 구성.
```css
*,
*::before,
*::after {
  box-sizing: border-box; /* 넓이 계산 예측가능 */
}
```

### 3.2 display와 흐름

```css
/* 문맥에 맞는 display */
nav ul { display: flex; gap: 1rem; }
figure { display: grid; gap: .5rem; }
```

### 3.3 위치/포지셔닝/스택킹 콘텍스트

```css
.card {
  position: relative;          /* 자식 absolute의 기준 */
  z-index: 1;                  /* 스택 레벨 제어(새 스택 생성 조건 주의: transform 등) */
}

.tooltip {
  position: absolute;
  top: 100%; left: 0;
  transform: translateY(.5rem);
}
```

## 4. 단위, 색, 타이포그래피

### 4.1 단위

- 상대 단위: `em`, `rem`, `%`, `vw`, `vh`, `svh/dvh`(안전한 뷰포트 높이), `ch`, `lh`
- 절대 단위: `px`, `pt`, `cm` 등 (웹에선 `px` 중심)

```css
html { font-size: 100%; }     /* 보통 16px */
h1 { font-size: 2rem; }       /* 32px */
.container { width: min(90vw, 72rem); }
```

### 4.2 색상

현대 CSS에서는 `color()` / `lab()` / `lch()` / `oklch()` 등도 등장.
```css
:root {
  --primary: oklch(63% 0.18 240); /* 가독성 좋은 현대 색공간 */
}

a { color: var(--primary); }
```

### 4.3 타이포그래피

```css
body {
  font-family: system-ui, -apple-system, Segoe UI, Roboto, sans-serif;
  line-height: 1.6;
  letter-spacing: 0.01em;
}

p + p { margin-top: 1em; }
```

**가변 폰트(variable fonts)** 활용:
```css
@font-face {
  font-family: "InterVar";
  src: url("/fonts/Inter.var.woff2") format("woff2-variations");
  font-weight: 100 900;
  font-display: swap;
}

h1 { font-family: "InterVar", system-ui; font-weight: 700; }
```

## 5. 레이아웃: Flexbox와 Grid

### 5.1 Flexbox (1차원)

```css
.nav {
  display: flex;        /* 주축 정렬 */
  gap: 1rem;
  align-items: center;  /* 교차축 정렬 */
  justify-content: space-between;
}
```

실전 시나리오: 가로 메뉴, 버튼 그룹, 카드 정렬

### 5.2 Grid (2차원) + Subgrid

```css
.grid {
  display: grid;
  grid-template-columns: repeat(12, 1fr); /* 12-column */
  gap: 1rem;
}

.card-list {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(16rem, 1fr));
  gap: 1.25rem;
}

/* subgrid 활용(지원 브라우저 확인) */
.article {
  display: grid;
  grid-template-columns: 1fr 3fr;
}
.article > header,
.article > section {
  display: grid;
  grid-template-columns: subgrid;
  grid-column: 1 / -1;
}
```

## 6. 반응형 디자인: 미디어/컨테이너 쿼리

### 6.1 미디어 쿼리

```css
/* 브레이크포인트 예시 */
@media (max-width: 768px) {
  .nav { flex-wrap: wrap; }
  .sidebar { display: none; }
}
```

### 6.2 컨테이너 쿼리(Container Queries)

컴포넌트 **부모 컨테이너 크기**에 따라 스타일 변경:
```css
/* 컨테이너 지정 */
.card-list {
  container-type: inline-size;
  container-name: cards;
}

/* 컨테이너 조건 */
@container cards (width < 600px) {
  .card { grid-template-columns: 1fr; }
}
```

## 7. 상태/테마: 사용자 환경 고려

### 7.1 다크 모드

```css
:root {
  color-scheme: light dark;
  --bg: #ffffff;
  --fg: #111111;

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0b0b0b;

```

### 7.2 모션 감소

```css
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: .001ms !important; animation-iteration-count: 1 !important; transition-duration: .001ms !important; }
}
```

## 8. 전환과 애니메이션, 변형

### 8.1 transition

```css
.button {
  background: #2563eb; color: white;
  transition: background .2s ease, transform .15s ease;
}
.button:hover { background: #1d4ed8; transform: translateY(-2px); }
```

### 8.2 @keyframes

```css
@keyframes fade-in-up {
  from { opacity: 0; transform: translateY(6px); }
  to   { opacity: 1; transform: translateY(0); }
}

.card {
  animation: fade-in-up .35s ease both;
}
```

### 8.3 transform/clip-path/filter

```css
.thumb:hover { transform: scale(1.03); }
.hero { filter: saturate(1.1) contrast(1.05); }
.avatar { clip-path: circle(50%); }
```


## 16. 유지보수: 구조화/모듈화/문서화

- 디렉토리 제안
```
styles/
  ├─ base/        /* reset, normalize, html/body, typography */
  ├─ components/  /* 버튼, 카드, 모달 등 */
  ├─ layouts/     /* grid, header, footer, sidebar */
  ├─ pages/       /* 페이지별 오버라이드 */
  ├─ utilities/   /* spacing, display, text helpers */
  └─ index.css    /* @import 혹은 @layer로 통합 */
```

- 주석/도큐멘테이션 유지, 스토리북과 연계 가능

## 20. 실전 시나리오 예제

### 20.1 블로그 레이아웃: 헤더/사이드바/본문/푸터

HTML 구조 예시:
```html
<header class="site-header">
  <a class="logo" href="/">MyBlog</a>
  <nav class="nav">
    <a href="/posts">Posts</a>
    <a href="/tags">Tags</a>
    <a href="/about">About</a>
  </nav>
</header>

<main class="layout">
  <aside class="sidebar">
    <div class="widget">
      <h3>Categories</h3>
      <ul class="menu">
        <li><a href="/tags/css">CSS</a></li>
        <li><a href="/tags/js">JavaScript</a></li>
        <li><a href="/tags/devops">DevOps</a></li>
      </ul>
    </div>
  </aside>

  <article class="content article" itemscope itemtype="https://schema.org/Article">
    <header>
      <h1 itemprop="headline">CSS란? — 핵심부터 실전까지</h1>
      <p class="meta" itemprop="author">by Do Hyun Kim</p>
    </header>
    <section itemprop="articleBody">
      <p>여기에 본문이 들어갑니다...</p>
    </section>
  </article>
</main>

<footer class="site-footer">© 2025 MyBlog</footer>
```

CSS:
```css
@layer base, layout, components, utilities;

/* base */
@layer base {
  :root {
    color-scheme: light dark;
    --bg: #ffffff;
    --fg: #111111;
    --muted: #6b7280;
    --brand: oklch(63% 0.18 240);
    --space-1: .25rem; --space-2: .5rem; --space-3: 1rem; --space-4: 1.5rem; --space-6: 2rem;
    --radius: .75rem;
    --max-w: 72rem;
  }
  @media (prefers-color-scheme: dark) {
    :root { --bg: #0b0b0b; --fg: #e6e6e6; --muted: #9ca3af; }
  }

  * { box-sizing: border-box; }
  html, body { height: 100%; }
  body { margin: 0; background: var(--bg); color: var(--fg); font-family: system-ui, sans-serif; line-height: 1.6; }
  img, svg, video { max-inline-size: 100%; block-size: auto; }
  a { color: var(--brand); text-decoration: none; }
  a:hover { text-decoration: underline; }
}

/* layout */
@layer layout {
  .site-header, .site-footer {
    position: sticky; inset-inline: 0;
    background: color-mix(in oklch, var(--bg) 90%, black 10%);
    backdrop-filter: blur(6px);
  }
  .site-header { top: 0; padding: var(--space-2) var(--space-3); display: flex; align-items: center; justify-content: space-between; }
  .site-footer { bottom: 0; padding: var(--space-3); text-align: center; color: var(--muted); }

  .logo { font-weight: 800; color: var(--fg); }
  .nav { display: flex; gap: var(--space-3); align-items: center; }

  .layout {
    display: grid;
    grid-template-columns: 16rem minmax(0, 1fr);
    gap: var(--space-6);
    width: min(90vw, var(--max-w));
    margin-inline: auto;
    padding: var(--space-6) var(--space-3);
    container-name: page; container-type: inline-size;
  }

  @container page (width < 800px) {
    .layout { grid-template-columns: 1fr; }
    .sidebar { order: 2; }
  }

  .sidebar { position: sticky; top: calc(2.5rem + var(--space-3)); align-self: start; }
  .content.article { min-height: 60vh; display: grid; gap: var(--space-4); }
  .content.article > header { display: grid; gap: var(--space-2); }
  .meta { color: var(--muted); font-size: .9rem; }
}

/* components */
@layer components {
  .widget {
    border: 1px solid color-mix(in oklch, var(--fg) 10%, transparent);
    border-radius: var(--radius);
    padding: var(--space-3);
    display: grid; gap: var(--space-2);
  }
  .menu { margin: 0; padding: 0; list-style: none; display: grid; gap: .25rem; }
  .menu a { display: inline-block; padding: .25rem .5rem; border-radius: .375rem; }
  .menu a:hover { background: color-mix(in oklch, var(--fg) 8%, transparent); }

  .btn {
    display: inline-flex; align-items: center; gap: .5rem;
```

## 23. 마이그레이션/정리 전략: “기존 CSS 블로그를 재정리” 로드맵

당신의 요청은 “CSS로 쓴 블로그를 **재정리**” 하는 것입니다. 다음 단계로 진행하기 좋습니다.

1) **자산 수집**
   - CSS 파일/부분 스타일/인라인 스타일 파악
   - 공통 패턴(색/여백/타입 스케일) 추출 → 변수화

2) **구조 재편**
   - `@layer`와 디렉토리 구조 도입: base/layout/components/utilities
   - 컴포넌트별 모듈화(카드/버튼/폼/네비게이션/페이지 섹션 등)

3) **현대 기능 도입**
   - `container queries`, `:has`, `@layer`, `color-mix`, `oklch`
   - 가능 범위에서 **점진적 향상**, 폴백 제공

4) **접근성/반응형 강화**
   - 포커스 스타일, 대비, `prefers-reduced-motion`, 다크 모드
   - 컨테이너 기반 반응형으로 **컴포넌트 재사용성 상승**

5) **성능 최적화**
   - 불필요한 중첩/과한 특이성 제거
   - 크리티컬 CSS 인라인 + 나머지 지연 로드

6) **테스트/디버그**
   - 주요 브라우저/디바이스 스냅샷
   - DevTools 그리드/Flex 오버레이로 레이아웃 확인
   - Lighthouse/Axis DevTools로 명도/접근성 확인

## 24. 체크리스트 요약

- 선택자/특이성/상속 규칙을 **명확히** 이해하고 적용
- **박스 모델/타입 스케일/색 시스템**을 변수로 표준화
- **Flex/Grid/컨테이너 쿼리**로 반응형 설계
- **다크 모드/감소 모션/포커스/대비**로 접근성 강화
- **@layer/@scope/:has/subgrid** 등 현대 기능은 지원 환경을 고려해 **점진적으로 도입**
- **도구체인**과 **린팅/포매팅**으로 일관성 확보
- 성능은 **단순 선택자/transform/opacity 중심 전환**으로 개선
