# CSS로 구현한 재밌는 UI 요소

## 공통 가이드

- 가능하면 **상태 = CSS 선택자**로 표현합니다. `:checked`, `:focus`, `:target`, `:has()`, `:valid` 등.
- 애니메이션은 **컴포지트 친화 속성**(`transform`, `opacity`)을 우선합니다.
- 색·거리·반지름 등은 **CSS 변수**로 추출해서 테마·다크모드·토큰과 자연스럽게 연결합니다.
- 접근성: 가능한 경우 **네이티브 컨트롤**(checkbox, radio, details)을 래핑하고, `aria-*`를 병행합니다.

```css
:root{
  --primary: #2563eb; /* 파랑 */
  --surface: #ffffff;
  --surface-2: #f5f7fb;
  --fg: #111827;
  --muted: #6b7280;
  --radius: 12px;
  --ring: 3px;
}
```

## 1. 토글 스위치 (checkbox + :checked)

핵심: `<input type="checkbox">`을 숨기고, 형제 `.slider`를 상태에 따라 스타일링.

```html
<label class="switch">
  <input type="checkbox" aria-label="설정 토글">
  <span class="slider" aria-hidden="true"></span>
  <span class="sr">설정</span>
</label>
```

```css
.switch{position:relative;display:inline-grid;place-items:center;width:56px;height:32px}
.switch input{position:absolute;opacity:0;inset:0;cursor:pointer}
.switch .slider{
  position:absolute;inset:0;border-radius:9999px;background:#d1d5db;transition:.25s;
}
.switch .slider::before{
  content:"";position:absolute;inset:4px auto auto 4px;width:24px;height:24px;
  background:#fff;border-radius:50%;transition:.25s;
}
.switch input:checked + .slider{background:var(--primary)}
.switch input:checked + .slider::before{transform:translateX(24px)}
/* 스크린리더만 읽히게 */
.sr{position:absolute;clip:rect(0,0,0,0);clip-path:inset(50%);height:1px;width:1px;overflow:hidden;white-space:nowrap}
```

활용: 다크모드 토글, 고급 옵션 온/오프

## 2. 아코디언 (details/summary)

네이티브 컨트롤로 키보드·접근성·애니메이션까지 안정적.

```html
<details class="acc">
  <summary>자주 묻는 질문 1</summary>
  <div class="acc__panel">답변 내용입니다.</div>
</details>
```

```css
.acc{border:1px solid #e5e7eb;border-radius:var(--radius);padding:.25rem .75rem;background:var(--surface)}
.acc[open]{box-shadow:0 0 0 var(--ring) color-mix(in oklch, var(--primary), #fff 80%)}
.acc summary{cursor:pointer;font-weight:600;list-style:none;padding:.5rem 0}
.acc summary::-webkit-details-marker{display:none}
.acc__panel{color:var(--muted);padding:.25rem 0 .75rem;animation:fade .2s ease}
@keyframes fade{from{opacity:0;transform:translateY(-4px)}}
```

## 3. 로딩 애니메이션 세트 (스피너, 도트, 스켈레톤)

CSS만으로 시각적 대기 상태 제공.

```html
<div class="loader"></div>
<div class="dots" aria-label="로딩중" role="status">
  <span></span><span></span><span></span>
</div>
<div class="skeleton" style="width:220px;height:16px"></div>
```

```css
.loader{
  border:4px solid #eef2ff;border-top-color:var(--primary);
  width:40px;height:40px;border-radius:50%;animation:spin .8s linear infinite
}
@keyframes spin{to{transform:rotate(360deg)}}

.dots{display:inline-flex;gap:6px;align-items:center}
.dots span{width:8px;height:8px;border-radius:50%;background:#c7d2fe;animation:bounce 1s infinite}
.dots span:nth-child(2){animation-delay:.15s}
.dots span:nth-child(3){animation-delay:.3s}
@keyframes bounce{50%{transform:translateY(-6px)}}

.skeleton{
  --sk:#e5e7eb;--hl:#f3f4f6;border-radius:8px;background:
  linear-gradient(90deg,var(--sk),var(--hl),var(--sk)) 0/200% 100%;
  animation:shimmer 1.2s infinite
}
@keyframes shimmer{to{background-position:200% 0}}
```

## 4. 카드 플립 (3D transform)

```html
<div class="flip">
  <div class="flip__inner">
    <div class="flip__face flip__face--front">앞면</div>
    <div class="flip__face flip__face--back">뒷면</div>
  </div>
</div>
```

```css
.flip{width:220px;height:140px;perspective:1000px}
.flip__inner{position:relative;width:100%;height:100%;transform-style:preserve-3d;transition:transform .6s}
.flip:hover .flip__inner{transform:rotateY(180deg)}
.flip__face{position:absolute;inset:0;display:grid;place-items:center;border-radius:12px;background:#f8fafc;backface-visibility:hidden}
.flip__face--back{transform:rotateY(180deg);background:#eef2ff}
```

## 5. 툴팁 (hover/focus, prefers-reduced-motion 고려)

```html
<button class="tip" aria-describedby="t1">도움말</button>
<span id="t1" class="tip__box" role="tooltip">툴팁 설명입니다</span>
```

```css
.tip{position:relative}
.tip__box{
  position:absolute;inset:auto auto calc(100% + 8px) 50%;
  transform:translateX(-50%);background:#111;color:#fff;padding:.375rem .5rem;
  border-radius:6px;white-space:nowrap;opacity:0;pointer-events:none;transition:opacity .15s;
}
.tip:is(:hover,:focus) + .tip__box{opacity:1}
@media (prefers-reduced-motion: reduce){.tip__box{transition:none}}
```

## 7. 다크모드 토글 (CSS 변수 + checkbox only 대안)

JS 없이도 가능하지만, 사용자 선호 기억은 JS 권장. 여기서는 CSS-only 데모.

```html
<input id="dm" type="checkbox" class="dm-visually-hidden">
<label for="dm" class="dm-toggle">다크모드</label>
<div class="dm-demo">본문</div>
```

```css
.dm-visually-hidden{position:absolute;opacity:0;width:1px;height:1px}
:root{--bg:#fff;--tx:#111}
#dm:checked ~ .dm-demo{--bg:#0b1020;--tx:#e5e7eb}
.dm-demo{background:var(--bg);color:var(--tx);padding:1rem;border-radius:12px}
.dm-toggle{display:inline-block;padding:.5rem .75rem;border:1px solid #ddd;border-radius:8px;cursor:pointer}
@media (prefers-color-scheme: dark){
  :root{--bg:#0b1020;--tx:#e5e7eb}
}
```

## 15. 이미지 비교 슬라이더 (range 스타일링)

```html
<div class="cmp">
  <img src="before.jpg" alt="" class="cmp__img">
  <img src="after.jpg" alt="" class="cmp__img cmp__img--top" style="--w:50%">
  <input type="range" class="cmp__range" min="0" max="100" value="50" aria-label="비교 슬라이더">
</div>
```

```css
.cmp{position:relative;max-width:640px;aspect-ratio:16/9}
.cmp__img{position:absolute;inset:0;object-fit:cover;width:100%;height:100%}
.cmp__img--top{clip-path:inset(0 calc(100% - var(--w)) 0 0)}
.cmp__range{position:absolute;inset:auto 0 12px 0;width:100%}
```

> CSS만으로는 슬라이더 값에 맞춰 `--w`를 변경하기 어렵습니다.
> 순수 CSS 데모가 목적이면 `:hover` 구간별 마커 + `:has()`로 단계형 비교를 구성할 수 있고, 실제 제품에는 한 줄 JS로 `--w` 동기화하는 것을 권장합니다.

## 18. 스크롤 진행 표시줄 (scroll-timeline 기반, 점진적)

지원 브라우저에서는 CSS만으로 구성. 미지원에서는 `position: sticky` + width 계산 JS 대체 가능.

```html
<div class="progress"></div>
<main id="content"> ... 긴 내용 ... </main>
```

```css
@scroll-timeline read{
  source:selector(#content);
  scroll-offsets: start 0, end 100%;
}
.progress{
  position:fixed;inset:0 auto auto 0;height:4px;background:var(--primary);
  width:0;animation:grow 1s linear both;animation-timeline:read;animation-range:0% 100%;
}
@keyframes grow{to{width:100vw}}
```

## 접근성·성능·호환성 체크리스트

- 포커스 가능 요소에는 **키보드 경로**와 `:focus-visible` 스타일을 제공합니다.
- 애니메이션은 **reflow 트리거**(top/left/width/height) 대신 `transform/opacity`를 사용합니다.
- `details/summary`, `radio/checkbox`, `range` 등 **네이티브 입력**을 래핑하여 스크린리더 호환을 담보합니다.
- 최신 기능(`:has`, `@scroll-timeline`, `container-type`)은 **보조적 향상**(progressive enhancement) 영역으로 두고 폴백을 설계합니다.

```css
/* 포커스 링 공통 */
:where(a,button,[role="tab"],summary,label,input):focus-visible{
  outline: var(--ring) solid color-mix(in oklch, var(--primary), #fff 70%);
  outline-offset: 2px;
}
```

## 마무리

- 상태는 `:checked`, `:target`, `:has()` 같은 **CSS 선택자**로 표현할수록 단순하고 견고합니다.
- 애니메이션은 **합성 단계**에 머물게 설계하고, `prefers-reduced-motion`를 존중합니다.
- 네이티브 컨트롤을 적절히 래핑하면 **접근성·키보드·모바일**까지 자연스럽게 해결됩니다.
- 실무에서는 위 컴포넌트들을 **토큰화된 변수/레이어** 체계에 편입하여 디자인 시스템과 일관되게 운영하세요.

이 모음은 스니펫으로 바로 복붙 가능하도록 구성했습니다.
필요 시 각 컴포넌트를 BEM 또는 유틸리티 클래스 전략에 맞춰 네이밍만 정리해 사용하면 됩니다.
