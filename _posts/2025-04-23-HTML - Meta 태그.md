# Meta 태그 심화 정리

## 1. 핵심 요약: `<head>` 최소 안전 세트

```html
<head>
  <!-- 1. 문자인코딩: 가장 먼저! -->
  <meta charset="UTF-8">

  <!-- 2. 호환성: (구)IE용. 오늘날 사실상 불필요하지만 레거시 호환 시 남김 -->
  <meta http-equiv="X-UA-Compatible" content="IE=edge">

  <!-- 3. 반응형: 모바일 필수 -->
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- 4. SEO: 50~60자 내의 <title> + 150~160자 description 권장 -->
  <title>문서 제목 — 사이트명</title>
  <meta name="description" content="이 페이지에 대한 150~160자 내외의 핵심 요약. 검색 결과 미리보기(스니펫)에 사용됩니다.">

  <!-- 5. 색상/모드: 브라우저 UI 색 & 다크모드 힌트 -->
  <meta name="theme-color" content="#0ea5e9" media="(prefers-color-scheme: light)">
  <meta name="theme-color" content="#0b7285" media="(prefers-color-scheme: dark)">
  <meta name="color-scheme" content="light dark">

  <!-- 6. 소셜 미리보기(OG + Twitter) -->
  <meta property="og:title" content="문서 제목 — 사이트명">
  <meta property="og:description" content="간결하고 매력적인 설명.">
  <meta property="og:type" content="article">
  <meta property="og:url" content="https://example.com/path">
  <meta property="og:image" content="https://example.com/og/cover.jpg">
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="문서 제목 — 사이트명">
  <meta name="twitter:description" content="간결하고 매력적인 설명.">
  <meta name="twitter:image" content="https://example.com/og/cover.jpg">

  <!-- 7. 색인/링크 추적(기본값: index,follow) → 상황 따라 조정 -->
  <meta name="robots" content="index,follow">

  <!-- 8. 리퍼러 정책: 프라이버시/분석 균형 -->
  <meta name="referrer" content="strict-origin-when-cross-origin">
</head>
```

> **순서 중요**: `charset`은 `<head>` 최상단에 두어야 파서가 즉시 올바른 인코딩을 적용합니다.

## 2. 기본 구조와 타입

```html
<!-- name + content 쌍 -->
<meta name="설명" content="값">

<!-- 문자셋 -->
<meta charset="UTF-8">

<!-- HTTP 헤더 유사 설정(일부만 유효) -->
<meta http-equiv="refresh" content="5; url=/moved">
```

**권장 원칙**
- 가능한 설정은 **HTTP 응답 헤더**에서 처리(CSP, 캐시 등).  
- `<meta>`는 문서-단위 힌트나 클라이언트 측 정보 제공에 집중.

## 3. SEO 관련 메타

### 3.1 `description`
```html
<meta name="description" content="150~160자 권장. 페이지 핵심 가치를 자연어로 서술. 키워드 나열 금지.">
```

- 검색 스니펫 후보. 길이 과하면 잘림.
- 페이지마다 **고유하게** 작성.

### 3.2 `robots` & 친구들
```html
<!-- 기본 허용 -->
<meta name="robots" content="index, follow">

<!-- 비공개/개발 중 페이지: 색인 금지 -->
<meta name="robots" content="noindex, nofollow">

<!-- Googlebot 전용 세분화 -->
<meta name="googlebot" content="noarchive, nosnippet">
<meta name="bingbot" content="max-snippet:160">
```

**주의**
- `noindex` 페이지가 **외부에서 강하게 링크**되면 캐노니컬/크롤 정책 상충이 생길 수 있음. 의도를 명확히.

### 3.3 캐노니컬/다국어(참고)
- **캐노니컬**은 `<link rel="canonical" href="...">` (메타가 아님).
- **다국어**는 `<link rel="alternate" hreflang="...">` 사용(역시 메타 아님).

## 4. 반응형 & 모바일 UX

### 4.1 `viewport`
```html
<!-- 기본 -->
<meta name="viewport" content="width=device-width, initial-scale=1">

<!-- 확대 제한(접근성 이슈 유의) -->
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">

<!-- 데스크톱 우선 사이트의 임시 대응(권장 X) -->
<meta name="viewport" content="width=1280">
```

### 4.2 iOS 포맷 감지 방지
```html
<!-- 전화번호 자동 링크 방지 (iOS Safari) -->
<meta name="format-detection" content="telephone=no">
```

### 4.3 색·모드
```html
<meta name="color-scheme" content="light dark">       <!-- 양쪽 테마 보유 -->
<meta name="theme-color" content="#0ea5e9" media="(prefers-color-scheme: light)">
<meta name="theme-color" content="#0b7285" media="(prefers-color-scheme: dark)">
```

## 5. 소셜 공유 미리보기(Open Graph / Twitter)

### 5.1 Open Graph(페북/카톡/디스코드 등)
```html
<meta property="og:title" content="제목">
<meta property="og:description" content="요약. 110자 내외 추천.">
<meta property="og:type" content="article"> <!-- website/article/product 등 -->
<meta property="og:url" content="https://example.com/post/123">
<meta property="og:image" content="https://example.com/og/cover.jpg">
<meta property="og:site_name" content="사이트명">
<meta property="og:locale" content="ko_KR">
```

### 5.2 확장(기사형)
```html
<meta property="article:published_time" content="2025-11-09T10:00:00+09:00">
<meta property="article:modified_time" content="2025-11-09T12:15:00+09:00">
<meta property="article:author" content="https://example.com/about">
<meta property="article:tag" content="HTML">
```

### 5.3 Twitter Card
```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="제목">
<meta name="twitter:description" content="요약">
<meta name="twitter:image" content="https://example.com/og/cover.jpg">
<meta name="twitter:site" content="@account"> <!-- 선택 -->
```

**팁**
- `og:url`은 **정규 URL** 1개로 고정.
- 이미지 1200×630(1.91:1) 전후, 용량 1MB 이내 권장.

## 6. 보안·프라이버시 관련 메타

### 6.1 Content Security Policy (가능하면 **HTTP 헤더**로)
```html
<!-- 메타로 간이 적용: 일부 한계가 있어 헤더 권장 -->
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; img-src 'self' https: data:; script-src 'self' 'unsafe-inline' https:">
```
- **권장**: 서버에서 `Content-Security-Policy` 응답 헤더로 제공 → 초기 파싱부터 강제.
- **메타 한계**: 외부 스크립트 로드 전에 도달하지 못할 수 있음.

### 6.2 Referrer Policy
```html
<meta name="referrer" content="strict-origin-when-cross-origin">
```
- 개인정보 보호와 분석 유효성 균형. 사이트 성격에 맞게 `no-referrer`~`origin-when-cross-origin` 선택.

### 6.3 기타(헤더 전용 사항)
- **Permissions-Policy**, **COOP/COEP**, **X-Frame-Options** 등은 **메타가 아닌 헤더**로만 설정.

## 7. 캐시/리다이렉트/호환 (http-equiv)

```html
<!-- (권장X) 메타 리다이렉트. 서버 301/302 권장 -->
<meta http-equiv="refresh" content="5; url=/new-location">

<!-- 캐시 제어: 실제로는 HTTP 헤더 Cache-Control 권장 -->
<meta http-equiv="Cache-Control" content="no-cache">

<!-- X-UA-Compatible: 레거시 IE. 최신 프로젝트는 생략 가능 -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">
```

## 8. 앱/스토어/딥링크·PWA 관련

```html
<!-- PWA: manifest는 link로 -->
<link rel="manifest" href="/manifest.webmanifest">

<!-- 모바일 웹앱 (과거 iOS) -->
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="application-name" content="앱이름">

<!-- 스토어 딥링크 -->
<meta name="apple-itunes-app" content="app-id=1234567890, affiliate-data=..., app-argument=https://example.com/app">
<meta name="google-play-app" content="app-id=com.example.app">
```

> 최신 iOS/안드로이드에서는 **Web App Manifest + Service Worker** 중심으로 설계.

## 11. 자주 하는 실수 & 체크리스트

| 실수 | 왜 문제인가 | 올바른 대응 |
|---|---|---|
| `charset`이 늦게 등장 | 초기 파싱 오류/깨짐 | `<head>` 첫 줄 근처 배치 |
| `description` 중복 | 스니펫 품질 저하 | 페이지마다 고유 요약 |
| `og:url` 각각 다름 | 중복 콘텐츠 인식 | 정규 URL 1개로 고정 |
| `noindex`와 강한 내부 링크 혼용 | 색인 정책 혼선 | 개발/프리뷰만 noindex |
| `refresh` 메타로 영구 이동 | SEO 가치 손실 | 서버 301/308 리다이렉트 |
| CSP를 메타만으로 | 초기 로드 구간 취약 | **HTTP 헤더로 CSP 설정** |
| 다크모드 이미지 미스매치 | 소셜 미리보기 대비 부족 | 이미지 대비/가독성 점검 |
| 뷰포트 확대 차단 | 접근성 저해 | `maximum-scale` 남용 금지 |

**퀵 체크**
- [ ] `charset` 맨 위, `viewport` 포함  
- [ ] 페이지 고유 `title/description`  
- [ ] OG/Twitter 세트 + 정규 `og:url`  
- [ ] `theme-color` 다크/라이트 대응  
- [ ] 민감 정책은 헤더(CSP/COOP/Permissions)  
- [ ] 프라이버시 `referrer` 정책 지정

## 14. 메타 vs 헤더 — 어떤 걸 쓰나?

| 항목 | 메타(`<meta>`) | HTTP 헤더 |
|---|---|---|
| 문자셋 | 가능(`charset`) | 가능(`Content-Type`) |
| SEO/소셜 | **메타** | 일부 헤더(없음) |
| CSP/보안정책 | 제한적(권장 X) | **권장** |
| 캐시/리디렉트 | 제한적(권장 X) | **권장** |
| 리퍼러 정책 | 가능(`meta name="referrer"`) | 가능(`Referrer-Policy`) |

> 원칙: **정책/보안/캐시/리다이렉트 → 헤더**,  
> **문서 요약·뷰포트·소셜 미리보기 → 메타**.

## 결론

- `<meta>`는 **보이지 않지만 영향은 크다**: SEO, 소셜 미리보기, 반응형, 색상/테마, 프라이버시.  
- **보안·정책·캐시**는 가급적 **HTTP 헤더**로.  
- 페이지마다 **고유한 `title/description/og`**를 유지하고,  
  다크/라이트·모바일 **현대 UX 신호(meta)**를 제공하라.

**점검 한 줄**: _“charset 맨 위, viewport OK, 고유한 SEO·OG, theme-color/ color-scheme, robots/referrer, 정책은 헤더로.”_
