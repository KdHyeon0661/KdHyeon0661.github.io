# 왜 디지털 신호처리인가 — 개관의 목적

이 문서는 “DSP 전체 가이드”의 **개관 파트**로서, 아래 세 가지를 간결하면서도 충분히 깊게 다룹니다.

1. **역사**: 통신·컴퓨팅·반도체 발전과 맞물린 DSP의 형성과 전개
2. **응용**: 오디오·영상·의료·레이다/소나·산업계 센서 데이터 분석 등 실제 문제
3. **GNU Octave**: MATLAB 대체로 무료/오픈소스 기반 실습 환경 정비 및 첫 실행

이후의 심화 실습(샘플링/DFT·FFT/윈도잉/필터/STFT 등)은 이 개관을 전제로 확장됩니다.

---

## 역사: 핵심 사건 타임라인으로 읽는 DSP

DSP 역사는 **정보이론·수치해석·반도체**의 세 축이 만나는 지점에서 진화했습니다. 사건을 시간순으로 짚으며 “왜 지금의 도구와 방법이 표준이 되었는가”를 정리합니다.

### 1940s–1950s: 신호·정보 이론의 토대

- **샤논(1948)**: 채널 용량·표본화 등 정보이론 정립. 아날로그를 디지털로 바꿔 다룰 수 있는 이론적 기반 확립.
- **나이퀴스트·샤논 표본화 정리**: 최대 주파수 \( f_{\max} \)를 포함하는 대역제한 신호는 \( F_s \ge 2 f_{\max} \)로 샘플링 시 원 복원이 가능.
  $$ F_s \ge 2 f_{\max} $$

### 1960s: FFT의 등장과 디지털 처리 실용화의 문

- **Cooley–Tukey FFT(1965)**: DFT의 계산복잡도를 \(O(N^2)\)→\(O(N\log N)\)으로 혁신. 스펙트럼 분석·콘볼루션 가속이 현실화.
  DFT 정의:
  $$
  X[k] = \sum_{n=0}^{N-1} x[n] e^{-j\frac{2\pi}{N}kn}
  $$

### 1970s: 전용 DSP 하드웨어와 필터 이론의 정비

- **전용 DSP 칩(초기형, 후에 TI TMS320 계열로 대중화)**: 실시간 필터링·모뎀·음성 코덱 구현이 가능해짐.
- **디지털 필터 이론 체계화**: IIR/FIR 설계, 아날로그-디지털 변환 기법(Bilinear Transform 등) 정착.

### 1980s: 공학 소프트웨어와 표준 신호처리 교과서

- **MATLAB(초창기)**: 행렬/벡터 기반 수치계산 환경이 교육·연구 표준으로 확산.
- **DSP 교과과정 표준화**: 아날로그→디지털 전환 교육이 공학대학의 공통 커리큘럼으로 자리 잡음.

### 1990s: 멀티레이트·웨이블릿·압축 표준

- **멀티레이트 처리·폴리페이즈 구조**: 샘플레이트 변환(업/다운샘플)과 필터뱅크 고도화.
- **웨이블릿**: 비정상 신호 분석·압축에 강점.
- **압축 표준(MPEG/JPEG/MP3 등)**: 산업계에서 DSP가 가치를 입증.

### 2000s: 소프트웨어 정의 무선(SDR)과 임베디드

- **SDR**: 하드웨어 대신 소프트웨어로 변조/복조·필터·동기화 구현.
- **임베디드**: 휴대기기·자동차 ECU·산업용 컨트롤러에 DSP 소프트웨어가 대거 탑재.

### 2010s: 대규모 데이터·딥러닝 접점

- **대규모 센서 데이터**: IoT/웨어러블/산업 설비에서 시계열 처리와 이상탐지가 필수.
- **딥러닝×DSP**: 전처리(필터링·STFT·멜스펙트럼)와 학습형 특성추출의 결합.

### 2020s–: 엣지 AI·혼합 워크플로

- **엣지 장치**: 제한된 연산/전력에서의 DSP+경량 모델 결합.
- **오픈소스 생태계**: MATLAB의 유료 라이선스 부담 없이도 **GNU Octave**·Python(SciPy/NumPy)로 교육·실무 프로토타이핑이 가능.

## 응용: “문제–신호–방법–지표”의 관점으로 보는 사례

현업에서는 **문제 정의 → 신호 특성 파악 → DSP 방법 선택 → 품질 지표**의 순서로 접근합니다. 분야별로 핵심 시나리오를 정리합니다.

### A. 오디오·음성

- **문제**: 잡음 제거, 에코/리버브 억제, 톤 보정, 음성 특징 추출(MFCC)
- **방법**: 저역/대역저지 필터, STFT 기반 스펙트럼 감쇠, 멜필터뱅크
- **지표**: SNR, PESQ, STOI
- **간단한 예**: 50/60 Hz 전원 험 제거 → 대역저지(notch) 필터

```octave
pkg load signal
Fs = 16000; t = (0:1/Fs:2-1/Fs)';
x  = sin(2*pi*440*t) + 0.1*sin(2*pi*60*t) + 0.02*randn(size(t));
% 60 Hz ±1 Hz 대역저지 FIR
b_bs = fir1(256, [59 61]/(Fs/2), 'stop', hamming(257));
y    = filtfilt(b_bs, 1, x);     % 위상왜곡 최소화를 위해 양방향 필터링
```

### B. 영상·이미지(기초 신호 처리 관점)

- **문제**: 경계 강화, 노이즈 억제, 주파수 영역 마스킹
- **방법**: 2D FIR(가우시안 스무딩), 주파수 도메인 필터링(DFT/DFTshift)
- **지표**: PSNR, SSIM

### C. 의료 바이오시그널(ECG/EEG/PPG)

- **문제**: 기저선 드리프트 제거, 근전도 잡음 억제, 특정 대역 활성 분석
- **방법**: 하이패스(IIR)로 천천히 변하는 드리프트 제거, 대역필터로 관심 대역 추출
- **지표**: 대역 에너지, 사건 관련 전위(ERP) 특성

```octave
pkg load signal
Fs = 250; t = (0:1/Fs:20-1/Fs)';
ecg = 0.5*sawtooth(2*pi*1.2*t, 0.5) + 0.05*randn(size(t)) + 0.002*t;
[b_hp, a_hp] = butter(3, 0.5/(Fs/2), 'high');   % 0.5 Hz 하이패스
ecg_corr = filtfilt(b_hp, a_hp, ecg);
```

### D. 레이다/소나·진동·산업 설비

- **문제**: 공진 주파수 측정, 이상(베어링 결함) 검출, 도플러 추정
- **방법**: 윈도잉+FFT, 대역통과, 시간-주파수 분석(STFT)
- **지표**: 피크 주파수, 스펙트럼 변화량, 엔벨로프 분석

```octave
pkg load signal
Fs=8000; N=4096; n=(0:N-1)'; t=n/Fs;
x = 0.2*sin(2*pi*250*t) + 0.05*randn(size(t));   % 공진 근처 신호
w = hanning(N);
X = fft(x.*w);
f = (0:N-1)*Fs/N;
plot(f(1:N/2), 20*log10(abs(X(1:N/2))+1e-12)); grid on; xlim([0,1000]);
xlabel('Hz'); ylabel('dB');
```

### E. 통신/SDR

- **문제**: 필터 뱅크·샘플레이트 변환, 동기화, 등화
- **방법**: 멀티레이트(업/다운샘플), 루프 필터, 매치드 필터

## GNU Octave 개관: MATLAB 없이도 충분히, 어디까지 가능한가

**GNU Octave**는 MATLAB과 높은 문법 호환성을 갖춘 **오픈소스 수치계산/시뮬레이션 환경**입니다. 교육·연구·프로토타이핑에 특히 유용합니다.

### 핵심 특징

- **무료/오픈소스(GPL)**: 라이선스 비용 없이 교육·개인·상업 초기 프로토타입에 활용
- **MATLAB 유사 문법**: 스크립트·함수·핸들 구조·플로팅 API가 대체로 호환
- **패키지 시스템**: `signal`, `image`, `control` 등 분야별 함수 제공
- **CLI/GUI 둘 다 지원**: 스크립팅 자동화와 대화형 작업에 모두 적합

```octave
pkg list               % 설치된 패키지 확인
pkg install -forge signal
pkg load signal
```

### MATLAB과의 차이/주의점

- **완전 호환은 아님**: Simulink, 일부 전용 Toolbox, 특정 클래스 시스템/핸들 그래픽 차이
- **성능/플로팅**: 대체로 충분하지만, 대형 데이터/고급 3D 그래픽은 차이가 날 수 있음
- **대안 생태계**: DSP에 필요한 핵심(FFT/윈도우/필터/스펙트로그램 등)은 `signal`로 대부분 충족

### 디렉토리/스크립트 워크플로

- 작업 폴더를 만들고, `.m` 스크립트와 데이터 파일을 관리
- 그래프·결과를 파일로 저장해 리포트/블로그에 반영

```octave
clear; close all; clc
Fs = 8000; t = (0:1/Fs:1-1/Fs);
x  = sin(2*pi*440*t) + 0.01*randn(size(t));
N  = 4096; w = hanning(min(N, numel(x)))';
xw = [x(1:length(w)) .* w, zeros(1, N-length(w))];
X  = fft(xw);
f  = (0:N-1)*Fs/N;
plot(f(1:N/2), abs(X(1:N/2))); xlabel('Hz'); ylabel('|X|'); grid on
print -dpng -r150 "spectrum_example.png"
```

### 빠른 시작: “만들고, 보고, 고친다”

1) **신호 만들기**: 합성 사인·펄스·노이즈
2) **보기**: 시간영역 파형, 주파수 스펙트럼(윈도우·NFFT 주의)
3) **고치기**: FIR/IIR 필터, 파라미터 스윕
4) **검증**: 응답(임펄스/스텝/치르프)과 지표(SNR/피크 등)

```octave
pkg load signal
Fs=16000; t=(0:1/Fs:1-1/Fs);
x = sin(2*pi*800*t) + 0.3*randn(size(t));

% 1) 시간영역
subplot(2,2,1); plot(t, x); xlim([0, 0.01]); title('Time'); xlabel('s');

% 2) FFT 스펙트럼
N=4096; w=hanning(N)';
xw=[x(1:N).*w, zeros(1,0)]; % 길이 맞추기
X=fft(xw, N); f=(0:N-1)*Fs/N;
subplot(2,2,2); plot(f(1:N/2), 20*log10(abs(X(1:N/2))+1e-12));
xlabel('Hz'); ylabel('dB'); title('Spectrum');

% 3) FIR 저역통과
b=fir1(200, 1200/(Fs/2), hamming(201));
y=filtfilt(b,1,x);
subplot(2,2,3); plot(t, [x(:), y(:)]); xlim([0, 0.01]); legend('x','y'); title('LPF result');

% 4) 전/후 스펙트럼 비교
Y=fft([y(1:N).*w, zeros(1,0)], N);
subplot(2,2,4); plot(f(1:N/2), 20*log10(abs(Y(1:N/2))+1e-12));
xlabel('Hz'); ylabel('dB'); title('Filtered Spectrum');
```

## 개념 핵심 체크(수식으로 요약)

- **표본화 주파수**
  $$ F_s \ge 2 f_{\max} $$
- **컨볼루션(선형 시불변 시스템 출력)**
  $$ y[n] = \sum_{k=-\infty}^{\infty} x[k]\,h[n-k] $$
- **DFT**
  $$ X[k] = \sum_{n=0}^{N-1} x[n] e^{-j\frac{2\pi}{N}kn} $$
- **주파수 해상도**
  $$ \Delta f = \frac{F_s}{N} $$

## 공부·실무 로드맵과 Octave로의 확장

- **초급**: 표본화/에일리어싱 → FFT/윈도우 → FIR/IIR 기초
- **중급**: STFT/스펙트로그램 → 멀티레이트(업/다운샘플) → 필터뱅크
- **응용**: 오디오 정화, 센서 이상탐지, 치르프/펄스 설계, 진동 진단
- **도구 확장**: Octave `signal` 중심으로, 필요시 `control`·`image` 패키지 병행
- **현업 이행**: Octave에서 프로토타입 → 임베디드(C/Fixed-Point)·파이프라인(배치/스트리밍) 이전

## 자주 묻는 질문(개관 범위 내)

- **Q. MATLAB 코드 그대로 Octave에서 돌아가나?**
  대부분의 기본 신호처리 코드는 동작합니다. 다만 특정 Toolbox 함수·핸들 그래픽·객체 지향 확장 등에서 차이가 있을 수 있어, 동일 함수의 존재 여부를 `help`/`which`로 확인하세요.

- **Q. 교육/블로그용 그림 품질은 충분한가?**
  2D 플롯·스펙트로그램·선형 필터 응답 등은 충분히 고품질로 산출 가능합니다. `print -dpng -r300` 등 출력을 조정하세요.

- **Q. 실시간 처리까지 가능한가?**
  Octave는 대화형/배치 분석에 강합니다. 하드 실시간 제어가 필요하면 C/C++·임베디드로 포팅하는 것이 보통의 경로입니다. 프로토타입 지식·계수·파라미터는 그대로 이전 가능합니다.

# 결론

- DSP는 **정보이론–수치해석–반도체** 진화가 교차한 산업 핵심 기술로 성장해 왔고, 오늘날엔 오디오·의료·산업·통신 전반에서 쓰입니다.
- **GNU Octave**는 비용 없이 학습·연구·초기 실무 검증에 충분한 기능을 제공하며, FFT/윈도우/필터/스펙트럼 분석 등 핵심 워크플로를 바로 실습할 수 있습니다.
- 이 개관을 바탕으로, 다음 글에서는 **샘플링/윈도잉/FFT**와 **FIR/IIR 설계**를 Octave 코드로 체계적으로 확장합니다.
