# HTML의 텍스트 구성 요소: 제목, 단락, 서식, 주석, 인용구

## 빠른 개요(한눈에)

| 범주 | 핵심 태그 | 핵심 포인트 |
|---|---|---|
| 제목(Heading) | `h1`~`h6` | 문서/섹션의 계층 구조. 시각적 크기 아닌 **의미**로 선택 |
| 단락(Paragraph) | `p`, `br` | 문단은 `p`, 강제 줄바꿈만 필요할 때 `br` |
| 서식(Formatting) | `strong`, `em`, `mark`, `del`, `ins`, `sub`, `sup`, `small`… | 의미 있는 강조 우선(`strong`, `em`), 단순 굵/기울임은 `b`, `i` |
| 주석(Comment) | `<!-- … -->` | 코드/문서 관리용 메모. 배포 전 민감정보 잔존 주의 |
| 인용(Quotation) | `blockquote`, `q`, `cite` | 블록 인용은 `blockquote`(+ `cite`), 인라인 인용은 `q` |

여기에 텍스트 관련 **보조 태그**까지 함께 쓰이면 품질이 올라갑니다:
`abbr`, `dfn`, `time`, `data`, `code`, `pre`, `kbd`, `samp`, `var`, `bdi`, `bdo`, `ruby`, `rt`, `rp`, `wbr` 등.

## — 문서 정보 구조의 뼈대

### 기본 사용

```html
<h1>가장 중요한 제목</h1>
<h2>하위 제목</h2>
<h3>더 하위 제목</h3>
<h4>...</h4>
<h5>...</h5>
<h6>가장 덜 중요한 제목</h6>
```

- 일반적으로 **문서의 주제는 `<h1>` 하나**로 표현합니다(페이지마다 1회 권장).
- 섹션 내부의 부제목은 `<h2>`부터 시작해서 **순차적 계층**을 유지합니다.
- 크기 조절은 CSS로 하고, **시맨틱은 h-레벨로 표현**합니다.

### 안티패턴 → 리팩터링

**나쁜 예(시각적 크기로 h-레벨 선택)**

```html
<h3 style="font-size:2rem">페이지 주제입니다</h3> <!-- h1이어야 함 -->
```

**좋은 예(의미로 선택하고, 크기는 CSS로)**

```html
<h1 class="title">페이지 주제입니다</h1>
```

```css
.title { font-size: 2rem; }
```

### 접근성·SEO 포인트

- **유일한 페이지 주제(h1)** 를 두고, h2/h3 등으로 **논리 계층**을 만든다.
- 스크린 리더는 헤딩 목록으로 **빠른 탐색**이 가능해진다.
- 헤딩 텍스트는 **키워드 남용 없이 정확한 의미**를 담는다.

## — 텍스트 블록의 기본 단위

### 기본 사용

```html
<p>이것은 하나의 문단입니다.</p>
<p>다른 문단이 여기에 있습니다.</p>
```

- `p`는 **의미적 문단**을 나타냅니다. 브라우저는 기본 여백을 적용합니다.

### 줄바꿈

```html
<p>줄 바꿈이<br>필요한 경우</p>
```

- 의미적 문단 분할은 `p`, **강제 줄바꿈만** 필요하면 `br`을 사용합니다.
- 연속 공백/개행은 HTML 파서가 정규화하므로, 시각적 줄나눔은 CSS로 제어하세요.

### 화이트스페이스/개행 제어

```html
<pre>    앞 공백이 유지되고
개행도
그대로</pre>

<p class="keepws">    공백 보존하고 개행도 유지하고 싶다</p>
```

```css
.keepws { white-space: pre-wrap; }
```

- 긴 코드/로그는 `pre`+`code` 조합이 적합합니다(아래 3장 참고).

## — 의미 있는 강조가 우선

### 의미 있는 서식 태그 표

| 태그 | 의미·용도 | 예시 |
|---|---|---|
| `strong` | **중요**한 강조(의미 있음) | 경고/핵심 |
| `em` | **강조**(강세, 의미 있음) | 문맥상 강조 |
| `b` | 단순 굵게(의미 없음) | 스타일만 필요할 때 |
| `i` | 단순 기울임(의미 없음) | 라틴어, 외래어, 서체적 구분 |
| `mark` | 하이라이트 | 검색 결과 강조 |
| `del` | 삭제된 내용(취소선) | 변경 이력 |
| `ins` | 추가된 내용(밑줄) | 변경 이력 |
| `u` | 밑줄(의미 없음) | 관습적 표기(주의: 링크 오인 가능) |
| `small` | 작은 텍스트 | 주석적 정보, 면책 문구 |
| `sub` | 아래 첨자 | H<sub>2</sub>O |
| `sup` | 위 첨자 | x<sup>2</sup> |

### 예제

```html
<p><strong>중요한</strong> 공지입니다.</p>
<p><em>의미 있는 기울임</em>으로 강조합니다.</p>
<p><mark>하이라이트</mark>된 텍스트입니다.</p>
<p><del>삭제된 내용</del>과 <ins>추가된 내용</ins>을 표시합니다.</p>
<p>물의 화학식은 H<sub>2</sub>O, 제곱은 x<sup>2</sup> 입니다.</p>
```

### 코드/입력/출력/변수 표기(텍스트 문서에 자주 필요)

```html
<p>명령은 <code>npm run build</code> 를 실행합니다.</p>
<pre><code>git clone https://example.com/repo.git
cd repo
npm i
</code></pre>
<p>사용자 입력은 <kbd>Ctrl</kbd> + <kbd>S</kbd> 로 저장합니다.</p>
<p>프로그램 출력 예: <samp>Saved successfully.</samp></p>
<p>수식에서 변수는 <var>n</var> 으로 표기합니다.</p>
```

> 접근성 관점에서 `<code>`, `<kbd>`, `<samp>`, `<var>`는 의미 분리를 돕습니다.

### 수학 표기(참고: 블로그 렌더러가 MathJax 지원 시)

- 아래와 같이 **수식은 반드시 $$로 감싸** 노출합니다.

$$
E = mc^2,\quad \int_a^b f(x)\,dx,\quad x^2 + y^2 = r^2
$$

> 위 수식은 HTML 서식 태그가 아닌 렌더링 파이프라인(MathJax 등)에 의해 처리됩니다.
> 단, 일반 텍스트 안의 간단한 첨자는 `sub/sup`로 충분합니다.

## — 문서를 관리하는 개발자 메모

### 기본

```html
<!-- 이것은 주석입니다. -->
<p>화면에 보이는 내용</p>
<!--
  여러 줄의 주석도 가능합니다.
  TODO: 이 섹션에 다이어그램 추가
-->
```

- 브라우저 화면에는 보이지 않지만, **소스 보기**로는 노출됩니다.
- **민감 정보**(API 키, 내부 경로, 이메일 등)를 주석에 남기지 마세요.
- HTML 주석 안에 연속 `--` 는 허용되지 않습니다.

**잘못된 예**

```html
<!-- wrong -- comment --> <!-- 파서 오류 위험 -->
```

## — 출처와 맥락까지 전달

### 블록 인용: `blockquote`

```html
<blockquote cite="https://example.com/article">
  웹은 정보 공유의 공간입니다.
</blockquote>
```

- 한 단락 이상의 **블록 단위 인용**에 적합합니다.
- `cite` 속성으로 **출처 URL**을 제공할 수 있습니다(노출은 의무 아님).
- 출처 텍스트를 함께 보여주려면 다음과 같이 구조화합니다.

```html
<figure>
  <blockquote>
    정보는 나눌수록 커진다.
  </blockquote>
  <figcaption>
    — <cite>웹의 철학</cite>, 팀 버너스리
  </figcaption>
</figure>
```

### 인라인 인용: `q`

```html
<p>그는 <q>시간은 금이다</q>라고 말했다.</p>
```

- **짧은 인라인 인용**에 적합합니다. 브라우저가 따옴표를 렌더링합니다(로캘에 따라 다를 수 있음).

### 인용 대상 표기: `cite`

```html
<p><cite>웹의 역사</cite>는 팀 버너스리가 썼다.</p>
```

- 책/문서/영화/작품 등의 **제목**을 표기(브라우저 기본 렌더링은 보통 기울임).

## 텍스트 보조 태그 — 문서 의미와 국제화 품질을 끌어올리는 요소들

### 약어/정의: `abbr`, `dfn`

```html
<p><abbr title="Search Engine Optimization">SEO</abbr>를 고려한 문서 구조</p>
<p><dfn id="def-article">article</dfn>은 독립적으로 배포 가능한 콘텐츠 블록입니다.</p>
```

### 시간/데이터: `time`, `data`

```html
<p>게시일: <time datetime="2025-11-09">2025-11-09</time></p>
<p>가격: <data value="39900">39,900원</data></p>
```

### 양방향 텍스트: `bdi`, `bdo`

```html
<p>사용자: <bdi>علي</bdi></p>
<p><bdo dir="rtl">ABC</bdo></p>
```

- `bdi`는 주변 텍스트 방향과 독립적으로 **이름/식별자**를 안전하게 표시합니다.
- `bdo`는 텍스트 방향을 **강제**합니다(특수한 경우만 사용).

### 루비 표기(동아시아): `ruby`, `rt`, `rp`

```html
<ruby>
  漢 <rt>한</rt>
  字 <rt>자</rt>
</ruby>
```

- `rp`는 루비를 지원하지 않는 브라우저에서 괄호 대체 텍스트를 제공할 때 사용합니다.

### 단어 줄바꿈 힌트: `wbr`

```html
<p>supercalifragilistic<wbr>expialidocious</p>
```

- 너무 긴 단어에 **적절한 줄바꿈 지점**을 힌트로 주어 레이아웃 깨짐을 방지합니다.

## 실전 텍스트 패턴 모음

### FAQ(접이식 상세)

```html
<section aria-labelledby="faq">
  <h2 id="faq">자주 묻는 질문</h2>
  <details>
    <summary>HTML은 프로그래밍 언어인가요?</summary>
    <p>아닙니다. 문서의 구조/의미를 표현하는 마크업 언어입니다.</p>
  </details>
  <details>
    <summary>제목 태그는 어떻게 고르나요?</summary>
    <p>문서/섹션 계층 구조에 따라 h1→h2→h3 순서를 지키세요.</p>
  </details>
</section>
```

### 경고/알림 블록(단순 HTML 버전)

```html
<aside role="note" aria-label="주의">
  <p><strong>주의:</strong> 외부 링크는 새 창을 열 경우 <code>rel="noopener noreferrer"</code>를 함께 사용하세요.</p>
</aside>
```

### 변경 이력

```html
<p>버전: <del>1.0</del> <ins datetime="2025-09-01">1.1</ins></p>
```

## 요점 정리

- **의미로 태그 선택**: 강조는 `strong/em`, 클릭 동작은 `button`, 탐색은 `a`.
- **헤딩 구조**: h1은 페이지 주제, 하위 섹션은 h2~h3 순차적.
- **대체 텍스트**: 의미 있는 이미지엔 `alt` 제공. 장식용 이미지는 `alt=""`.
- **인용**: `blockquote`+`figcaption`+`cite`로 출처를 명확히.
- **키보드 탐색**: 텍스트 링크는 포커스 가능한 `<a>`를 사용하고, 스크립트로 `div`를 클릭가능하게 만드는 패턴은 피합니다.

## SEO와 정보 구조

- **고유한 `<title>` + 설명 메타**를 유지합니다.
- **헤딩 텍스트에 문서 핵심 키워드**를 자연스럽게 포함(남용 금지).
- 인용/정의/약어/시간/데이터 태그는 **의미 정보를 풍부하게** 하여 검색 이해도를 높입니다.
- 복제/유사 페이지엔 **정규 URL(canonical)** 을 지정합니다.

```html
<head>
  <title>HTML 텍스트 구성 요소 완전 정리</title>
  <meta name="description" content="제목/단락/서식/주석/인용구와 보조 태그까지 실무 예제로 정리" />
  <link rel="canonical" href="https://example.com/posts/html-text" />
</head>
```

## 국제화(i18n)·타이포그래피 고려

- 문서 루트에 **언어** 지정: `<html lang="ko">`
- 숫자/날짜는 `<time>`로 표준화, 다국어 UI에 대비.
- 긴 조합어/영문 용어는 `wbr`로 줄바꿈 힌트.
- 동아시아 읽기 보조는 `ruby` 사용.

## 성능·보안(텍스트 맥락에서)

- 텍스트만 담는 페이지라도 이미지/폰트가 있다면 **지연 로딩**과 **preload**를 적절히 활용.
- 사용자 입력을 텍스트 노드로 삽입할 땐 `textContent` 사용으로 **XSS** 차단.
- 외부 링크 새창(`target="_blank"`)엔 `rel="noopener noreferrer"`.

```html
<div id="out"></div>
<script>
  const user = "<img src=x onerror=alert(1)>";
  // 안전: 태그로 해석되지 않음
  document.getElementById('out').textContent = user;
</script>
```

<body>
  <header>
    <h1 class="page-title">HTML 텍스트 구성 요소 모범 사례</h1>

      <a href="/">홈</a>
      <a href="/posts">글</a>
      <a href="/about">소개</a>
    </nav>
  </header>

  <main id="content">
    <article>
      <header>
        <h2>개요</h2>
        <p><time datetime="2025-11-09">2025-11-09</time></p>
      </header>

      <section>
        <h3>제목과 단락</h3>
        <p>문서 주제는 <strong>의미로 고른</strong> <code>h1</code>이 담당합니다.</p>
        <p>문단은 <code>p</code>로, 강제 줄바꿈만 필요하면 <code>br</code>을 씁니다.</p>
      </section>

        <h3>서식</h3>

      <section>

        </figure>

      </section>

      <section>
        <h3>보조 태그</h3>

        <p>게시일: <time datetime="2025-11-09">2025-11-09</time>, 가격: <data value="39900">39,900원</data></p>
        <p>사용자: <bdi>علي</bdi>, 방향 강제: <bdo dir="rtl">ABC</bdo></p>
        <p><ruby>漢<rt>한</rt>字<rt>자</rt></ruby></p>
      </section>

      <footer>
        <p>태그: <a href="/tags/html">#html</a>, <a href="/tags/semantics">#semantics</a></p>
      </footer>
    </article>
  </main>

  <footer>
    <small>© 2025</small>
  </footer>

  <!-- 배포 전 민감 정보가 주석에 남아 있지 않은지 확인하세요 -->
</body>
</html>

## 적용 체크리스트

- [ ] 문서 언어/인코딩/뷰포트: `<html lang>`, `<meta charset>`, `<meta name="viewport">`
- [ ] 헤딩 계층: h1은 페이지 주제, 하위는 h2→h3…
- [ ] 문단/줄바꿈: 문단은 `p`, 줄바꿈은 `br`, 장문/코드는 `pre`+`code`
- [ ] 의미 강조: `strong/em` 우선, `b/i`는 스타일용일 때만
- [ ] 변경 이력: `del/ins`, 중요도 낮은 텍스트는 `small`
- [ ] 인용: 블록은 `blockquote`(+`figcaption`/`cite`), 인라인은 `q`
- [ ] 보조 태그: `abbr/dfn/time/data/code/kbd/samp/var/bdi/bdo/ruby/wbr`
- [ ] 접근성: 링크는 `<a>`, 동작 트리거는 `<button>`, 대체 텍스트/헤딩 구조 점검
- [ ] SEO: `<title>`, `description`, 헤딩 텍스트 의미 최적화, `canonical`
- [ ] 보안: 사용자 입력은 `textContent`, 외부 새창 링크엔 `rel="noopener noreferrer"`

---

## 흔한 함정과 해결

1) **시각적 크기=헤딩 레벨**로 착각
→ 헤딩은 **문서 구조**, 크기는 **CSS**로.

2) **`div`만으로 문서 작성**
→ 인용은 `blockquote`, 용어는 `dfn`, 약어는 `abbr`, 시간은 `time` 등 **의미 태그** 활용.

3) **`u`로 밑줄 남발**
→ 링크로 오인 가능. 의미가 없다면 다른 시각적 강조로 대체.

4) **긴 영어/URL 줄바꿈 문제**
→ `wbr`로 줄바꿈 힌트, CSS의 `overflow-wrap: anywhere;` 병행.

5) **주석에 민감 정보**
→ 배포 전 빌드/검수 단계에서 **정적 점검**(lint)으로 제거.

## 짧은 실습: 초안 리팩터링

**초안**

```html
<h3 style="font-size:2rem">문서 주제</h3>
<p>여기는 내용입니다.
다음 줄입니다.</p>
<div><b>주의</b>: 외부 링크는 새 창으로 열립니다.</div>
```

**리팩터링**

```html
<h1 class="title">문서 주제</h1>
<p>여기는 내용입니다.<br>다음 줄입니다.</p>
<aside role="note" aria-label="주의">
  <p><strong>주의:</strong> 외부 링크를 새 창으로 열 경우 <code>rel="noopener noreferrer"</code>를 사용하세요.</p>
</aside>
```

```css
.title { font-size: 2rem; }
```
